    # 150.哈希表题目

    # 总结

    哈希表可以实现元素的快速查找, 所以有需要查找的场景可以用到此方法.

    哈希表会占用一定的额外空间, 属于空间换时间的操作.

    # 有效的字母异位词(242-简单)🌸💯

    示例 1: 输入: s = "anagram", t = "nagaram"  输出: true

示例 2: 输入: s = "rat", t = "car"  输出: false
    1
    2
    3

    说明: 可以假设字符串只包含小写字母.

    https://leetcode.cn/problems/valid-anagram/description/ (opens new window)

    # 字符计数法

    数组其实就是一个简单哈希表, 而且这道题目中字符串只有小写字符, 那么就可以定义一个数组, 来记录字符串 s 里字符出现的次数. 需要定义一个多大的数组呢, 定一个数组叫做 record, 大小为 26 就可以了, 初始化为 0, 因为字符 a 到字符 z 的 ASCII 也是 26 个连续的数值. 为了方便举例, 判断一下字符串 s = "monorepo", t = "repomono".

    ​​

    定义一个数组用来上记录字符串 s 里字符出现的次数.

    需要把字符映射到数组也就是哈希表的索引下标上, 因为字符 a 到字符 z 的 ASCII 是 26 个连续的数值, 所以字符 a 映射为下标 0, 相应的字符 z 映射为下标 25.

    再遍历字符串 s 的时候, 只需要将 s[i] - 'a' 所在的元素做 +1 操作即可, 并不需要记住字符 a 的 ASCII, 只要求出一个相对数值就可以了. 这样就将字符串 s 中字符出现的次数, 统计出来了.

    那看一下如何检查字符串 t 中是否出现了这些字符, 同样在遍历字符串 t 的时候, 对 t 中出现的字符映射哈希表索引上的数值再做 -1 的操作.

    那么最后检查一下, record 数组如果有的元素不为零 0, 说明字符串 s 和 t 一定是谁多了字符或者谁少了字符, return false. 最后如果 record 数组所有元素都为零 0, 说明字符串 s 和 t 是字母异位词, return true.

    代码如下:

    public boolean isAnagram(String s, String t) {
    if (s.length() != t.length()) {
        return false;
    }

    // 字母表
    int[] counter = new int[26];

    for (int i = 0; i < s.length(); i++) {
        counter[s.charAt(i) - 'a']++;
        counter[t.charAt(i) - 'a']--;
    }
    for (int count : counter) {
        if (count != 0) return false;
    }
    return true;
}

    # 字符计数法

    此题是上一个题 有效的字母异位词(242-简单) 的升级版本.

    由于互为字母异位词的两个字符串相同字母出现的次数一定是相同的, 故可以将每个字母出现的次数使用字符串表示, 并作为哈希表的键, 然后按照键进行哈希分组即可.

    public List<List<String>> groupAnagrams(String[] strs) {
    Map<String, List<String>> map = new HashMap<>();
    for (String str : strs) {
        // 获取当前字符串的字母技术Key
        String key = getCntKey(str);
        List<String> list = map.getOrDefault(key, new ArrayList<>());
        list.add(str);
        map.put(key, list);
    }
  
    return new ArrayList<>(map.values());
}

private String getCntKey(String str) {
    int[] cnt = new int[26];
    for (char c : str.toCharArray()) {
        cnt[c - 'a']++;
    }
    return new String(cnt, 0, cnt.length);
}

    # 字符计数法

    与 有效的字母异位词(242-简单) 的思路基本一样. 只不过本题允许字符有剩余没有用完的, 而不允许用超.

    代码如下:

    public boolean canConstruct(String ransomNote, String magazine) {
    if (ransomNote.length() > magazine.length()) {
        return false;
    }

    // 字母表
    int[] counter = new int[26];

    // 统计magazine中的字符的个数
    for (int i = 0; i < magazine.length(); i++) {
        counter[magazine.charAt(i) - 'a']++;
    }

    // 遍历ransomNote, 每次减少一个字符
    for (int i = 0; i < ransomNote.length(); i++) {
        counter[ransomNote.charAt(i) - 'a']--;
    }

    for (int count : counter) {
        if (count < 0) {
            return false;
        }
    }

    return true;
}

    # 字符串中的第一个唯一字符(387-简单)🌸🔑
    # 题目描述

    在一个字符串中找到第一个只出现一次的字符, 并返回它的位置.

    输入: abacc  输出: b
    1

    https://leetcode.cn/problems/first-unique-character-in-a-string/description/

    # 哈希计数法

    与 有效的字母异位词(242-简单) 的思路基本一样. 遍历两次, 第一次遍历用长度 26 字符的数组统计每个字符出现的次数, 第二次遍历, 找到第一个次数为 1 的字符即可.

    代码如下:

    public int firstUniqChar(String s) {
    if (s == null || s.isEmpty()) {
        return -1;
    }

    // 记录每个字符出现的次数
    int[] counter = new int[26];
  
    // 统计每个字符出现的次数
    for (char c : s.toCharArray()) {
        counter[c - 'a']++;
    }

    // 找到第一个出现次数为1的字符
    for (char c : s.toCharArray()) {
        if (counter[c - 'a'] == 1) {
            return s.indexOf(c);
        }
    }

    return -1;
}

    # 最长连续数字序列(128-中等)🌸💯
    # 题目描述

    给定一个未排序的整数数组 nums, 找出数字连续的最长序列(不要求序列元素在原数组中连续)的长度.

    请实现时间复杂度为 O(n) 的算法解决此问题.

    示例 1:  输入: nums = [100,4,200,1,3,2]   输出: 4  
解释: 最长数字连续序列是 [1, 2, 3, 4]. 它的长度为 4. 
示例 2:  
输入: nums = [0,3,7,2,5,8,4,6,0,1]   输出: 9
    1
    2
    3
    4
    # 哈希法

    注意要求用 O(N) 的复杂度实现, 时间要求高, 可能就要用到额外空间了, 空间换时间.

    为了能够快速获取到某个数是否存在, 可以使用哈希的方式, 将不同的数全部放到哈希表 HashSet 中, 再次获取时即可用常量时间.

    只需要逐一遍历 HashSet 中的元素, 然后找出当前元素的连续序列, 逐一找到所有元素的最长连续序列即可. 由于所有元素都在 HashSet 中, 所以找连续的元素是很容易的.

    ​​

    注意: 这里存在一个剪枝的操作. 比如计算 4 的最长序列时, 先看一下比 4 小的元素是否存在, 可以发现 3 是存在的, 这时就可以直接跳过 4 的处理. 因为前一个元素存在时, 前一个元素的最长序列必然比当前元素更长.

    public int longestConsecutive(int[] nums) {
    if (nums <mark> null || nums.length </mark> 0) {
        return 0;
    }

    Set<Integer> numSet = new HashSet<>();
    for (int num : nums) {
        numSet.add(num);
    }

    int maxLen = 0;
    for (int num : numSet) {
        maxLen = Math.max(maxLen, getMaxLen(numSet, num));
    }

    return maxLen;
}

private int getMaxLen(Set<Integer> numSet, int num) {
    // 剪枝操作, 如果当前数字的前一个数字存在则直接返回最小值
    if (numSet.contains(num - 1)) {
        return Integer.MIN_VALUE;
    }

    // 找出当前数字开始的最长连续序列
    int curLen = 0;
    while (numSet.contains(num)) {
        curLen++;
        num++;
    }
    return curLen;
}

    # 哈希法

    题目中说了会无限循环, 那么也就是说求和的过程中, sum 会重复出现, 当遇到了要快速判断一个元素是否出现集合里的时候, 就要考虑哈希法了.

    对于一个数计算其各个数字的平方和为 sum, 使用 HashSet 来保存出现过的 sum, 如果重复了说明就会死循环, 这样就可以结束循环, 否则一直找到 sum 为 1 为止.

    代码如下:

    public boolean isHappy(int n) {
    Set<Integer> numSet = new HashSet<>();
    while (true) {
        // 计算下一个数
        n = getNextNum(n);
        // 如果等于1则是快乐数
        if (n == 1) {
            return true;
        }
        // 如果出现重复数字则不是快乐数, 直接返回否则会陷入死循环
        if (numSet.contains(n)) {
            return false;
        }
        numSet.add(n);
    }
}

private int getNextNum(int n) {
    int res = 0;
    while (n > 0) {
        int temp = n % 10;
        res += temp * temp;
        n = n / 10;
    }
    return res;
}

    # Hash表法

    这道题本身如果通过暴力遍历的话也是很容易解决的, 时间复杂度在 O(n2). 由于哈希查找的时间复杂度为 O(1), 所以可以利用哈希容器 map 降低时间复杂度, 属于空间换时间的思想.

    遍历数组 nums, i 为当前下标, 每个值都判断 map 中是否存在 target - nums[i] 的 key 值. 如果存在则找到了两个值, 如果不存在则将当前的 (nums[i], i) 存入 map 中, 继续遍历直到找到为止.

    ​​

    代码如下:

    public int[] twoSum(int[] nums, int target) {
    Map<Integer, Integer> value2Index = new HashMap<>();

    for (int i = 0; i < nums.length; i++) {
        // 遍历当前元素, 并在 map 中寻找是否有匹配的 key
        if (value2Index.containsKey(target - nums[i])) {
            return new int[]{value2Index.get(target - nums[i]), i};
        }
        // 如果没找到匹配对, 就把访问过的元素和下标加入到 map 中
        value2Index.put(nums[i], i);
    }

    return new int[]{-1, -1};
}

    # 双指针法

    注意: 此法不能用于上面的原题, 只能用于找出和为 target 的两个数, 因为排序已经把顺序打乱了

    双指针遍历: 先排序, 然后用头尾两个指针, 分别开始遍历, 两个数字和大于 target 时, 右指针向前移动, 小于 target 时左指针向后移动. 直到两个指针相遇.

    public int[] twoSum(int[] nums, int target) {
    // Base case
    if(nums <mark> null || nums.length </mark> 0) return new int[]{};
    // 排个序
    Arrays.sort(nums);
    int i = 0;
    int j = nums.length - 1;
    // 左右指针不相遇
    while(i < j) {
        int sum = nums[i] + nums[j];
        if(sum > target) {
            j--;
        } else if(sum < target) {
            i++;
        } else {
            // 返回这两个数
            return new int[]{nums[i], nums[j]};
        }
    }
    return new int[]{};
}

    # 哈希法

    将单词字符串按照空格进行切分, 切分后单词数量应该与字符序列长度一致.

    维护两个 HashMap, 字符到单词的映射 char2Word, 单词到字符的映射 word2Char. 逐一遍历获取对应位置的字符和单词, 如果两个 map 中不存在对应的映射则加入到对应的 map, 否则对比映射关系是否一致, 不一致直接不合法.

    ​​

    代码如下:

    public boolean wordPattern(String pattern, String s) {
    if (pattern <mark> null || s </mark> null) {
        return false;
    }

    // 字符串按照空格分隔
    String[] words = s.split(" ");

        if (char2Word.containsKey(c) && !char2Word.get(c).equals(words[i])) {

            return false;
        } else {

        }
    }

    # 题目描述

    在一个长度为 n 的数组里的所有数字都在 0 到 n-1 的范围内. 数组中某些数字是重复的, 但不知道有几个数字是重复的, 也不知道每个数字重复几次. 请找出数组中任意一个重复的数字.

    # 数组哈希法

    这个题就是数组大小和数的范围都给了. 要求时间复杂度 O(N) , 空间复杂度 O(1) . 因此不能使用排序的方法, 也不能使用额外的标记数组.

    对于这种数组元素在 [0, n - 1] 范围内的问题, 可以将值为 i 的元素调整到第 i 个位置上进行求解. 如果当前位置的值不等于索引则不断进行交换, 如果交换之后已经存在, 说明重复了, 这其实就是在​利用数组自身进行哈希.

    以 (2, 3, 1, 0, 2, 5) 为例, 遍历到位置 4 时, 该位置上的数为 2, 但是第 2 个位置上已经有一个 2 的值了, 因此可以知道 2 重复:

    ​​

    public int findRepeatNumber(int[] nums) {
    if (nums == null || nums.length <= 0) {
        return -1;
    }
    // 遍历数组
    for (int i = 0; i < nums.length; i++) {
        // 如果当前位置不等于索引则不断进行交换
        while (nums[i] != i) {
            // 如果当前的索引位置已经有元素了则说明重复了
            if (nums[i] == nums[nums[i]]) {
                return nums[i];
            }
            // 交换元素
            swap(nums, i, nums[i]);
        }
    }

    return -1;
}

private void swap(int[] nums, int i, int j) {
    int t = nums[i];
    nums[i] = nums[j];
    nums[j] = t;
}

    # 哈希法

    利用值域与数字下标空间大小的等同关系，可以构造一种对应关系，使得每个数出现在它应该出现的位置：对于值为 k 的数字，将其应该出现在的位置定为 k − 1。

    基于此，可以从前往后遍历 nums，并尝试将当前处理到的 nums[i] 放到目标位置 nums[i] − 1 处。如果一个数在尝试移动到它应该出现的位置时，发现 nums[nums[i] − 1] = nums[i]​，则说明该数字出现了超过一次。此时将 nums[i] 加入答案，由于此时没有发生交换，而 nums[i] 占用的仍是其他数字的目标位置，为了防止 nums[i] 与其他数字发生交换后，再次被检验并重复加入答案，所以将 nums[i] 置为负数，并在遍历过程中跳过负数。

    public List<Integer> findDuplicates(int[] nums) {
    List<Integer> duplicates = new ArrayList<>();

    for (int i = 0; i < nums.length; i++) {
        int num = nums[i];
        // 计算索引
        int index = Math.abs(num) - 1;

        // 如果是正数则转为负数, 表示第一次遇到
        if (nums[index] > 0) {
            nums[index] = -nums[index];
        } else {
            // 否则说明已经遇到过了, 说明是重复元素
            duplicates.add(index + 1);
        }
    }
  
    return duplicates;
}

    # 原地哈希法

    题目要求只能使用常数级别的空间，而要找的数一定在 [1, N + 1] 左闭右闭（这里 N 是数组的长度）这个区间里。因此，可以就把原始的数组当做哈希表来使用。事实上，哈希表其实本身也是一个数组；

    要找的数就在 [1, N + 1] 里，最后 N + 1 这个元素不用找。因为在前面的 N 个元素都找不到的情况下，才返回 N + 1。

    那么可以采取这样的思路：就把 1 这个数放到下标为 0 的位置，2 这个数放到下标为 1 的位置，按照这种思路整理一遍数组。然后再遍历一次数组，第 1 个遇到的它的值不等于下标的那个数，就是要找的缺失的第一个正数。
    这个思想就相当于自己编写哈希函数，这个哈希函数的规则特别简单，那就是数值为 i 的数映射到下标为 i - 1 的位置。

    ​​

    代码如下：

    public int firstMissingPositive(int[] nums) {
    int len = nums.length;

    // 原地哈希, 数值为 i 的数映射到下标为 i - 1 的位置
    for (int i = 0; i < len; i++) {
        while (nums[i] > 0 && nums[i] <= len && nums[nums[i] - 1] != nums[i]) {
            // 满足在指定范围内, 并且没有放在正确的位置上, 才交换
            // 例如: 数值 3 应该放在索引 2 的位置上
            swap(nums, nums[i] - 1, i);
        }
    }

    // 找到第一个不符合要求的索引
    for (int i = 0; i < len; i++) {
        if (nums[i] != i + 1) {
            return i + 1;
        }
    }

    // 都正确则返回数组长度 +1
    return len + 1;
}

private void swap(int[] nums, int m, int n) {
    int temp = nums[m];
    nums[m] = nums[n];
    nums[n] = temp;
}

    # 哈希表+前缀和法

    前缀和的定义：s[0] = 0, s[i + 1] = nums[0] + nums[1] + ... + nums[i]。

    下标从 i 到 j − 1 的非空连续子数组的元素和等于 k，即:

    s[j] − s[i] = k
    1

    枚举 j，上式变成:

    s[i] = s[j] − k
    1

    根据上式，计算 s[i] 的个数，等价于计算在 s[j] 左边的 s[j] − k 的个数。

    这可以在遍历 s[j] 的同时，用一个哈希表 cnt 统计 s[j] 的个数。那么枚举到 s[j] 时，从哈希表中就可以找到有 cnt[s[j] − k] 个 s[i]，即为元素和等于 k 的子数组个数，加入答案。

    以 nums = [1,1,−1,1,−1], k = 1 为例，其前缀和 s = [0,1,2,1,2,1]。

    ​​

    用哈希表记录前缀和的个数，就能 O(1) 算出左边有多少个 s[j] − k。

    因此总计有 0+1+1+1+2+1=6 个和为 k = 1 的子数组。

    代码如下：

    public int subarraySum(int[] nums, int k) {
    int len = nums.length;
    // 构造前缀和数组
    int[] s = new int[len + 1];
    for (int i = 0; i < len; i++) {
        s[i + 1] = s[i] + nums[i];
    }

    int res = 0;
    // 使用HashMap记录每个前缀和出现的次数
    Map<Integer, Integer> sum2Cnt = new HashMap<>(len + 1);

    // 遍历前缀和数组
    for (int sj : s) {
        // 如果有某个前缀和值s[i]使得当前前缀和减去该值等于k, 满足条件则记录这样的子数组数
        res += sum2Cnt.getOrDefault(sj - k, 0);

        // 当前前缀和出现的次数+1
        sum2Cnt.put(sj, sum2Cnt.getOrDefault(sj, 0) + 1);
    }
  
    return res;
}

    # 相邻元素计数法

    本题注意就是求两个相邻元素个数的最大值。注意必须是相邻的两个数，只有一个数则不合法。

    因此，先使用哈希表统计所有元素出现的个数。然后遍历所有的元素，如果当前元素 + 1 也存在，说明这两个元素是相邻的，则计算两个元素出现的次数和。

    public int findLHS(int[] nums) {
    // 计数的 map
    Map<Integer, Integer> cntMap = new HashMap<>();
    // 统计所有元素出现的次数
    for (int num : nums) {
        cntMap.put(num, cntMap.getOrDefault(num, 0) + 1);
    }
  
    int res = 0;
    // 遍历计算当前元素及其下一个元素出现的次数和(因为和谐序列差值最大为1)
    for (int num : cntMap.keySet()) {
        if (cntMap.containsKey(num + 1)) {
            res = Math.max(res, cntMap.get(num + 1) + cntMap.get(num));
        }
    }
  
    return res;
}

    # 哈希表法

    算法如下：

    1. 统计每个元素的频数：首先统计数组中每个元素的出现次数，以确定数组的度。这里用一个 map 来记录。
    2. 记录每个元素的起始和结束位置：记录每个元素在数组中第一次和最后一次出现的位置，以便计算包含该元素的最短连续子数组的长度。这里用两个 map 来分别记录某个元素第一次和最后一次出现的位置。
    3. 找到频数最大的元素：找到频数最大的元素。
    4. 计算最短子数组的长度：对于所有频数最大的元素，计算其对应的最短连续子数组的长度(也就是从两个左右位置 map 中获取其范围)，并取最小值。

    代码如下：

    public int findShortestSubArray(int[] nums) {
    // 用于记录每个元素的频数
    Map<Integer, Integer> num2Count = new HashMap<>();
    // 用于记录每个元素第一次出现的位置
    Map<Integer, Integer> num2Left = new HashMap<>();
    // 用于记录每个元素最后一次出现的位置
    Map<Integer, Integer> num2Right = new HashMap<>();

    // 遍历数组,填充上述三个Map
    for (int i = 0; i < nums.length; i++) {
        int num = nums[i];
        // 记录元素第一次出现的位置
        if (!num2Left.containsKey(num)) {
            num2Left.put(num, i); 
        }
        // 更新元素最后一次出现的位置
        num2Right.put(num, i);
        // 更新元素出现的次数
        num2Count.put(num, num2Count.getOrDefault(num, 0) + 1); 
    }

    // 找到数组的度(即最大频数)
    int maxDegree = 0;
    for (int count : num2Count.values()) {
        maxDegree = Math.max(maxDegree, count);
    }

    // 找到所有频数等于度的元素, 并找出最短子数组的长度
    int minLength = Integer.MAX_VALUE;
  
    for (int num : num2Count.keySet()) {
        if (num2Count.get(num) != maxDegree) {
            continue;
        }
        // 根据元素出现的左右范围找出当前元素的长度并更新最短结果
        int length = num2Right.get(num) - num2Left.get(num) + 1;
        minLength = Math.min(minLength, length);
    }

    return minLength;
}
