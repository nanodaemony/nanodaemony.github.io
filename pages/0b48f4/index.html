    # 深度优先搜索理论基础🌸
    # dfs与bfs区别

    提到深度优先搜索(dfs), 就不得不说和广度优先搜索(bfs)有什么区别:

    • dfs 是可一个方向去搜, 不到黄河不回头, 直到遇到绝境了, 搜不下去了, 再换方向(换方向的过程就涉及到了回溯).
    • bfs 是先把本节点所连接的所有节点遍历一遍, 走到下一个节点的时候, 再把连接节点的所有节点遍历一遍, 搜索方向更像是广度, 四面八方的搜索过程.

    # 广度优先搜索理论基础🌸

    广搜(bfs)是一圈一圈的搜索过程, 和深搜(dfs)是一条路跑到黑然后再回溯.

    # 广搜的使用场景

    广搜的搜索方式就适合于解决两个点之间的最短路径问题. 因为广搜是从起点出发, 以起始点为中心一圈一圈进行搜索, 一旦遇到终点, 记录之前走过的节点就是一条最短路.

    当然, 也有一些问题是广搜 和 深搜都可以解决的, 例如岛屿问题, 这类问题的特征就是不涉及具体的遍历方式, 只要能把相邻且相同属性的节点标记上就行.

    举个例子, 用一个方格地图, 假如每次搜索的方向为 上下左右(不包含斜上方), 那么给出一个 start 起始位置, 那么 BFS 就是从四个方向走出第一步.

    # 代码框架

    大家应该好奇, 这一圈一圈的搜索过程是怎么做到的, 是放在什么容器里, 才能这样去遍历.

    很多资料都是直接说用队列来实现. 其实, 仅仅需要一个容器, 能保存要遍历过的元素就可以, 那么用队列, 还是用栈, 甚至用数组, 都是可以的.

    用队列的话, 就是保证每一圈都是一个方向去转, 例如统一顺时针或者逆时针. 因为队列是先进先出, 加入元素和弹出元素的顺序是没有改变的. 如果用栈的话, 就是第一圈顺时针遍历, 第二圈逆时针遍历, 第三圈有顺时针遍历. 因为栈是先进后出, 加入元素和弹出元素的顺序改变了.

    那么广搜需要注意转圈搜索的顺序吗? 不需要! 所以用队列, 还是用栈都是可以的, 但大家都习惯用队列了, 所以下面也用队列来讲, 但要清楚并不是非要用队列, 用栈也可以.

    # 深度优先搜索法

    使用深度优先搜索进行求解. 回溯是深度优先搜索的一种特例, 它在一次搜索过程中需要设置一些本次搜索过程的局部状态, 并在本次搜索结束之后清除状态. 而普通的深度优先搜索并不需要使用这些局部状态, 虽然还是有可能设置一些全局状态.

    // 全局合法数量计数器
private int counter = 0;
private int rows;
private int cols;
private int digitSum;
// 标记已经访问过的数组
private boolean[][] visited;
// 上下左右四个移动方向
private final int[][] moves = {{0, 1}, {0, -1}, {1, 0}, {-1, 0}};

public int wardrobeFinishing(int rows, int cols, int digitSum) {
    this.rows = rows;
    this.cols = cols;
    this.digitSum = digitSum;
    // 标记是否已经访问的数组
    visited = new boolean[rows][cols];
    dfs(0, 0);
    return counter;
}

private void dfs(int row, int col) {
    // 递归终止条件: 如果越界或者已经访问过的格子就退出
    if (row < 0 || row >= rows || col < 0 || col >= cols || visited[row][col]) {
        return;
    }
    // 标记已经访问的位置
    visited[row][col] = true;

    // 不符合标准的格子直接退出
    if (getSum(row, col) > digitSum) {
        return;
    }

    // 合格计数器++
    counter++;
    // 朝四个方向DFS
    for (int[] move : moves) {
        dfs(row + move[0], col + move[1]);
    }
}

// 求一个格子的各位之和
private int getSum(int i, int j) {
    int res = 0;
    int a = i;
    while (a != 0) {
        res = res + a % 10;
        a = a / 10;
    }
    a = j;
    while (a != 0) {
        res = res + a % 10;
        a = a / 10;
    }
    return res;
}

    # 深度优先遍历

    接下来使用深搜三部曲来分析:

    确认递归函数, 参数

    首先 dfs 函数一定要存一个图, 用来遍历的, 还要存一个目前遍历的节点, 定义为 x.

    确认终止条件

    什么时候就找到一条路径了? 当目前遍历的节点为最后一个节点的时候, 就找到了一条从出发点到终止点的路径.

    当前遍历的节点定义为 x, 最后一点节点就是 graph.size() - 1​ (因为题目描述是找出所有从节点 0 到节点 n - 1 的路径并输出). 所以当 x 等于 graph.size() - 1​ 的时候就找到一条有效路径.

    处理目前搜索节点出发的路径

    直接遍历当前结点的所有邻接结点, 将当前选择的结点加入路径中, 并选择这个结点进行 DFS 递归, 递归后再回溯这个选择, 进而继续选择下一个结点.

    本题整体代码如下:

    private List<List<Integer>> resList = new ArrayList<>();
private LinkedList<Integer> path = new LinkedList<>();

public List<List<Integer>> allPathsSourceTarget(int[][] graph) {
    // 注意, 0号节点要加入 cnt 数组中
    path.add(0);
    dfs(graph, 0);
    return resList;
}

public void dfs(int[][] graph, int x) {
    // 终止条件: 如果当前节点是n - 1, 那么就保存这条路径
    if (x == graph.length - 1) {
        resList.add(new ArrayList<>(path));
        return;
    }
  
    // 遍历当前节点的所有邻接节点
    for (int index = 0; index < graph[x].length; index++) {
        int nextNode = graph[x][index];
        // 当前结点加入路径中
        path.add(nextNode);
        // 递归当前选择的结点
        dfs(graph, nextNode);
        // 回溯
        path.removeLast();
    }
}

    # 广度优先搜索

    由于是求最短的路径, 所以可以考虑使用广度优先搜索.

    以示例 1 为例, 从这个图中可以看出 hit 到 cog 的路线, 不止一条, 有三条, 一条是最短的长度为 5, 两条长度为 6.

    本题只需要求出最短路径的长度就可以了, 不用找出路径.

    所以这道题要解决两个问题:

    • 图中的线是如何连在一起的
    • 起点和终点的最短路径长度

    首先题目中并没有给出点与点之间的连线, 而是要自己去连, 条件是字符只能差一个, 所以判断点与点之间的关系, 要自己判断是不是差一个字符, 如果差一个字符, 那就是有链接.

    然后就是求起点和终点的最短路径长度, 这里​无向图求最短路, 广搜最为合适​ , 广搜只要搜到了终点, 那么一定是最短的路径. 因为广搜就是以起点中心向四周扩散的搜索. 本题如果用深搜, 会比较麻烦, 要在到达终点的不同路径中选则一条最短路. 而广搜只要达到终点, 一定是最短路.

    另外需要有一个注意点:

    • 本题是一个无向图, 需要用标记位, 标记着节点是否走过, 否则就会死循环!
    • 本题给出集合是数组型的, 可以转成 set 结构, 查找更快一些.

    代码如下:

    public int ladderLength(String beginWord, String endWord, List<String> wordList) {
    if (!wordList.contains(endWord)) {
        return 0;
    }
    // 已经访问过的数组
    Set<String> visited = new HashSet<>();
    // BFS队列
    Queue<String> queue = new LinkedList<>();
    queue.offer(beginWord);
    visited.add(beginWord);
  
    // BFS的步数, 也就是最短的路径长度
    int step = 0;
  
    while (!queue.isEmpty()) {
        // 当前层的结点个数
        int size = queue.size();
        step++;
        for (int i = 0; i < size; i++) {
            // 弹出处理一个单词
            String curWord = queue.poll();
          
            // 遍历单词序列, 一个一个看能否转换过去 能则加入到队列中进行下一轮
            for (String word : wordList) {
                // 已经遍历的不再重复遍历
                if (visited.contains(word)) {
                    continue;
                }
                // 不能转换的直接跳过
                if (!canConvert(curWord, word)) {
                    continue;
                }
                // 找到目标单词
                if (word.equals(endWord)) {
                    return step + 1;
                }
                // 保存访问过的单词. 同时把单词放进队列. 用于下一层的访问
                visited.add(word);
                queue.offer(word);
            }
        }
    }
    return 0;
}

public boolean canConvert(String s1, String s2) {
    if (s1.length() != s2.length()) {
        return false;
    }
  
    int count = 0;
    for (int i = 0; i < s1.length(); ++i) {
        if (s1.charAt(i) != s2.charAt(i)) {
            count++;
            if (count > 1) {
                return false;
            }
        }
    }
    return count == 1;
}

    // 已经访问过则继续
    if (visited[roomKey]) {

    }

    // 标记为可以开锁访问
    visited[roomKey] = true;


    // 深度搜索剩下的房间
    for (int k : rooms.get(roomKey)) {
        dfs(rooms, k);
    }
}

    for (boolean key : visited) {
        if (!key) return false;
    }
  
    return true;
}

    # 广度优先遍历法

    广度优先遍历中, 首先需要​找到所有腐烂的橘子, 这些橘子同时作为第一层​ , 同时开始往上下左右扩散, 也就是进行广度优先遍历, 遍历一轮则表示经过了一分钟. 需要注意两个边界 case:

    • 如果一来就没有新鲜的橘子, 那么直接返回 0.
    • 如果最后还剩下无法腐烂的新鲜橘子, 那么最后返回 -1.

    因此, 整体还需要维护一下当前存在的新鲜橘子的数量, 最后腐败完之后, 再看看是否还有新鲜的橘子.

    代码如下:

    private int[][] moves = {{-1, 0}, {1, 0}, {0, -1}, {0, 1}};

public int orangesRotting(int[][] grid) {
    if (grid <mark> null || grid.length </mark> 0) {
        return -1;
    }

    int rows = grid.length;
    int cols = grid[0].length;
    Queue<int[]> queue = new LinkedList<>();
    int freshOranges = 0;

    // 初始化队列和统计新鲜橘子数量
    for (int i = 0; i < rows; i++) {
        for (int j = 0; j < cols; j++) {
            // 注意: 腐烂的橘子全部加入队列中, 也就是假设这一层已经有很多个结点了
            if (grid[i][j] <mark> 2) {
                queue.offer(new int[]{i, j});
            } else if (grid[i][j] </mark> 1) {
                freshOranges++;
            }
        }
    }

    // 没有新鲜的橘子
    if (freshOranges == 0) {
        return 0;
    }

    int minute = 0;

    // 开始BFS
    while (!queue.isEmpty()) {
        // 记录当前层的结点数
        int size = queue.size();

        for (int i = 0; i < size; i++) {
            int[] point = queue.poll();

            // 往四个方向走并将合法的橘子加入到队列中
            for (int[] direction : moves) {
                int newRow = point[0] + direction[0];
                int newCol = point[1] + direction[1];

                // 检查新的位置是否在矩阵边界内且是新鲜橘子
                if (newRow >= 0 && newRow < rows && newCol >= 0 && newCol < cols && grid[newRow][newCol] == 1) {
                    grid[newRow][newCol] = 2;  // 将新鲜橘子变为腐烂
                    freshOranges--;  // 新鲜橘子的数量减少
                    queue.offer(new int[]{newRow, newCol});
                }
            }
        }

        // 则增加分钟数
        minute++;
    }

    // 如果还有新鲜橘子则返回-1 否则返回轮数-1(因为第一列广度遍历时多了一轮)
    return freshOranges == 0 ? minute - 1 : -1;
}

    # 深度优先搜索法

    定义暴搜函数为 boolean dfs(int i, int j)​，代表当前处理到 s1 的第 i 个字符，s2 的第 j 个字符，能否凑成 s3 的前 i + j 个字符。

    将输入的三个字符串转换为字符数组，方便访问。获取 s1​、s2​ 和 s3​ 的长度。如果 s1​ 和 s2​ 的长度之和不等于 s3​ 的长度，直接返回 false​。否则，调用深度优先搜索（DFS）方法从索引 (0, 0)​ 开始检查。

    深度优先搜索方法 dfs：

    • 如果当前索引之和 i + j​ 达到 s3​ 的长度，表示成功匹配，返回 true​。
    • 检查 s1​ 的当前字符是否与 s3​ 的当前索引字符匹配，如果匹配则递归检查下一个字符，将 i​ 增加1。
    • 检查 s2​ 的当前字符是否与 s3​ 的当前索引字符匹配，如果匹配则递归检查下一个字符，将 j​ 增加1。
    • 返回是否通过匹配形成 s3​ 的结果。

    该代码通过递归方式检查 s1​ 和 s2​ 是否可以按照字符交错方式形成 s3​。

    private char[] cs1, cs2, cs3;
private int len1, len2, totalLen;

public boolean isInterleave(String s1, String s2, String s3) {
    cs1 = s1.toCharArray(); // 将s1转换为字符数组
    cs2 = s2.toCharArray(); // 将s2转换为字符数组
    cs3 = s3.toCharArray(); // 将s3转换为字符数组
    len1 = s1.length();     // s1的长度
    len2 = s2.length();     // s2的长度
    totalLen = s3.length(); // s3的长度

    // 如果s1和s2的长度之和不等于s3的长度, 则无法通过交错匹配形成s3
    if (len1 + len2 != totalLen) {
        return false;
    }

    // 使用DFS进行递归检查
    return dfs(0, 0);
}

// 深度优先搜索（DFS）方法, 递归检查索引i和j处的交错字符匹配情况
boolean dfs(int i, int j) {
    // 如果到达s3的末尾, 表示成功匹配
    if (i + j == totalLen) {
        return true;
    }

    boolean ans = false;

    // 检查s1的当前字符是否与s3匹配, 如果匹配则递归检查下一个字符
    if (i < len1 && cs1[i] == cs3[i + j]) {
        ans |= dfs(i + 1, j); // i增加1, j不变
    }

    // 检查s2的当前字符是否与s3匹配, 如果匹配则递归检查下一个字符
    if (j < len2 && cs2[j] == cs3[i + j]) {
        ans |= dfs(i, j + 1); // j增加1, i不变
    }

    // 返回匹配结果
    return ans;
}

// 递归检查索引i和j处的交错字符匹配情况

    if (cache[i][j] != 0) {
        return cache[i][j] <mark> 1;

    if (i + j </mark> totalLen) {

    if (j < len2 && cs2[j] == cs3[i + j]) {
        ans |= dfs(i, j + 1); // j增加1, i不变

    // 将结果存入缓存 (-1:不可行, 1:可行)
    cache[i][j] = ans ? 1 : -1;

    // 返回匹配结果
    return ans;
}

    # 回溯+剪枝法

    遍历所有点为起点的情况, 取最大值即可. 由于 A-B-C-D 路径中同时也把 B, C, D 三点的结果计算出来了, 所以存在重复计算, 这里使用一个 dp 数组用于记忆化搜索, 遇到重复情况时剪枝.

    private final int[][] moves = {{0, 1}, {0, -1}, {1, 0}, {-1, 0}};
private int[][] memo;

public int longestIncreasingPath(int[][] matrix) {
    if (matrix <mark> null || matrix.length </mark> 0 || matrix[0] <mark> null || matrix[0].length </mark> 0) {
        return 0;
    }

    // 缓存每个位置最长的递增子序列的大小
    memo = new int[matrix.length][matrix[0].length];
    int maxLen = 0;

    // 把每个位置都搜索一遍
    for (int i = 0; i < matrix.length; i++) {
        for (int j = 0; j < matrix[0].length; j++) {
            maxLen = Math.max(maxLen, dfs(matrix, Integer.MIN_VALUE, i, j));
        }
    }

    return maxLen;
}

private int dfs(int[][] matrix, int curNum, int i, int j) {
    int row = matrix.length;
    int col = matrix[0].length;
    // 校验是不是数组越界
    if (i < 0 || i >= row || j < 0 || j >= col || matrix[i][j] <= curNum) {
        return 0;
    }

    // 记忆化搜索(剪枝): 如果当前位置已经搜索过, 直接返回结果值
    if (memo[i][j] != 0) {
        return memo[i][j];
    }

    // 四个方向去搜索, 找到最大的值
    int max = 0;
    for (int[] move : moves) {
        max = Math.max(max, dfs(matrix, matrix[i][j], i + move[0], j + move[1]));
    }
    // (缓存)记录当前位置的最长递增子序列长度
    memo[i][j] = max + 1;
    return max + 1;
}

    # 广度优先搜索

    注意题目中每座岛屿只能由水平方向或竖直方向上相邻的陆地连接形成. 也就是说斜角度链接是不算了, 例如示例二, 是三个岛屿, 如图:

    这道题题目是 DFS, BFS, 并查集, 基础题目.

    本题思路, 是用遇到一个没有遍历过的节点陆地, 计数器就加一, 然后把该节点陆地所能遍历到的陆地都标记上. 在遇到标记过的陆地节点和海洋节点的时候直接跳过. 这样计数器就是最终岛屿的数量.

    那么如果把节点陆地所能遍历到的陆地都标记上, 就可以使用 DFS, BFS 或者并查集. 不少同学用广搜做这道题目的时候超时了. 这里有一个广搜中很重要的细节: 根本原因是只要加入队列就代表走过, 就需要标记, 而不是从队列拿出来的时候再去标记走过.

    因为如果从队列拿出节点, 再去标记这个节点走过, 就会发生下图所示的结果, 会导致很多节点重复加入队列.

    队列中的节点就表示已经走过的节点. 所以只要加入队列, 立即标记该节点走过. 如果允许修改数组的话, 可以直接把访问过的元素修改为 0.

    本题完整广搜代码:

    for (int i = 0; i < grid.length; i++) {
        for (int j = 0; j < grid[0].length; j++) {
            // 如果当前格子是陆地且未被访问过则表示找到一个岛屿
            if (grid[i][j] == '1') {
                count++;
                bfs(grid, i, j);
            }
        }
    }

    // 还原数组
    for (int i = 0; i < grid.length; i++) {
        for (int j = 0; j < grid[0].length; j++) {
            if (grid[i][j] == '2') {
                grid[i][j] = '1';
            }
        }
    }

    return count;
}

// 将这片岛屿上的所有陆地都访问到
public void bfs(char[][] grid, int x, int y) {
    // 当前格子加入队列
    Queue<int[]> queue = new LinkedList<>();
    queue.offer(new int[]{x, y});
    // 标记已经访问过
    grid[x][y] = '2';

    while (!queue.isEmpty()) {
        // 弹出元素进行BFS
        int[] cur = queue.poll();
        // 往四个方向遍历
        for (int[] move : moves) {
            int nextx = cur[0] + move[0];
            int nexty = cur[1] + move[1];

            // 当前格子未访问且是一个陆地, 将其加入到队列并标记为已访问
            if (grid[nextx][nexty] == '1') {
                queue.offer(new int[]{nextx, nexty});
                grid[nextx][nexty] = '2';
            }
        }

    # 深度优先搜索

    下面的代码使用的是深度优先搜索 DFS 的做法. 为了统计岛屿数量同时不重复记录, 每当搜索到一个岛后, 就将这个岛 "淹没" -- 将这个岛所占的地方从 "1" 改为 "2", 这样就不用担心后续会重复记录这个岛屿了. 而 DFS 的过程就体现在淹没这一步中. 最后将更改后的 2 改完原来的 1 以还原最开始的数组.

        dfs(grid, i + move[0], j + move[1]);

    # 深度优先搜索

    本题的一个暴力想法, 应该是遍历地图尝试将每一个 0 改成 1, 然后去搜索地图中的最大的岛屿面积. 这样计算地图的最大面积: 遍历地图 + 深搜岛屿, 时间复杂度为 n * n.

    其实每次深搜遍历计算最大岛屿面积, 都做了很多重复的工作. 只要用一次深搜把每个岛屿的面积记录下来就好.

    第一步: 一次遍历地图, 得出各个岛屿的面积, 并做编号记录. 可以使用 map 记录, key 为岛屿编号, value 为岛屿面积.

    第二步: 在遍历地图, 遍历 0 的方格(因为要将 0 变成 1), 并统计该 1(由 0 变成的 1)周边岛屿面积, 将其相邻面积相加在一起, 遍历所有 0 之后, 就可以得出 选一个 0 变成 1 之后的最大面积.

    拿如下地图的岛屿情况来举例: (1 为陆地)

    第一步, 则遍历题目, 并将岛屿到编号和面积上的统计, 过程如图所示:

    这个过程时间复杂度 n * n.

    第二步过程如图所示:

    也就是遍历每一个 0 的方格, 并统计其相邻岛屿面积, 最后取一个最大值. 这个过程的时间复杂度也为 n * n. 所以整个解法的时间复杂度, 为 n * n + n * n 也就是 n^2.

    当然这里还有一个优化的点, 就是可以不用 visited 数组, 因为有 mark 来标记, 所以遍历过的 grid[i][j] 是不等于 1 的.

    代码如下:

    class Island {
    int islandKey;
    int islandSize;

    public Island(int islandKey, int islandSize) {
        this.islandKey = islandKey;
        this.islandSize = islandSize;
    }
}

// 用于四个方向的移动(上, 右, 下, 左)
private final int[][] moves = {{-1, 0}, {0, 1}, {1, 0}, {0, -1}};
// 用于格式化坐标的字符串
private static final String X_Y_FORMAT = "%d-%d";
// 当前岛屿的面积
private int currentIslandArea;

public int largestIsland(int[][] grid) {
    Map<String, Island> grid2Island = new HashMap<>();
    int islandId = 0, maxArea = 0;

    // 第一次遍历: 识别所有岛屿并计算其面积
    for (int i = 0; i < grid.length; i++) {
        for (int j = 0; j < grid[0].length; j++) {
            if (grid[i][j] == 1) {
                // 存储当前岛屿的所有格子的坐标
                List<String> gridKeys = new ArrayList<>();
                currentIslandArea = 0;
                // 深度优先搜索, 填充 grid2Island 和 gridKeys, 更新 currentIslandArea
                dfs(grid, i, j, gridKeys);

                // 将岛屿的所有坐标对应到同一个岛屿对象
                for (String key : gridKeys) {
                    grid2Island.put(key, new Island(islandId, currentIslandArea));
                }
                // 下面是一个新的岛
                islandId++;
                // 更新初始岛屿的最大面积
                maxArea = Math.max(maxArea, currentIslandArea);
            }
        }
    }

    // 第二次遍历: 试图通过将一个 0 变成 1 来形成最大的岛屿
    for (int i = 0; i < grid.length; i++) {
        for (int j = 0; j < grid[0].length; j++) {
            if (grid[i][j] == 0) {
                // 存储已经检查过的岛屿
                Set<Integer> seen = new HashSet<>();
                int newArea = 1; // 从将一个 0 变成 1 开始

                // 检查四个方向
                for (int[] move : moves) {
                    int ni = i + move[0];
                    int nj = j + move[1];
                    if (ni >= 0 && ni < grid.length && nj >= 0 && nj < grid[0].length && grid[ni][nj] == 2) {
                        // 获取相邻位置的岛屿信息
                        Island island = grid2Island.get(String.format(X_Y_FORMAT, ni, nj));
                        // 如果此岛屿尚未检查过, 增加其面积
                        if (island != null && seen.add(island.islandKey)) {
                            newArea += island.islandSize;
                        }
                    }
                }

                // 更新最大面积
                maxArea = Math.max(maxArea, newArea);
            }
        }
    }

    return maxArea;
}

// 深度优先搜索用于标记和计算岛屿面积
private void dfs(int[][] grid, int x, int y, List<String> gridKeys) {
    if (x < 0 || y < 0 || x >= grid.length || y >= grid[0].length || grid[x][y] != 1) {
        return;
    }

    grid[x][y] = 2;  // 将当前格子标记为已访问
    gridKeys.add(String.format(X_Y_FORMAT, x, y));
    currentIslandArea++;

    // 继续向四个方向搜索
    for (int[] move : moves) {
        int nx = x + move[0];
        int ny = y + move[1];
        dfs(grid, nx, ny, gridKeys);
    }

    # 深度优先搜索

    题目指定 words 中出现的单词长度不会超过 10，可以考虑使用「回溯算法」。起始先将所有 words 出现的单词放到 Set 结构中，然后以 board 中的每个点作为起点进行暴搜。由于题目规定在一个单词中每个格子只能被使用一次，因此还需要一个 vis 数组来记录访问过的位置：

    • 如果当前暴搜到的字符串长度超过 10，直接剪枝；
    • 如果当前搜索到的字符串在 Set 中，则添加到答案；同时了防止下一次再搜索到该字符串，需要将该字符串从 Set 中移除。

private List<String> resList = new ArrayList<>();

private int n, m;
private boolean[][] visited = new boolean[15][15];

public List<String> findWords(char[][] boards, String[] words) {
    board = boards;
    m = board.length;
    n = board[0].length;
  
    set.addAll(Arrays.asList(words));
  
    StringBuilder builder = new StringBuilder();
    for (int i = 0; i < m; i++) {
        for (int j = 0; j < n; j++) {
            visited[i][j] = true;
            builder.append(board[i][j]);
            dfs(i, j, builder);
            visited[i][j] = false;
            builder.deleteCharAt(builder.length() - 1);

    }
    return resList;

private void dfs(int i, int j, StringBuilder sb) {
    // 剪枝操作
    if (sb.length() > 10) {

    if (set.contains(sb.toString())) {
        resList.add(sb.toString());
        set.remove(sb.toString());

        int dx = i + d[0], dy = j + d[1];

        // 标记访问
        visited[dx][dy] = true;
        sb.append(board[dx][dy]);
    
        dfs(dx, dy, sb);
    
        // 回溯, 撤销访问
        visited[dx][dy] = false;
        sb.deleteCharAt(sb.length() - 1);
    }
}

    # 广度优先遍历

    这里寻找的是变化的最少变化次数，其实就是求变化的最短路径，因此自然就可以想到用广度优先遍历方式求解。

    代码总体思路：

    • 利用队列保存有效的字符串。
    • 只要队列不空，就持续循环：记录当前队列的长度，对队列中该长度的字符串逐一遍历：如果搜索到 end，直接返回当前的步数 step。否则，对当前字符串中的每个字符，都转变成 ACGT 四个字符，看新形成的字符串是否遇到过。如果没遇到过，就放入队列之中，使之可以进入下一轮搜索。
    • 完成一轮搜索，步数 + 1。

    其中判断字符串是否遇到过，有两种方法：

    • 使用 set 保存所有已经遇到过的字符串。
    • 直接从 bank 中删除已经遇到过的字符串。

    代码如下：

    // 四个基因值(其实就是类似矩阵中搜索的上下左右四个方向)
private final char[] genes = new char[]{'A', 'C', 'G', 'T'};

public int minMutation(String start, String end, String[] bank) {
    // 将bank转换为Set
    Set<String> bankSet = new HashSet<>(Arrays.asList(bank));

    Set<String> visited = new HashSet<>();
    Queue<String> queue = new LinkedList<>();

    // 将起始基因加入队列
    queue.offer(start);
    visited.add(start);

    int step = 0;
    while (!queue.isEmpty()) {
        int size = queue.size();
        for (int i = 0; i < size; i++) {
            String cur = queue.poll();
            // 找到目标基因
            if (cur.equals(end)) {
                return step;
            }

            for (char gene : genes) {
                char[] chars = cur.toCharArray();
                // 每个位置都尝试替换为四个基因
                for (int j = 0; j < chars.length; j++) {
                    // 保存旧位置的字符

    List<List<Integer>> adjacency = new ArrayList<>();
    // 初始化队列, 用于存储入度为0的课程

        adjacency.get(cp[1]).add(cp[0]);

        }
    }

    return numCourses </mark> 0;
}

        // 如果当前城市与邻接城市相连且邻接城市未访问过
        if (isConnected[i][j] == 1 && !visited[j]) {
            // 当前城市标记为已访问过
            visited[j] = true;
            dfs(isConnected, cities, j);
        }
    }
}
