    # 元素查找题目

    # 搜索二维矩阵(240-中等)🌸🔑💯
    # 题目描述

    给定一个二维数组, 具有如下特征:

    • 每行中的整数从左到右按非严格递增顺序排列.
    • 每行的第一个整数大于前一行的最后一个整数.

    给定一个数, 判断这个数是否在该二维数组中.

    输入: matrix = [[1,3,5,7],[10,11,16,20],[23,30,34,60]], target = 30
输出: true
    1
    2

    https://leetcode.cn/problems/search-a-2d-matrix/description/

    # 解题思路

    要求时间复杂度 O(M + N) , 空间复杂度 O(1) . 其中 M 为行数, N 为 列数.

    由于矩阵是有排序特点的, 所以不用按照普通的遍历方法来查找, 而是要学会利用这个特点.

    从右上角开始往下查找, 找到第一个大于目标值的行, 说明目标值就可能在这行, 然后再从右往左逐一查找即可.

    ​​

    代码如下:

    public boolean searchMatrix(int[][] matrix, int target) {
    if (matrix <mark> null || matrix.length </mark> 0 || matrix[0].length == 0) {
        return false;
    }
    // 行数
    int cow = matrix.length;
    // 列数
    int col = matrix[0].length;
  
    // 从右上角开始往下找
    for (int i = 0; i < cow; i++) {
        if (matrix[i][col - 1] < target) {
            continue;
        }
        // 找到一个比目标值大的行则开始从右往左找
        for (int j = col - 1; j >= 0; j--) {
            if (matrix[i][j] == target) {
                return true;
            }
        }
    }

    return false;
}

    # 数组中大于一半的元素(169-简单)🌸🔑💯
    # 题目描述

    给定一个大小为 n 的数组, 找到其中的多数元素. 多数元素是指在数组中出现次数大于 【n/2】 的元素. 可以假设数组是非空的, 并且给定的数组总是存在多数元素.

    输入: [3,2,3] 输出: 3
输入: [2,2,1,1,1,2,2] 输出: 2

    https://leetcode-cn.com/problems/majority-element/ (opens new window)

    # 排序法

    排序: 排序之后出现超过数组长度一半的元素一定在中间位置.

    当然这样写就不能满足题意了, 因为时间复杂度超标了.

    # 遍历计数法

    摩尔投票法: 时间复杂度为 O(N). 使用 cnt 来统计一个元素出现的次数, 当遍历到的元素和统计元素相等时, 令 cnt++ , 否则令 cnt-- . 如果前面遍历了 i 个元素, 且 cnt == 0, 说明前 i 个元素没有 majority, 或者有 majority, 但是出现的次数少于 i / 2, 因为如果多于 i / 2 的话 cnt 就一定不会为 0. 这样就可以更换新的元素来统计了. 此时剩下的 n - i 个元素中, majority 的数目依然多于 (n - i) / 2, 因此继续查找就能找出 majority.

    动图如下:

    代码如下:

    public int majorityElement(int[] nums) {
    int majority = 0, cnt = 0;

    // 遍历数组
    for (int num : nums) {
        // 票数为0时则直接更新majority为当前的值
        if (cnt == 0) {
            majority = num;
        }
        // 如果相同则投票++,否则投票--
        if (majority == num) {
            cnt++;
        } else {
            cnt--;
        }
    }
    return majority;
}

    分析二分查找的一个技巧是: 不要出现 else, 而是把所有情况用 else if 写清楚, 这样可以清楚地展现所有细节.

    # 基本二分查找(寻找一个数)

    这个场景是最简单的, 即在一个不存在重复元素的升序数组中搜索一个数, 如果存在, 返回其索引, 否则返回 -1.

    int binarySearch(int[] nums, int target) {
    int left = 0, right = nums.length - 1;

    // 搜索区间为左右闭区间[left, right]
    while(left <= right) {
        // 计算中值
        int mid = left + (right - left) / 2;
        // 如果中值小于目标值, 说明目标值一定在mid位置的右边 因此更新左边界为mid+1
        if (nums[mid] < target) {
            left = mid + 1;
            // 如果中值大于目标值, 说明目标值一定在mid位置的左边 因此更新右边界为mid-1
        } else if (nums[mid] > target) {
            right = mid - 1;
            // 找到目标值直接返回
        } else if(nums[mid] == target) {
            return mid;
        }
    }
  
    // 退出while循环的条件是left > right了, 肯定不存在了, 直接返回-1
    return -1;
}

    1, 为什么 while 循环的条件中是 <=, 而不是 < ?

    答: 因为初始化 right​ 的赋值是 nums.length - 1​, 即最后一个元素的索引, 而不是 nums.length​.

    这二者可能出现在不同功能的二分查找中, 区别是: 前者相当于两端都闭区间 [left, right]​, 后者相当于左闭右开区间 [left, right)​, 因为索引大小为 nums.length​ 是越界的.

    这个算法中使用的是前者 [left, right]​ 两端都闭的区间. 这个区间其实就是每次进行搜索的区间.

    什么时候应该停止搜索呢? 当然, 找到了目标值的时候可以终止:

    if(nums[mid] == target)
    return mid;
    1
    2

    但如果没找到, 就需要 while 循环终止, 然后返回 -1. 那 while 循环什么时候应该终止? 搜索区间为空的时候应该终止, 意味着没得找了, 就等于没找到.

    ​while(left <= right)​ 的终止条件是 left == right + 1​, 写成区间的形式就是 [right + 1, right]​, 或者带个具体的数字进去 [3, 2]​, 可见这时候区间为空, 因为没有数字既大于等于 3 又小于等于 2 的吧. 所以这时候 while 循环终止是正确的, 直接返回 -1 即可.

    ​while(left < right)​ 的终止条件是 left == right​, 写成区间的形式就是 [left, right]​, 或者带个具体的数字进去 [2, 2]​, 这时候区间非空, 还有一个数 2, 但此时 while 循环终止了. 也就是说这区间 [2, 2]​ 被漏掉了, 索引 2 没有被搜索, 如果这时候直接返回 -1 就是错误的.

    当然, 如果非要用 while(left < right)​ 也可以, 既然已经知道了出错的原因, 就打个补丁好了:

    //...
while(left < right) {
    // ...
}
return nums[left] == target ? left : -1;

    2, 为什么 ​left = mid + 1​​ , ​​right = mid - 1​​ ? 而看有的代码是 ​right = mid​​ 或者 ​left = mid​​ , 没有这些加加减减, 到底怎么回事?

    答: 这也是二分查找的一个难点, 不过只要能理解前面的内容, 就能够很容易判断.

    刚才明确了「搜索区间」这个概念, 而且本算法的搜索区间是两端都闭的, 即 [left, right]​. 那么当发现索引 mid​ 不是要找的 target​ 时, 下一步应该去搜索哪里呢?

    当然是去搜索 [left, mid - 1]​ 或者 [mid + 1, right]​ 对不对? 因为 ​mid​​ 已经搜索过, 应该从搜索区间中去除.

    3, 此算法有什么缺陷?

    比如说有序数组 nums = [1,2,2,2,3]​, target​ 为 2, 此算法返回的索引是 2, 没错. 但是如果想得到 target​ 的左侧边界, 即索引 1, 或者想得到 target​ 的右侧边界, 即索引 3, 这样的话此算法是无法处理的.

    下面就来讨论这两种二分查找的算法.

    # 寻找左侧边界的二分查找

    即在一个存在重复元素的非递减数组中寻找一个数字最开始出现的位置.

    以下是最常见的代码形式, 其中的标记是需要注意的细节:

    int leftBound(int[] nums, int target) {
    if (nums.length == 0) return -1;
    int left = 0;
    int right = nums.length; // 注意

    while (left < right) { // 注意
        int mid = (left + right) / 2;
        if (nums[mid] == target) {
            right = mid;
        } else if (nums[mid] < target) {
            left = mid + 1;
        } else if (nums[mid] > target) {
            right = mid; // 注意
        }
    }
    return left;
}

    1, 为什么 while 中是 ​<​ ​ 而不是 ​<=​ ​?

    答: 用相同的方法分析, 因为 right = nums.length​ 而不是 nums.length - 1​. 因此每次循环的「搜索区间」是 [left, right)​ 左闭右开. while(left < right)​ 终止的条件是 left == right​, 此时搜索区间 [left, left)​ 为空, 所以可以正确终止.

    这里先说一个搜索左右边界和上面这个算法的一个区别: 前面普通算法的 ​right​​ 不是 ​nums.length - 1​​ 吗, 为啥这里非要写成 ​nums.length​​ 使得「搜索区间」变成左闭右开呢?

    因为对于搜索左右侧边界的二分查找, 这种写法比较普遍, 就拿这种写法举例了, 保证以后遇到这类代码可以理解. 你非要用两端都闭的写法反而更简单, 会在后面写相关的代码, 把三种二分搜索都用一种两端都闭的写法统一起来, 耐心往后看就行了.

    2, 为什么没有返回 -1 的操作? 如果 ​nums​​ 中不存在 ​target​​ 这个值, 怎么办?

    答: 因为要一步一步来, 先理解一下这个「左侧边界」有什么特殊含义:

    对于这个数组, 算法会返回 1. 这个 1 的含义可以这样解读: nums​ 中小于 2 的元素有 1 个.

    比如对于有序数组 nums = [2,3,5,7]​, target = 1​, 算法会返回 0, 含义是: nums​ 中小于 1 的元素有 0 个.

    再比如说 nums = [2,3,5,7], target = 8​, 算法会返回 4, 含义是: nums​ 中小于 8 的元素有 4 个.

    综上可以看出, 函数的返回值(即 left​ 变量的值)取值区间是闭区间 [0, nums.length]​, 所以简单添加两行代码就能在正确的时候 return -1:

    while (left < right) {
    //...
}
// target 比所有数都大
if (left <mark> nums.length) return -1;
// 类似之前算法的处理方式
return nums[left] </mark> target ? left : -1;

    3, 为什么 ​left = mid + 1​​ , ​​right = mid​​ ? 和之前的算法不一样?

    答: 这个很好解释, 因为「搜索区间」是 [left, right)​ 左闭右开, 所以当 nums[mid]​ 被检测之后, 下一步的搜索区间应该去掉 mid​ 分割成两个区间, 即 [left, mid)​ 或 [mid + 1, right)​.

    if (nums[mid] == target)

    至此, 整个算法就写完了, 完整代码如下:

    // 左闭右闭区间寻找左边界
int leftBound(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    // 左闭右闭的搜索区间
    while (left <= right) {
        int mid = left + (right - left) / 2;
        // 如果中值小于目标值, 说明目标值一定在mid位置的右边 因此更新左边界为mid+1
        if (nums[mid] < target) {
            left = mid + 1;
            // 如果中值大于目标值, 说明目标值一定在mid位置的左边 因此更新右边界为mid-1
        } else if (nums[mid] > target) {
            right = mid - 1;
            // 与目标值相同. 注意别返回, 需要更新搜索区间的right来逐渐找出左侧边界
        } else if (nums[mid] == target) {
            right = mid - 1;
        }
    }
    // while循环退出条件是left > right, 此时left可能越界
    // 最后要检查 left 越界的情况
    if (left >= nums.length || nums[left] != target) {
        return -1;
    }

    return left;
}

    # 寻找右侧边界的二分查找

    即在一个存在重复元素的非递减数组中寻找一个数字最后出现的位置.

    类似寻找左侧边界的算法, 这里也会提供两种写法, 还是先写常见的左闭右开的写法, 只有两处和搜索左侧边界不同, 已标注:

    int rightBound(int[] nums, int target) {
    if (nums.length == 0) return -1;
    int left = 0, right = nums.length;

    while (left < right) {
        int mid = (left + right) / 2;
        if (nums[mid] == target) {
            left = mid + 1; // 注意
        } else if (nums[mid] < target) {
            left = mid + 1;
        } else if (nums[mid] > target) {
            right = mid;
        }
    }
    return left - 1; // 注意
}

    # 二分法(左闭右闭)

    这道题目的前提是数组为有序数组, 同时题目还强调数组中无重复元素, 因为一旦有重复元素, 使用二分查找法返回的元素下标可能不是唯一的, 这些都是使用基本二分法的前提条件, 当看到题目描述满足如上条件的时候, 可要想一想是不是可以用二分法了.

    二分查找涉及的很多的边界条件, 逻辑比较简单, 但就是写不好. 例如到底是 while(left < right)​ 还是 while(left <= right)​, 到底是 right = middle​, 还是要 right = middle - 1​ 呢? 二分法经常写乱, 主要是因为对区间的定义没有想清楚, 区间的定义就是不变量. 要在二分查找的过程中, 保持不变量, 就是在 while 寻找中每一次边界的处理都要坚持根据区间的定义来操作, 这就是循环不变量规则.

    下面定义 target 是在一个在左闭右闭的搜索区间里, 也就是 [left, right] (这个非常重要) .

    区间的定义这就决定了二分法的代码应该如何写, 因为定义 target 在 [left, right] 区间, 所以有如下两点:

    • ​while (left <= right)​ 要使用 <=​, 因为 left == right​ 是有意义的, 所以使用 <=​.
    • ​if (nums[middle] > target)​ right 要赋值为 middle - 1, 因为当前这个 nums[middle] 一定不是 target, 那么接下来要查找的左区间结束下标位置就是 middle - 1.

    例如在数组: 1,2,3,4,7,9,10,11 中查找元素 3, 如图所示:

    ​​

    不好定义区间的话, 画个图就非常明确了. 代码如下:

    public int search(int[] nums, int target) {
    int left = 0, right = nums.length - 1;

    // 搜索区间为左右闭区间[left, right]
    while (left <= right) {
        // 计算中值
        int mid = left + (right - left) / 2;

        // 找到目标值直接返回
        if (nums[mid] == target) {
            return mid;
            // 如果中值小于目标值, 说明目标值一定在mid位置的右边 因此更新左边界为mid+1
        } else if (nums[mid] < target) {
            left = mid + 1;
            // 如果中值大于目标值, 说明目标值一定在mid位置的左边 因此更新右边界为mid-1
        } else if (nums[mid] > target) {
            right = mid - 1;
        }
    }

    // 退出while循环的条件是left > right了, 肯定不存在了, 直接返回-1
    return -1;
}

    # 二分法

    本题就是基础二分法的变化版本. 区别在于二分查找找不到元素返回 -1, 这里找不到元素返回 left 索引即可. 因为循环式 while (left <= right), 退出条件是 left > right, 因此退出循环时, left 就是需要插入的位置.

    代码如下：

    public int searchInsert(int[] nums, int target) {
    if (nums <mark> null || nums.length </mark> 0) return -1;

    // nums = [1,3,5,6,7], target = 4
    int left = 0, right = nums.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        // 找到元素了直接返回索引
        if (nums[mid] == target) {
            return mid;
            // 如果中值小于目标值, 说明目标值一定在mid位置的右边 因此更新左边界为mid+1
        } else if (nums[mid] < target) {
            left = mid + 1;
            // 如果中值大于目标值, 说明目标值一定在mid位置的左边 因此更新右边界为mid-1
        } else if (nums[mid] > target) {
            right = mid - 1;
        }
    }
    // 退出while循环的条件是left > right了, 有可能存在溢出的问题, 所以判断一下
    if (left > nums.length - 1) return nums.length;

    // 只有最后这里与二分查找不同, 返回left即可
    return left;
}

    public int countTarget(int[] nums, int target) {
    // 二分查找左右边界
    int left = leftBound(nums, target);
    int right = rightBound(nums, target);
    if (left <mark> -1 || right </mark> -1) return 0;
    // 计算个数
    return right - left + 1;
}

private int binarySearch(int[] nums, double target) {
    int left = 0, right = nums.length - 1;
    // 二分寻找应该插入的位置
    while (left <= right) {
        int mid = (right + left) / 2;
        if (nums[mid] < target) {
            left = mid + 1;
        } else if (nums[mid] > target) {
            right = mid - 1;
        } else if (nums[mid] == target) {
            left = mid;
        }
    }
    return left;

    # 二分寻找左右边界法

    与题目 数字在排序数组中出现的次数 基本一模一样, 只是改一下返回的结果为左右边界即可.

    代码如下：

    public int[] searchRange(int[] nums, int target) {
    // 二分查找左右边界
    int left = leftBound(nums, target);
    int right = rightBound(nums, target);
    if (left <mark> -1 || right </mark> -1) {
        return new int[]{-1, -1};
    }
    // 返回左右边界
    return new int[]{left, right};
}

// 左闭右闭区间寻找左边界
int leftBound(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    // 左闭右闭的搜索区间
    while (left <= right) {
        int mid = left + (right - left) / 2;
        // 如果中值小于目标值, 说明目标值一定在mid位置的右边 因此更新左边界为mid+1
        if (nums[mid] < target) {
            left = mid + 1;
            // 如果中值大于目标值, 说明目标值一定在mid位置的左边 因此更新右边界为mid-1
        } else if (nums[mid] > target) {
            right = mid - 1;
            // 与目标值相同. 注意别返回, 需要更新搜索区间的right来逐渐找出左侧边界
        } else if (nums[mid] == target) {
            right = mid - 1;
        }
    }
    // while循环退出条件是left > right, 此时left可能越界
    // 最后要检查 left 越界的情况
    if (left >= nums.length || nums[left] != target) {
        return -1;
    }

    return left;
}

// 左闭右闭区间寻找右边界
int rightBound(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    // 左闭右闭的搜索区间
    while (left <= right) {
        int mid = left + (right - left) / 2;
        // 如果中值小于目标值, 说明目标值一定在mid位置的右边 因此更新左边界为mid+1
        if (nums[mid] < target) {
            left = mid + 1;
            // 如果中值大于目标值, 说明目标值一定在mid位置的左边 因此更新右边界为mid-1
        } else if (nums[mid] > target) {
            right = mid - 1;
            // 与目标值相同. 注意别返回, 需要更新搜索区间的left来逐渐找出右侧边界
        } else if (nums[mid] == target) {
            left = mid + 1;
        }
    }
    // 最后要检查 right 越界的情况
    if (right < 0 || nums[right] != target) {
        return -1;
    }
    return right;
}

    # 题解思路

    排序数组中的搜索问题, 首先想到 二分法 解决. 根据题意, 数组可以按照以下规则划分为两部分.

    • 左子数组: nums[i] = i;
    • 右子数组: nums[i] != i;

    缺失的数字等于 "右子数组的首位元素" 对应的索引; 因此考虑使用二分法查找 "右子数组的首位元素" .

    public int takeAttendance(int[] nums) {

    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] == mid) {
            left = mid + 1;
        } else {
            // 缺失的数字在mid的左边
            right = mid - 1;
        }
    }

    if (left >= nums.length) return nums.length;

    return left;
}

    # 旋转排序数组中的最小值(153-中等)🌸🔑💯
    # 题目描述

    把一个不存在重复元素的升序数组进行旋转, 找出旋转数组的最小值.

    输入: nums = [3,4,5,1,2]      输出: 1
输入: nums = [4,5,6,7,0,1,2]  输出: 0
    1
    2

    https://leetcode.cn/problems/find-minimum-in-rotated-sorted-array/description/ (opens new window)

    # 二分法

    关键在于确定对半分得到的两个数组哪一个是旋转数组, 哪一个是递增数组. 很容易知道递增数组的第一个元素一定小于等于最后一个元素.

    • 这里 while 里面取 left < right 即可, 因为 left == right 时, 就代表只有一个元素了, 此时就算找到了.
    • 取中值后, 对比 nums[left] 与 nums[right] 值, 如果 nums[left] < nums[right]​, 则说明右半部分是递增的, 最小值在左边. 此时把 right 指向 mid, 注意这里有可能 mid 本身就是最小值, 所以不能取 mid - 1. 否则左边是递增的, 由于 left 到 mid 是递增的, 所以 mid 一定不是最小值, 所以 left 指向 mid + 1.

    ​​

    代码如下:

    public int findMin(int[] nums) {
    if (nums.length == 0) {
        return 0;
    }

    int left = 0, right = nums.length - 1;
    // 5 6 7 8 9 1 2 3 4
    // 二分查找搜索
    while (left < right) {
        // 取中值
        int mid = left + (right - left) / 2;
        // 如果mid小于right的值, 说明右边是递增的
        if (nums[mid] < nums[right]) {
            // 最小值在左边, 此时更新right到mid的位置 注意这里不是mid-1 因为mid自己可能就是最小值
            right = mid;
        } else {	// 说明mid左边是递增的
            // 这里说明目标值一定在mid的右边, 更新左索引到mid+1
            left = mid + 1;
        }
    }

    // 指针相遇, 返回这个值
    return nums[left];
}

    public int minNumberInRotateArray2(int[] nums) {
    if (nums.length == 0) return 0;
    int left = 0, right = nums.length - 1;
    while (left < right) {
        int mid = left + (right - left) / 2;
        // 左右中值都一样
        if (nums[left] <mark> nums[mid] && nums[mid] </mark> nums[right]) {
            // 此时从左到右顺序查找
            return minNumber(nums, left, right);
            // 中间小于右边
        } else if (nums[mid] <= nums[right]) {
            right = mid;
            // 中间小于左边
        } else {
            left = mid + 1;
        }
    }
    return nums[left];
}

private int minNumber(int[] nums, int left, int right) {
    for (int i = left; i < right; i++) {
        if (nums[i] > nums[i + 1]) {
            return nums[i + 1];
        }
    }
    return nums[left];
}

    # 二分法

    此题与 旋转排序数组中的最小值 非常类似. 但是本题是找指定的值, 由于发生了旋转, 所以在判断左边还是右边有序之后, 还需要一些补充的逻辑.

    由于题目说数字了无重复, 举个例子: 1 2 3 4 5 6 7​. 可以大致分为两类:

    • 第一类 2 3 4 5 6 7 1​ 这种, 也就是 nums[start] <= nums[mid]​. 此例子中就是 2 <= 5. 这种情况下, 左半部分有序. 因此如果 nums[start] <= target < nums[mid]​, 则继续在左半部分找, 否则去右半部分找. 第二类 6 7 1 2 3 4 5​
    • 这种, 也就是 nums[start] > nums[mid]​. 此例子中就是 6 > 2. 这种情况下, 右半部分有序. 因此如果 nums[mid] < target <= nums[end]​, 则在右半部分找, 否则去左半部分找.

    代码如下:

    public int search(int[] nums, int target) {
    if (nums <mark> null || nums.length </mark> 0) {
        return -1;
    }
    int left = 0;
    int right = nums.length - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;
        // 找到目标值直接返回
        if (nums[mid] == target) {
            return mid;
        }

        // 说明右边部分有序
        if (nums[mid] < nums[right]) {
            // 说明目标值在右边部分, 更新左边界
            if (nums[mid] < target && target <= nums[right]) {
                left = mid + 1;
                // 否则更新右边界
            } else {
                right = mid - 1;
            }
            // 说明左边部分有序
        } else {
            // 说明目标值在左边部分, 更新右边界
            if (nums[mid] > target && target >= nums[left]) {
                right = mid - 1;
                // 否则更新左边界
            } else {
                left = mid + 1;
            }
        }
    }
    return -1;
}

    # 二分法

    注意本题只需要逐渐爬坡就行了, 取中值之后, 对比中值与相邻的元素, 往爬坡的方向二分即可. 需要注意的是, 这里 right 是 nums.length - 2, 因为要比较 mid 和 mid + 1, 如果 right 是 nums.length - 1 会越界.

    public int findPeakElement(int[] nums) {
    if (nums <mark> null || nums.length </mark> 0) return -1;
    // 注意这里right是nums.length - 2, 因为要比较mid和mid+1, 如果right是nums.length-1会越界
    int left = 0, right = nums.length - 2;
    while (left <= right) {
        int mid = left + (right - left) / 2;

        if (nums[mid] > nums[mid + 1]) {
            right = mid - 1;
            // 否则峰值在右边
        } else {
            left = mid + 1;
        }
    }
    return left;
}

    # 二分查找法

    由于 x 平方根的整数部分 ans 是满足 k * k ≤ x 的最大 k 值，因此可以对 k 进行二分查找，从而得到答案。

    二分查找的下界为 0，上界可以粗略地设定为 x。在二分查找的每一步中，只需要比较中间元素 mid 的平方与 x 的大小关系，并通过比较的结果调整上下界的范围。由于所有的运算都是整数运算，不会存在误差，因此在得到最终的答案 res 后，也就不需要再去尝试 res + 1 了。

    public int mySqrt(int x) {
    int left = 0, right = x, res = -1;
  
    // 退出条件是left > right
    while (left <= right) {
        // 计算中值
        int mid = left + (right - left) / 2;
    
        // 如果中值的平方小于等于x, 则更新结果, 并向右移动
        if ((long) mid * mid <= x) {
            res = mid;
            left = mid + 1;
            // 否则向左移动
        } else {
            right = mid - 1;
        }
    }
  
    return res;
}

    public float mySqrt(float num, float e) {

    float res = 0;

            float mid = (left + right) / 2;
        
            if (mid * mid < num - e) {
                left = mid;
            } else if (mid * mid > num + e) {
                right = mid;
            } else {
                res = mid;
                break;
            }
        }
    }
