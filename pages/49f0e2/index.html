    # LRU算法实现

    设计并实现最近最少使用(LRU) 缓存的数据结构. 它应该支持以下操作: get 和 put.

    • get(key): 如果键存在于缓存中, 则获取键的值(总是正数), 否则返回 -1.
    • put(key, value): 如果键不存在, 请设置或插⼊值. 当缓存达到其最大容量时, 它应该在插入新项目之前,
      使最近最少使用的项目无效.

    进阶: 是否可以在 O(1) 时间复杂度内执行两项操作?

    示例:

    LRUCache cache = new LRUCache(2);	// 设置最大容量为2
cache.put(1, 1);
cache.put(2, 2);
cache.get(1); // 返回1
cache.put(3, 3); // 去除key2
cache.get(2); // 返回-1(未找到key2)
cache.get(3); // 返回 3
cache.put(4, 4); // 去除key1
cache.get(1); // 返回-1(未找到key1)
cache.get(3); // 返回3
cache.get(4); // 返回4

    双向链表+哈希表实现

    采用这两种数据结构的组合, get 操作就可以在 O(1) 时间复杂度内完成了. 由于 put 操作要删除的节点⼀般是尾部节点, 所以可以用⼀个变量 tail 时刻记录尾部节点的位置, 这样 put 操作也可以在 O(1) 时间内完成了.

    这里实现就是需要先来一个结点类.

    private static class LRUNode {
    String key;
    Object value;
    // 注意这里是双向链表
    LRUNode next;
    LRUNode pre;

    public LRUNode(String key, Object value) {
        this.key = key;
        this.value = value;
    }
}

    注意: 不管是 put 方法还是 get 方法, 都需要将访问的结点重新放到链表头部, 所以会抽出一个公共的方法.

    public class LRUCache {

    Map<String, LRUNode> map = new HashMap<>();
    LRUNode head;
    LRUNode tail;
    // 缓存最大容量
    int capacity;

    public LRUCache(int capacity) {
        this.capacity = capacity;
    }

    /**
     * 插入元素
     */
    public void put(String key, Object value) {
        // 说明缓存中没有任何元素, 此时需要进行初始化
        if (head == null) {
            head = new LRUNode(key, value);
            // 头尾结点相同
            tail = head;
            // 数值放入缓存
            map.put(key, head);
        }

        // 缓存查询数据
        LRUNode node = map.get(key);

        // 说明缓存中已经存在这个元素了
        if (Objects.nonNull(node)) {
            // 更新值
            node.value = value;
            // 把这个结点从链表删除并且插⼊到头结点
            removeAndInsert(node);
        } else {
            // 说明当前缓存不存在这个值
            LRUNode newHead = new LRUNode(key, value);
            // 如果此处溢出则需要删除最近最近未使用的节点
            if (map.size() >= capacity) {
                map.remove(tail.key);
                // 删除尾结点
                tail = tail.pre;
                tail.next = null;
            }

            map.put(key, newHead);
            // 将新的结点插入链表头部
            newHead.next = head;
            head.pre = newHead;
            head = newHead;
        }
    }

    /**
     * 从缓存中取值
     */
    public Object get(String key) {
        LRUNode node = map.get(key);
        // 如果有值则需要将这个结点放到链表头部
        if (Objects.nonNull(node)) {
            // 把这个节点删除并插⼊到头结点
            removeAndInsert(node);
            return node.value;
        }

        return null;
    }

    private void removeAndInsert(LRUNode node) {
        // 特殊情况先判断, 例如该节点是头结点或是尾部节点
        if (node <mark> head) {
            return;
        } else if (node </mark> tail) {
            tail = node.pre;
            tail.next = null;
        } else {
            node.pre.next = node.next;
            node.next.pre = node.pre;
        }

        // 插⼊到头结点
        node.next = head;
        node.pre = null;
        head.pre = node;
        head = node;
    }
}

    # 抢红包算法

    # 1.题目

    例如一个人在群里发了 100 块钱的红包, 群里有 10 个人一起来抢红包, 每人抢到的金额随机分配. 一般规则:

    • 所有人抢到的金额之和要等于红包金额, 不能多也不能少.
    • 每个人至少抢到 1 分钱.
    • 要保证红包拆分的金额尽可能分布均衡, 不要出现两极分化太严重的情况.
    # 2.二倍均值法

    二倍均值法: 假设剩余红包金额为 m 元, 剩余人数为 n, 那么有如下公式:

    每次抢到的金额 = 随机区间 [0.01, m / n × 2 - 0.01] 元.

    这个公式保证了每次随机金额的平均值是相等的, 不会因为抢红包的先后顺序而造成不公平.

    举个例子: 假设有 5 个人, 红包总额 100 元. 100 ÷ 5 × 2 = 40, 所以第 1 个人抢到的金额随机范围是 [0.01, 39.99] 元, 在正常情况下, 平均可以抢到 20 元. 假设第 1 个人随机抢到了 20 元, 那么剩余金额是 80 元. 80 ÷ 4 × 2 = 40, 所以第 2 个人抢到的金额的随机范围同样是 [0.01, 39.99] 元, 在正常的情况下, 还是平均可以抢到 20 元. 假设第 2 个人随机抢到了 20 元, 那么剩余金额是 60 元. 60 ÷ 3 × 2 = 40, 所以第 3 个人抢到的金额的随机范围同样是 [0.01, 39.99] 元, 平均可以抢到 20 元. 以此类推, 每一次抢到金额随机范围的均值是相等的.

    注意: 传入的钱是以分为单位, 所以最后结果列表中红包是按分计算的, 需要自己转换为元.

    public static List<Integer> divideRedPackage(int money, int people) {
    List<Integer> moneyList = new ArrayList<>();
    int restMoney = money;
    int restPeople = people;
  
    Random random = new Random();
  
    // 这里对people-1个人进行这个算法, 最后一个人直接取剩下的金额
    for (int i = 0; i < people - 1; i++) {
        // 随机取值的范围: [1, 剩余人均金额的2倍-1] 单位是:分
        int curMoney = random.nextInt(restMoney / restPeople * 2 - 1) + 1;
        moneyList.add(curMoney);
        // 剩余金额和人数减少
        restMoney = restMoney - curMoney;
        restPeople--;
    }
  
    // 最后还剩一份
    moneyList.add(restMoney);
  
    return moneyList;
}

public static void main(String[] args) {
    List<Integer> amountList = divideRedPackage(1000, 10);
    for (int amount : amountList) {
        // 由于结果是分, 这里转换为元
        System.out.println("抢到金额:" + new BigDecimal(amount).divide(new BigDecimal(100)));
    }
}

    # 实现生产者与消费者模型

    # 1.wait和notify实现

    可以将生产者和消费者需要的方法写在公共类中. 也就是创建一个资源类.

    static class Resource {
    // 最大容量
    private static final int CAPACITY = 5;
    // 缓存
    LinkedList<String> linkedList = new LinkedList<>();

    /**
     * 生产者方法
     */
    public void product() throws Exception {
        // 注意这里是对资源linkedList加锁(不能对this加锁)
        synchronized (linkedList) {
            // 容量已满则等待
            while (CAPACITY == linkedList.size()) {
                System.out.println("仓库已满, 【生产者】: 暂时不能执行生产任务!");
                linkedList.wait();
            }
            // 生产一个产品
            linkedList.push("李四");
            System.out.println("【生产者】: 生产了一个产品\t【现仓储量为】:" + linkedList.size());
            // 通知消费者进行消费
            linkedList.notifyAll();
        }
    }

    /**
     * 消费者方法
     */
    public void customer() throws Exception {
        // JDK的void notifyAll()解除那些在该对象上调用wait()方法的线程的阻塞状态.
        synchronized (linkedList) {
            // 多线程判断中使用while, 不要使用if否则会出现虚假唤醒问题
            while (linkedList.isEmpty()) {
                System.out.println("【消费者】: 仓库无货, 暂时不能执行消费任务!");
                linkedList.wait();
            }

            // 有产品则消费
            linkedList.pop();
            System.out.println("【消费者】: 消费了一个产品\t【现仓储量为】:" + linkedList.size());

            // 通知生产者进行生产
            linkedList.notifyAll();
        }
    }
}

    # 2.await与signalAll实现

    将公共的属性和方法放在 Resouce 类中, 在资源类中使用 Lock 中的 lock() 进行加锁, 控制并发操作. 使用 await() 方法阻塞线程, 使用 signalAll() 唤醒线程.

    资源类:

    static class Resouce {
    private final int MAX_CAPACITY = 3;
    private final int MIN_CAPACITY = 0;
    private int number = 0;
    // 显式锁
    private final Lock lock = new ReentrantLock();
    private final Condition condition = lock.newCondition();

    // 生产方法
    public void product() {
        try {
            // 显式加锁
            lock.lock();
            // 如果生产的数量大于最大值则阻塞
            while (number >= MAX_CAPACITY) {
                System.out.println("【生产者】: 当前仓库满了, 停止生产");
                // 阻塞等待
                condition.await();
            }
            number++;
            System.out.println("【生产者】: 生产了一个产品\t【现仓储量为】:" + number);
            // 生产完毕唤醒所有等待的线程
            condition.signalAll();
        } catch (InterruptedException e) {
            e.printStackTrace();
        } finally {
            // 释放锁
            lock.unlock();
        }
    }

    // 消费方法
    public void consumer() {
        try {
            // 加锁
            lock.lock();
            // 没有产品可以消费则阻塞等待
            while (number <= MIN_CAPACITY) {
                System.out.println("【消费者】: 当前没有产品可以消费");
                // 阻塞等待
                condition.await();
            }
            number--;
            System.out.println("【消费者】: 消费了一个产品\t【现仓储量为】:" + number);
            // 消费完毕唤醒所有等待的线程
            condition.signalAll();
        } catch (InterruptedException e) {
            e.printStackTrace();
        } finally {
            // 释放锁
            lock.unlock();
        }
    }
}

    public class Resouce {

    private int MAX_VALUE = 3;
    private int MIN_VALUE = 0;
    private int number = 0;
    // 显式锁
    private Lock lock = new ReentrantLock();
    private Condition condition = lock.newCondition();

    // 生产方法
    public void product() {
        try {
            lock.lock();
            // 如果生产的数量大于最大值则阻塞
            while (number >= MAX_VALUE) {
                System.out.println("【生产者】: 当前仓库满了, 停止生产");
                condition.await();
            }
            number++;
            System.out.println("【生产者】: 生产了一个产品\t【现仓储量为】:" + number);
            condition.signalAll();
        } catch (InterruptedException e) {
            e.printStackTrace();
        } finally {
            lock.unlock();
        }
    }

    // 消费方法
    public void consumer() {
        try {
            lock.lock();
            // 如果消费的值=0则阻塞
            while (number <= MIN_VALUE) {
                System.out.println("【消费者】: 当前没有产品可以消费");
                condition.await();
            }
            number--;
            System.out.println("【消费者】: 消费了一个产品\t【现仓储量为】:" + number);
            condition.signalAll();
        } catch (InterruptedException e) {
            e.printStackTrace();
        } finally {
            lock.unlock();
        }
    }
}

    # 覆写equals方法

    写出完美equals方法的建议(源自 JavaCore):

    • 检测 this 与 otherObject 是否引用同一对象(地址相同).
    • 检测 otherObject 是否为 null(null 就肯定不是了).
    • 比较 this 与 otherObject 是否属于同一个类(类型不同也不是).
    • 将 otherObject 类型转换为相应的类类型变量.
    • 现在开始对所有需要比较的域依次进行比较. 使用 == 比较基本类型域, 使用 equals 比较对象域. 这里就需要看对不同类自己的情况写.
    // 以下是Employee类的equals方法
public boolean equals(Object otherObject){
     // 检测this与otherObject是否引用同一对象
    if(this <mark> otherObject) return true;   
    // 检测otherObject是否为null
    if(otherObject </mark> null) return false;   
    // 比较this与otherObject是否属于同一种类型
    if(this.getClass() != otherObject.geClass()) return false;
    // 将otherObject转换为相应的类类型变量: 类型转换
    Employee other = (Employee) otherObject;  
    // 现在开始对所有需要比较的域进行比较. 使用 <mark> 比较基本类型域, 使用equals比较对象域. 
    return Object.equals(name, other.name) && salary </mark> other.salary && Objects.equals(hireDay, other.hireDay);
}
