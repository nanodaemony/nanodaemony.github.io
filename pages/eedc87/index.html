    # 链表类题目总结🌸

    # 1.链表类问题解题技巧
    # (1)使用额外空间

    使用额外空间(如 HashMap, 数组)来辅助操作.

    场景: 复制链表的复制, 判断链表是否有环等等.

    额外空间会增加空间复杂度, 而且很多题 O(1) 空间就行了, 不过使用额外空间在笔试中很不错.

    # (2)引入dummy结点

    对于头结点可能会改变的题目, 引入 dummy 结点是基操, 最后返回 dummy.next 即可.

    场景: 删除某个结点(可能把头结点删了), 两两一个交换结点(头结点会被交换出去)等.

    # (3)快慢指针或双指针

    由于链表无法直接寻址, 因此可以引入多个指针. 快慢指针是非常巧妙且常见的技巧!

    场景: 找链表中间结点, 判断链表是否有环, 删除倒数第 K 个结点, 奇偶链表等.

    # (4)牢记链表逆序的方法

    很多题都需要链表逆序! 逆序可以使用栈, 递归, 迭代等方式实现. 栈比较费空间, 递归较难理解, 利用迭代可以通过画图的方式推出迭代路线.

    面试一定要掌握迭代的方式逆序.

    场景: 判断链表是否是回文结构(逆序后半部分), 两个链表数字相加(这个题用栈逆序贼方便, 看实际情况吧), 反转指定范围链表等.

    # (5)学会画迭代思路图

    不少题目使用额外空间很容易, 但是空间浪费, 但可以通过迭代的方式做出来, 因此要学会分析题目画出迭代的关系, 指针的变化步骤与方式.

    场景: 链表逆序(迭代方式), 两两一组交换链表, 奇偶链表等.

    # (6)其他

    链表是空节点, 或者有一个值和一个指向下一个链表的指针, 因此很多链表问题可以用递归来处理.

    遇到需要逆序的情况, 如果不好处理, 可以考虑使用栈.

    链表和树的问题, 一般都可以有递归和迭代两种写法.

    这里 fast 和 slow 初始时都指向的是 head 节点, 由于 fast 一次移动个节点, 所以需要判断 "fast.next.next != null​", 移动完成后, "slow.next" 就是指向后半部分链表的开头.

    为什么 slow.next 一定是后半链表的开头? 这里需要分两种情况.

    结点数目为奇数

    这种情况下前面半条链表比后面半条链表多一个节点. fast 移动到最后一个节点就会停止, 此时 slow 到 4 的位置, slow.next 就是后半部分的开头.

    结点数目为偶数

    这种情况下两条链表节点数目一致. fast 移动到倒数第二个节点就会停止, 此时 slow 到 3 的位置, slow.next 就是后半部分的开头.

    # 单指针遍历两次法

    使用单个指针, 遍历两次即可, 第一次记录链表长度, 第二次遍历长度的一半即可.

    public ListNode middleNode(ListNode head) {
    // 第一次遍历求链表长度
    int length = 0;
    ListNode cur = head;
    while (cur != null) {
        length++;
        cur = cur.next;
    }

    // 第二次遍历只走一半
    int k = 0;
    cur = head;
    while (k < length / 2) {
        k++;
        cur = cur.next;
    }
    return cur;
}

    # 双指针法

    使用快慢双指针, 快指针一次走两步, 慢指针一次走一步. 当快指针到链尾部, 慢指针就在中间了. 具体图示参考基础部分.

    public ListNode middleNode(ListNode head) {
    // 初始时快慢指针都在头部
    ListNode slow = head, fast = head;
    // 迭代退出条件为快指针到了尾部
    while (fast != null && fast.next != null) {
        // 慢指针移动一步 快指针移动两步
        slow = slow.next;
        fast = fast.next.next;
    }
    return slow;
}

    # 逐个遍历插入排序法

    与传统插入排序不同, 无法对已排序链表从尾至首遍历进行插入. 这里选择从头往后遍历进行逐渐的插入.

    • 引入值为无穷小的 dummy 结点.
    • 对已排序链表部分从首至尾遍历, 每次都从 dummy 开始找到所有小于当前遍历元素的位置, 将当前元素进行插入.

    ​​

    代码如下:

    public ListNode insertionSortList(ListNode head) {
    if (head <mark> null || head.next </mark> null) {
        return head;
    }

    // 引入dummy结点
    ListNode dummy = new ListNode(Integer.MIN_VALUE);

    // 遍历链表处理每一个结点
    while (head != null) {
      
        // 每次都从dummy开始遍历找到合适的位置插入
        ListNode cur = dummy;

        // 找到合适的插入位置
        while (cur.next != null && cur.next.val < head.val) {
            cur = cur.next;
        }

        // 插入当前结点
        ListNode temp = head;
        head = head.next;
        temp.next = cur.next;
        cur.next = temp;
    }

    return dummy.next;
}

    # Map法

    使用一个 Map 来保存原有结点与副本结点的对应关系, 然后根据原结点的关系从 map 中取对应的副本结点并设置关系. map 中的结点的指针关系依然没变.

    private static RandomListNode copyListWithRandomPointer(RandomListNode head) {
    // 使用Map存储原始节点与对应复制后节点的关系
    HashMap<RandomListNode, RandomListNode> nodeMap = new HashMap<>();

    // 遍历第一次将全部节点放入map中, map中结点的指针关系依然没变
    RandomListNode cur = head;
    while (cur != null) {
        nodeMap.put(cur, new RandomListNode(cur.value));
        cur = cur.next;
    }
  
    cur = head;
    // 不断从map中取之前的节点设置副本的关系
    while (cur != null) {
        nodeMap.get(cur).next = nodeMap.get(cur.next);
        nodeMap.get(cur).random = nodeMap.get(cur.random);
        cur = cur.next;
    }
    return nodeMap.get(head);
}

    # 迭代复制法(空间较优)

    空间复杂度降为 O(1) .

    • 遍历两次, 第一次遍历在每个节点的后面复制一个值一样的结点.
    • 第二次遍历时根据原有结点的随机指针设置复制后的随机指针(因为原结点与复制结点是挨着的, 所以有位置关系).
    • 最后将复制后的链表与原链表完全分开即可(这是必须要做的!!!).

    动图如下:

    代码如下.

    public Node copyRandomList(Node head) {
    // Base case
    if (head == null) {
        return null;
    }
    // 每个结点重复复制一次
    Node cur = head;
    while (cur != null) {
        Node temp = new Node(cur.val);
        temp.next = cur.next;
        cur.next = temp;
        cur = cur.next.next;
    }

    // 从头开始复制随机指针
    cur = head;
    while (cur != null) {
        // 处理随机指针
        if (cur.random != null) {
            // 当前结点与下一个结点的随机指针指向的结点也是相邻的!!!
            cur.next.random = cur.random.next;
        }
        // 指针移动两步
        cur = cur.next.next;
    }
  
    // 将两条链表完全拆分
    cur = head;
    Node copyHead = cur.next;
    Node copyCur = copyHead;
    while (copyCur.next != null) {
        // 移动原链表指针
        cur.next = cur.next.next;
        cur = cur.next;
        // 移动复制链表指针
        copyCur.next = copyCur.next.next;
        copyCur = copyCur.next;
    }
    // 结束标志null
    cur.next = null;
    return copyHead;
}

    # 辅助set法

    利用额外空间进行解答. 首先将数组内容放入一个 Set 中, 用于快速判断是否包含该元素. 遍历链表, 每个结点都去 Set 中看是否包含这个值. 如果当前值在 set 中但是下一个值不在 set 中, 则认为找到一个组件. 注意需要处理变量到末尾的情况.

    public int numComponents(ListNode head, int[] nums) {
    // 引入Set
    Set<Integer> set = new HashSet();
    for (int num: nums) {
        set.add(num);
    }

    ListNode cur = head;
    int res = 0;

    while (cur != null) {
        // 如果set包含这个值但不包含下一个值,则组件+1
        if (set.contains(cur.val)) {
            // 已经到链表末尾,当前的也算一个组件
            if (cur.next == null) {
                res++;
                break;
            }

            // 如果下一个值不包含则找到一个组件
            if (!set.contains(cur.next.val)) {
                res++;
            }
        }
        cur = cur.next;
    }
    return res;
}

    # LRU缓存(146-中等)🌸💯🔋
    # 题目描述

    请设计并实现一个满足 LRU (最近最少使用) 缓存约束的数据结构.

    实现 LRUCache 类:

    • LRUCache(int capacity) 正整数作为容量 capacity 初始化 LRU 缓存.
    • int get(int key) 如果关键字 key 存在于缓存中, 则返回关键字的值, 否则返回 -1.
    • void put(int key, int value) 如果关键字 key 已经存在, 则变更其数据值 value; 如果不存在, 则向缓存中插入该组 key-value. 如果插入操作导致关键字数量超过 capacity, 则应该逐出最久未使用的关键字.

    函数 get 和 put 必须以 O(1) 的平均时间复杂度运行.

    https://leetcode.cn/problems/lru-cache/description/ (opens new window)

    # 双向链表+哈希表实现

    为了快速获取元素, 可以使用哈希表. 为了能够方便在链表中移动元素, 可以使用双向链表.

    • 初始化: 初始化两个结点 head 和 tail 并链接起来, 这两个结点实际上不存储数据, 只是为了方便寻找队头结点和队尾结点. 同时初始化一个 map, 并记录 LRU 的最大容量值.
    • put 方法: 首先看 map 中是否包含当前元素, 是则直接更新元素值, 并将其移动到队头结点. 否则新增一个结点, 将其插入到队头结点, 同时也要加入到 map 中. 如果元素个数超了, 直接删除队尾结点.
    • get 方法: 看 map 中是否包含当前元素, 是则直接更新元素值, 并将其移动到队头结点.

    这里无论是 get 还是 put, 都存在将已有结点移动到队头的操作, 所以可以抽取一个方法. 同时, 为了方便的表示结点, 需要自己定义一个双向链表的结点类.

    ​​

    代码如下:

    class LRUCache {

    // 双向链表节点类
    class Node {
        int key;
        int value;
        Node prev;
        Node next;

        public Node(int key, int value) {
            this.key = key;
            this.value = value;
        }
    }

    // 记录最大大小
    private int maxSize;
    // 存储key到结点的映射
    private Map<Integer, Node> map = new HashMap<>();

    // 双向链表的头尾指针
    private Node head = new Node(Integer.MAX_VALUE, Integer.MAX_VALUE);
    private Node tail = new Node(Integer.MAX_VALUE, Integer.MAX_VALUE);

    public LRUCache(int capacity) {
        this.maxSize = capacity;
        // 初始化头尾指针
        head.next = tail;
        tail.prev = head;
    }

    public int get(int key) {
        // 不存在缓存值
        if (!map.containsKey(key)) {
            return -1;
        }
        Node curNode = map.get(key);
        // 移动当前结点为队头结点
        refreshNodeToHead(curNode);
        return curNode.value;
    }

    public void put(int key, int value) {
        // 结点已经存在, 刷新结点值并刷新结点位置到队头
        if (map.containsKey(key)) {
            // 更新结点值
            map.get(key).value = value;
            // 使结点成为队头结点
            refreshNodeToHead(map.get(key));
            return;
        }

        // 新增一个结点并放在队头
        Node newNode = new Node(key, value);
        // 注意map中同样需要记录该结点
        map.put(key, newNode);
        insert2Head(newNode);

        // 如果元素个数超了 直接删除队尾结点
        if (map.size() > maxSize) {
            removeLastNode();
        }
    }

    // 删除队尾结点
    private void removeLastNode() {
        map.remove(tail.prev.key);
        tail.prev = tail.prev.prev;
        tail.prev.next = tail;
    }

    // 插入结点到最前面成为队头结点
    private void insert2Head(Node node) {
        node.next = head.next;
        head.next.prev = node;
        head.next = node;
        node.prev = head;
    }

    // 刷新结点到最前面成为队头结点
    private void refreshNodeToHead(Node node) {
        // 将结点从中间去掉
        node.prev.next = node.next;
        node.next.prev = node.prev;
        // 把结点移动到最前面
        insert2Head(node);
    }
}

    # 迭代法

    也叫头插法. 时间复杂度 O(N), 空间复杂度 O(1) 的算法. 引入 dummy 结点, 然后把第一个结点先弄好, 之后每次将新的结点插入到 dummy 结点的后面一个. (1)temp 结点指向 head; (2)head 后移; (3)temp 指向 dummy 的后一个结点; (4)dummy 指向 temp. 此时新的 temp 结点处理完成.

    public ListNode reverseList(ListNode head) {

    if (head <mark> null || head.next </mark> null) {
        return head;
    }

    ListNode dummy = new ListNode(0);

    while (head != null) {
        // 临时结点指向head
        ListNode temp = head;
        // head往后移动,此时temp指向的就是需要操作的结点
        head = head.next;
        // 将temp结点插入dummy后面
        temp.next = dummy.next;
        dummy.next = temp;
    }

    return dummy.next;
}

    # 迭代法

    将链表分为 3 个部分, 2 个不反转部分和 1 个反转的部分.

    • 前面的不反转部分, 保留最后一个节点, 后面的不反转部分, 保留第一个节点.
    • 对于反转部分, 要保留第一个端点.
    • 反转结束后, 将三个部分拼接起来.

    ​​

    由于反转的可能涉及第一个结点的变动, 所以引入 dummy 结点. 同时引入一个 cur 指针, 从前往后一直遍历.

    这里假设 start = 4, end = 6. cur 指针需要移动 3 次到达不需要反转部分的末尾即 originEnd, 记录一下. 其下一个结点就是需要反转部分的开始, 即 revStart, 记录一下. 此时让 cur 再走 3 次, 到达需要反转部分的末尾, 此时其下一个结点即末尾无需反转的开始, 记录一下. 同时这里将需反转部分与最后的部分的连接断开. 最后对中间部分进行反转, 然后依次进行拼接即可. 注意: 这里 revStart 反转后就成了反转部分的末尾, 因此直接与 lastStart 进行拼接即可.

    public ListNode reverseBetween(ListNode head, int start, int end) {
    // 引入dummy与cur结点
    ListNode dummy = new ListNode(0);
    dummy.next = head;
    ListNode cur = dummy;

    // 找第一部分不用变的末尾结点
    for(int i = 0; i < start - 1; i++) {
        cur = cur.next;
    }
    // 记录第一部分不用变化的末尾节点
    ListNode originEnd = cur;

    // 记录需要反转部分的起始结点
    ListNode revBegin = originEnd.next;
    // end - start = 2
    for(int i = 0; i <= end - start; i++) {
        cur = cur.next;
    }

    // 此时p为需要变的部分的末尾, 其后面一个结点即为后面不需要变得部分的头结点
    ListNode lastStart = cur.next;

    // 当前cur指向需要反转链表的最后一个节点, 此时断开指针, 这样方便进行链表反转
    cur.next = null;

    // 反转链表. 经过反转之后返回了第二部分的新的头结点, revBegin指针已经指向了反转后链表的末尾
    ListNode newHead = reverse(revBegin);
  
    // 第一部分最后的结点连接到反转后的链表头部
    originEnd.next = newHead;
    // 反转后的链表末尾连接最后一段的开头
    revBegin.next = lastStart;

    return dummy.next;
}

    首先使用双指针方式找到链表中间结点, 然后从此处反转链表的右半部分, 使其最后指向中间结点, 然后分别从两个链表的开头进行比较是否为回文串, 注意最后需要将反转的链表恢复为原样.

    public boolean isPalindrome(ListNode head) {
    if (head <mark> null || head.next </mark> null) {
        return true;
    }

    // 快慢双指针找到前半部分的末尾
    ListNode firstHalfEnd = getEndOfFirstHalf(head);

    // 反转后半部分并得到后半部分开头
    ListNode secondHalfStart = reverse(firstHalfEnd.next);

    // 挨着检查两段链表的数值是否相同
    ListNode cur1 = head;
    ListNode cur2 = secondHalfStart;
  
    boolean result = true;
    while (result && cur2 != null) {
        // 记录结果(这里不能直接返回, 因为要把后面的半部分逆序回来)
        if (cur1.val != cur2.val) {
            result = false;
        }
        cur1 = cur1.next;
        cur2 = cur2.next;
    }
  
    // 注意把后面的半部分逆序回来并连接到前半部分
    firstHalfEnd.next = reverse(secondHalfStart);
  
    return result;
}

private ListNode getEndOfFirstHalf(ListNode head) {

    ListNode slow = head;

        fast = fast.next.next;
        slow = slow.next;
    }
    return slow;
}

    # 快慢双指针迭代反转法

    题目要求算法只能使用常数的额外空间. 说明不能用递归, 只能用迭代的方式进行, 可以画个图理清每一个临界位置的指向.

    ​​

    • 引入 dummy 结点, 并初始化双指针 fast 和 slow. 然后进入死循环遍历.
    • 通过 k 次循环将 fast 指针移动到需要反转部分的末尾. 循环中就可能遇到空, 此时退出循环即可.

    • 然后需要定义 remainHead 指向后面剩余未处理的头结点, 定义 revHead 指向本轮需要反转部分的头结点, 以便将翻转元素前后两部分连接起来.
    • 断开需反转部分和剩余部分的指针, 并进行反转, 反转后将结果拼接到 slow 指针之后.
    • 最后重新建立连接, 并将快慢指针重置到当前的 revHead 处, 持续进行下一轮循环.

    public ListNode reverseKGroup(ListNode head, int k) {
    ListNode dummy = new ListNode(0);
    // 将dummy指向head
    dummy.next = head;

    // 快慢双指针
    ListNode fast = dummy;
    ListNode slow = dummy;

    while (true) {
        // fast遍历到需要翻转部分的最后一个结点的位置
        for (int i = 0; i < k && fast != null; i++) {
            fast = fast.next;
        }
        // 说明fast到了链表尾部,且此轮反转长度不够,直接跳出循环
        if (fast == null) {
            break;
        }
      
        // 缓存剩下的未处理链表的头结点
        ListNode remainHead = fast.next;
        // 断开链表 此时存在两条链表, 前面的链表是需要反转的部分, 后面的链表是剩下的未处理部分
        fast.next = null;
      
        // slow指向的是需要翻转部分的第一个结点
        ListNode revHead = slow.next;

        // 然后通过翻转方法reverse()后接在slow后面
        slow.next = reverse(revHead);

        // 将两部分链表连接起来
        revHead.next = remainHead;

        // 一轮反转后重置指针
        slow = revHead;
        fast = revHead;
    }

    return dummy.next;
}

        return head;
    }

    return newHead;

    # 逐一比较迭代法

    每次比较 k 个节点(每个链表的首节点), 获得最小值的节点. 然后不断将节点接在最终有序链表的后面.

    public ListNode mergeKLists(ListNode[] lists) {
    // K条链表
    int k = lists.length;

    ListNode dummy = new ListNode(0);
    ListNode cur = dummy;

    // 遍历处理所有链表, 直到所有链表都为空
    while (true) {
        // 记录最小的节点
        ListNode minNode = null;

        // 记录最小节点所在的链表位置
        int minPointer = -1;
        // 遍历所有链表找到最小的头节点的链表
        for (int i = 0; i < k; i++) {
            // 如果当前链表为空则跳过
            if (lists[i] <mark> null) {
                continue;
            }
            // 如果当前链表的头节点比之前的最小节点小则更新结点和指针
            if (minNode </mark> null || lists[i].val < minNode.val) {
                minNode = lists[i];
                minPointer = i;
            }
        }

        // 如果没有找到最小的节点则说明所有链表都为空了, 直接退出
        if (minPointer == -1) {
            break;
        }

        // 找到了最小的节点, 将其加入到新链表中
        cur.next = minNode;
        cur = cur.next;

        // 更新本次处理链表的头节点
        lists[minPointer] = lists[minPointer].next;
    }

    return dummy.next;
}

    # 分治递归法

    可以对链表进行两两合并, 最终即可得到合并的链表, 但这样会造成链表的重复合并. 可以使用分治的思想进行合并(类似于归并排序). 即找到处于中间位置的链表, 然后分别合并左边部分与右边部分, 再将两部分结果合并. 如下图所示.

    这里合并两条有序链表时采用了前述 "合并两个有序的链表" 的递归代码, 整体代码如下:

    // 迭代法
public ListNode mergeKLists(ListNode[] lists) {
    // Base case
    if (lists <mark> null || lists.length </mark> 0) {
        return null;
    }
    return merge(lists, 0, lists.length - 1);
}

private ListNode merge(ListNode[] lists, int left, int right) {
    // 只有一条链表, 直接返回
    if (left == right) {
        return lists[left];
    }

    // 找到中间位置的链表
    int mid = left + (right - left) / 2;
    // 递归合并左右两部分链表
    ListNode l1 = merge(lists, left, mid);
    ListNode l2 = merge(lists, mid + 1, right);

    // 最后归并合并两个列表
    return mergeTwoLists(l1, l2);
}

// 合并两个有序链表的基础代码
private ListNode mergeTwoLists(ListNode l1, ListNode l2) {
    if (l1 <mark> null) return l2;
    if (l2 </mark> null) return l1;

    if (l1.val < l2.val) {
        l1.next = mergeTwoLists(l1.next, l2);
        return l1;
    } else {
        l2.next = mergeTwoLists(l1, l2.next);
        return l2;
    }
}

    public ListNode sortList(ListNode head) {
    // Base case: 拆分到只剩一个结点或者没有结点
    if(head <mark> null || head.next </mark> null) {
        return head;
    }

    // 快慢双指针寻找链表中点
    ListNode fast = head;
    ListNode slow = head;
    while (fast.next != null && fast.next.next != null) {
        fast = fast.next.next;
        slow = slow.next;
    }

    // 记录后半部分的开头并断开两部分的指针
    ListNode secondHead = slow.next;
    slow.next = null;

    // 分治, 递归对前后两部分链表进行排序
    ListNode slowTemp = sortList(head);
    ListNode fastTemp = sortList(secondHead);

    return mergeTwoLists(slowTemp, fastTemp);
}

public ListNode mergeTwoLists(ListNode list1, ListNode list2) {
    // Base case
    if (list1 <mark> null || list2 </mark> null) {
        return list1 != null ? list1 : list2;

    // 到这里说明两个链表均不为空, 指定方法返回的head
    ListNode head = null;

    if (list1.val < list2.val) {
        head = list1;
        head.next = mergeTwoLists(list1.next, list2);
    } else {
        head = list2;
        head.next = mergeTwoLists(list1, list2.next);
    }

    // 返回合并后的头结点
    return head;
}

    public ListNode partition(ListNode head, int x) {
    if (head <mark> null || head.next </mark> null) {
        return head;

    ListNode dummyLess = new ListNode(0);
    ListNode dummyMore = new ListNode(0);

    ListNode curMore = dummyMore;

            curLess.next = head;
            curLess = curLess.next;

            curMore.next = head;
            curMore = curMore.next;

        head = head.next;

    curMore.next = null;
  
    // 链表拼接并返回
    curLess.next = dummyMore.next;
  
    return dummyLess.next;
}

    # 迭代拆分法

    1. 先计算链表长度.
    2. 计算出每条链表至少可以分 cutCnt 个节点, 由于存在可能不能刚好均分的情况, 所以需要计算一下前 moreCnt 条链表可以多分一个节点.

    public ListNode[] splitListToParts(ListNode root, int k) {
    // 统计链表长度
    int length = 0;
    ListNode cur = root;
    while (cur != null) {
        length++;
        cur = cur.next;
    }

    // 计算每个部分的平均长度
    int avgLength = length / k;
    // 计算需要多一个结点的链表数
    int moreCnt = length % k;

    ListNode[] res = new ListNode[k];
  
    // 拆分链表为k个部分
    for (int i = 0; i < k; i++) {
        ListNode dummy = new ListNode(-1);
      
        // 计算当前部分的链表长度, 如果有多余的则多一个结点
        int curLength = moreCnt > 0 ? avgLength + 1 : avgLength;
        moreCnt--;

        // 持续拼接链表
        ListNode temp = dummy;
        while (curLength-- > 0) {
            temp.next = root;
            temp = temp.next;
            root = root.next;
        }
      
        // 记录结果
        temp.next = null;
        res[i] = dummy.next;
    }

    return res;
}

    由于是不断从一头一尾取元素. 先利用快慢指针找到链表的中间结点. 将后半部分链表逆序(可以迭代或者递归逆序). 最后将前后两个部分的链表交替连接合并起来即可.

    public void reorderList(ListNode head) {

    if (head <mark> null || head.next </mark> null || head.next.next == null) {
        return;

    // 利用快慢双指针找中点将链表分成两部分
    ListNode slow = head;
    ListNode fast = head;
    while (fast.next != null && fast.next.next != null) {
        slow = slow.next;
        fast = fast.next.next;
    }
  
    // 断开指针. 将链表分成两个部分, newHead为后半部分链表的头部
    ListNode secondHead = slow.next;
    slow.next = null;

    secondHead = reverse(secondHead);

    while (secondHead != null) {
        ListNode temp = secondHead.next;
        secondHead.next = head.next;
        head.next = secondHead;
        head = secondHead.next;
        secondHead = temp;
    }
}

private ListNode reverse(ListNode head) {

        return head;
    }
    ListNode newHead = reverse(head.next);
    head.next.next = head;
    head.next = null;
    return newHead;
}

    题目的进阶版是不能修改原链表. 然而需要实现逆序的操作, 此时可以考虑采用栈进行逆序. 此外, 每次处理一轮之后, 新生成的结点直接采用头插法进行链表构造就能省去最后链表逆序的操作. 采用栈之后的代码也很类似.

    public ListNode addTwoNumbers(ListNode list1, ListNode list2) {
    if (list1 <mark> null || list2 </mark> null) {
        return list1 == null ? list2 : list1;
    }

    // 使用栈存储链表元素, 模拟逆序链表
    Stack<Integer> stack1 = new Stack<>();
    Stack<Integer> stack2 = new Stack<>();
    while (list1 != null) {
        stack1.push(list1.val);
        list1 = list1.next;
    }
    while (list2 != null) {
        stack2.push(list2.val);
        list2 = list2.next;

    int carry = 0;
    ListNode dummy = new ListNode(0);
    ListNode cur = dummy;

    // 持续计算到两个栈都为空
    while (!stack1.isEmpty() || !stack2.isEmpty()) {

        int num1 = stack1.isEmpty() ? 0 : stack1.pop();
        int num2 = stack2.isEmpty() ? 0 : stack2.pop();

        // 刷新进位值
        carry = sum / 10;

        // 直接采用头插法构造结果(最后就无需再次逆序结果)
        ListNode temp = new ListNode(sum % 10);
        temp.next = cur.next;
        cur.next = temp;

    if (carry > 0) {
        ListNode temp = new ListNode(carry);
        temp.next = cur.next;
        cur.next = temp;
    }

    # 连接成环再断开法

    将整个链表收尾相连成一个环, 然后计算出链表长度, 并取模计算出实际需要移动的结点数, 最后再断开指针即可.

    • 求出链表的长度 length. 并找到链表的末尾结点.
    • 由于是循环移动, 所以 k = k % length 取模就是需要右移的距离.
    • 连接成环. 将原始链表首尾相连.
    • 移动头结点到最终需要断开环的位置, 移动次数画个图计算一下即可, 比如上图中只需要移动 1 次.
    • 断开环. 此时指针的下一个结点即是新的头结点, 返回它即可. 这里需要断开慢指针与新头结点的指针.
    public ListNode rotateRight(ListNode head, int k) {
    if (head <mark> null || head.next </mark> null) {
        return head;
    }

    // 计算链表长度
    int length = 1;
    ListNode cur = head;
    while (cur.next != null) {
        length++;
        cur = cur.next;
    }

    // 计算实际需要移动的次数
    k = k % length;

    // 无需旋转直接返回
    if (k == 0) {
        return head;
    }

    // 连接尾部到头部形成一个环
    cur.next = head;

    // 找到新的头和新的尾
    int moveCnt = length - k;

    // 移动头结点, 执行完成后head指向旋转后的末尾结点
    for (int i = 1; i < moveCnt; i++) {
        head = head.next;
    }
  
    // 最终的尾结点指针
    ListNode tail = head;
    ListNode newHead = tail.next;
    // 断开环
    tail.next = null; 

    return newHead;
}

    # 双指针法

    如果两个无环链表相交, 那么从相交节点开始, 一直到两个链表终止的这段, 二者共享.

    • 分别统计两条链表的长度.
    • 对比两个末尾结点, 如果不是同一个结点则肯定没有相交.
    • 重新将两个指针分别指向链表开头. 计算链表长度的差值 gap, 让长的一条链表先移动 gap.
    • 最后两个指针一起移动, 直到指针相遇即为第一个相交结点.

    注意编码技巧: 这里可以让一个指针始终指向较长的链表, 这样方便移动.

    public static ListNode getIntersectionNode(ListNode head1, ListNode head2) {
    if (head1 <mark> null || head2 </mark> null) {
        return null;
    }
    // 计算两个链表的长度
    int len1 = 0, len2 = 0;

    ListNode cur1 = head1;
    ListNode cur2 = head2;

    while (cur1.next != null) {
        cur1 = cur1.next;
        len1++;
    }
    while (cur2.next != null) {
        cur2 = cur2.next;
        len2++;
    }
  
    // 比较尾部两个指针, 如果不相同则肯定不相交
    if (cur1 != cur2) {
        return null;
    }

    // 编码技巧: 让cur1始终指向较长的一根链表 cur2指向较短的链表
    cur1 = (len1 - len2 > 0) ? head1 : head2;
    cur2 = (cur1 == head1) ? head2 : head1;
    // 求两个链表的长度差
    int gap = Math.abs(len1 - len2);
    // 链表长的先走gap步
    while (gap != 0 && cur1 != null) {
        cur1 = cur1.next;
        gap--;
    }

    // 两个链表同时运动找第一个相同的点
    while (cur1 != cur2) {
        cur1 = cur1.next;
        cur2 = cur2.next;
    }
  
    return cur1;
}

    # 交换指针法

    设 A 的长度为 a + c, B 的长度为 b + c, 其中 c 为尾部公共部分长度, 可知 a + c + b = b + c + a.

    当访问链表 A 的指针访问到链表尾部时, 令它从链表 B 的头部重新开始访问链表 B; 同样地, 当访问链表 B 的指针访问到链表尾部时, 令它从链表 A 的头部重新开始访问链表 A. 这样就能控制访问 A 和 B 两个链表的指针能同时访问到交点.

    这个解法在两个指针都指向另外一个链表后, 它们已经在“同一起跑线”, 所以两个指针同时移动一位, 当他们为同一结点时 while 循环结束, 返回交结点.

    public ListNode getIntersectionNode(ListNode head1, ListNode head2) {
    ListNode cur1 = head1, cur2 = head2;
    while (cur1 != cur2) {
        // cur1为空时指向head2
        cur1 = (cur1 == null) ? head2 : cur1.next;
        // cur2为空时指向head1
        cur2 = (cur2 == null) ? head1 : cur2.next;
    }
    return cur1;
}

    # 在O(1)时间内删除链表中指定的一个任意结点🔑🌸
    # 题目描述

    编写一个函数, 使其可以删除某个链表中给定的(可能是末尾)节点, 将只被给定要求被删除的节点.

    # 赋值删除法

    ① 如果该节点不是尾节点, 那么可以直接将下一个节点的值赋给该节点, 然后令该节点指向下下个节点, 再删除下一个节点, 时间复杂度为 O(1).

    ② 否则, 就需要先遍历链表, 找到节点的前一个节点, 然后让前一个节点指向 null, 时间复杂度为 O(N) .

    综上, 如果进行 N 次操作, 那么大约需要操作节点的次数为 N-1+N=2N-1, 其中 N-1 表示 N-1 个不是尾节点的每个节点以 O(1) 的时间复杂度操作节点的总次数, N 表示 1 个尾节点以 O(N) 的时间复杂度操作节点的总次数. (2N-1)/N ~ 2, 因此该算法的平均时间复杂度为 O(1) .

    其实就是大部分情况下是 O(1), 仅删除尾部是 O(N).

    public ListNode deleteNode(ListNode head, ListNode tobeDelete) {
    if (head <mark> null || tobeDelete </mark> null) return null;

    // 需要删除的是头结点
    if (head == tobeDelete) {
        return head.next;
    }
    // 要删除的节点不是尾节点, 此时直接将下一个节点的值赋给当前节点, 然后删除下一个节点即可
    if (tobeDelete.next != null) {
        // 将下下个结点的值赋给被删的下一个结点
        tobeDelete.val = tobeDelete.next.val;
        // 覆盖下一个结点即可
        tobeDelete.next = tobeDelete.next.next;

        // 说明删除的结点是尾结点
    } else {
        // 从头开始遍历找到当前结点并删除
        ListNode cur = head;
        while (cur.next != tobeDelete)
            cur = cur.next;
        cur.next = null;
    }
    return head;
}

    # 迭代删除法

    已经排序说明相同的元素都是挨在一起的. 引入 dummy 节点, dummy 节点引导的链表就是删除后的链表. 遍历原链表, 依次判断原链表头结点是否与 dummy 链表尾结点相同, 不相同则添加到 dummy 链表末尾, 相同则移动忽略.

    由于是把节点挪到了新链表, 所以需要注意两链表直接的指针的断开.

    public ListNode deleteDuplicates2(ListNode head) {
    if(head <mark> null || head.next </mark> null) {
        return head;

    // 引入dummy结点
    ListNode dummy = new ListNode(Integer.MIN_VALUE);
    ListNode cur = dummy;

    while (head != null) {
        // 判断当前dummy链表的最后一个结点是否是当前head的值, 如果是就忽略, 不是就将此结点加入dummy链表
        if(cur.val != head.val) {

            head = head.next;
            cur = cur.next;
            // 断开指针
            cur.next = null;
        } else {
            head = head.next;
        }
    }

    return dummy.next;
}

    # 递归法

    指定递归方法需要返回的头结点, 然后判断下一个结点与当前头结点的值的关系, 如果两个值不同, 则当前头结点的下一个节点就是递归下一个结点的结果; 如果值相同, 则需要跳过后面所有值相同的结点, 并继续递归后面剩下的链表.

    public ListNode deleteDuplicates(ListNode head) {
    // Base case
    if (head <mark> null || head.next </mark> null) {
        return head;
    }

    // 指定当前方法需要返回的最终的头结点
    ListNode newHead = head;

    // 如果当前结点与下一个结点值不相同则保留此结点
    if (head.val != head.next.val) {
        // 此结点的下一个结点就是下一个结点的递归返回的链表
        newHead.next = deleteDuplicates(head.next);
    } else {
        // 出现重复结点则只保留当前这个结点 并跳过所有相同的结点
        newHead = newHead.next;
        // 跳过所有相同的结点
        int sameValue = head.val;
        while (head != null && head.val == sameValue) {
            head = head.next;
        }
      
        // 递归调用处理后面的链表
        newHead.next = deleteDuplicates(head);
    }

    return newHead;
}

    # 删除排序链表中的重复元素II(82-中等)🌸🔑💯🔋
    # 题目描述

    给定一个排序链表, 删除所有含有重复数字的节点, 只保留原始链表中 没有重复出现 的数字.

    输入: 1->2->3->3->4->4->5  输出: 1->2->5
输入: 1->1->1->2->3   输出: 2->3

    2

    https://leetcode-cn.com/problems/remove-duplicates-from-sorted-list-ii (opens new window)

    # 递归法

    Base case: 如果 head 为空, 或者只有一个结点时直接返回 head.

    其余情况分两种:

    如果 head.val != head.next.val​, 说明头节点的值不等于下一个节点的值, 所以当前 head 节点须保留; 但并不知道 head.next 节点要不要保留, 因此需要对 head.next 进行递归, 即对剩下的 head.next 作为头节点的链表递归去除值重复的节点.

    如果 head.val == head.next.val​, 说明头节点的值等于下一个节点的值, 所以当前的 head 节点必须删除, 并且 head 之后所有与 head.val 相等的节点也都需要删除; 删除到哪个节点为止? 需要用 cur 指针一直向后遍历寻找到与 head.val 不等的节点. 此时 cur 之前的节点都不保留了, 因此返回 deleteDuplicates(cur).

    代码如下, 这个写法与 删除排序链表中的重复元素(83) 的递归法基本类似:

    public ListNode deleteDuplicates(ListNode head) {
    // Base case
    if (head <mark> null || head.next </mark> null) {
        return head;
    }

    ListNode newHead = null;
  
    // 下面分两种情况构造头结点的下一个结点!!!!
  
    // 如果当前结点与下一个结点值不相同则保留此结点
    if (head.val != head.next.val) {
        newHead = head;
        newHead.next  = deleteDuplicates(head.next);
    } else {
        int sameValue = head.val;
        // 跳过所有相同的结点
        while (head != null && head.val == sameValue) {
            head = head.next;
        }
        // 递归处理后面的链表
        newHead = deleteDuplicates(head);
    }

    return newHead;
}

    # 迭代法

    迭代滑动窗口解决. 头结点可能被删, 引入结点 dummy. 创建 cur 指针指向链表的头部. 接着对 cur 指针迭代, 对比 cur 指针的下一个结点与下下一个结点的值是否相等. 迭代过程中, 如果 cur.next.val == cur.next.next.val​ 说明此时有重复元素, 此时创建一个临时指针 temp, 指向 cur 的下一个节点, 即 temp 指向第一个重复元素所在的位置. 通过 while 循环去重, 去重后 temp 指向的是重复元素中的最后一个位置. 最后 cur.next = temp.next 即可消除重复中间的元素.

    代码如下:

    public ListNode deleteDuplicates(ListNode head) {
    // 可能要删掉头结点-引入dummy结点
    ListNode dummy = new ListNode(Integer.MAX_VALUE);
    dummy.next = head;
    ListNode cur = dummy;

    while (cur.next != null && cur.next.next != null) {
        // 说明出现了重复的结点区间
        if (cur.next.val == cur.next.next.val) {
            // 来一个temp节点指向重复区间的第一个结点
            ListNode temp = cur.next;
            int sameValue = temp.val;
            // 循环去重, 直到temp指向重复区间的最后一个结点
            while (temp.next != null && temp.next.val == sameValue) {
                temp = temp.next;
            }

            // 直接删掉重复的部分
            cur.next = temp.next;

            // 没有重复则移动指针
        } else {
            cur = cur.next;
        }
    }
    return dummy.next;
}

    # 单指针两次遍历法

    删除倒数第 n 个结点, 也就是删除从列表开头数起的第 (len - n + 1)​个结点, 其中 len 是列表的​长度.

    public ListNode removeNthFromEnd(ListNode head, int n) {
    // 引入dummy结点
    ListNode dummy = new ListNode(0);
    dummy.next = head;

    // 遍历找链表长度
    int length  = 0;
    ListNode cur = head;
    while (cur != null) {
        length++;
        cur = cur.next;
    }

    // 计算还需要正向走的步数
    int gap = length - n;
    cur = dummy;
    while (gap > 0 && cur != null) {
        gap--;
        cur = cur.next;
    }

    // 删除结点
    cur.next = cur.next.next;
    return dummy.next;
}

    # 单指针两次遍历法

    第一次遍历先找到链表的长度, 链表长度减去倒数个数也就是需要正向走的结点数, 第二次遍历从头开始遍历正向走的结点数即可.

    public ListNode trainingPlan(ListNode head, int cnt) {
  
    // 获取链表长度
    int length = 0;
    ListNode cur = head;
    while (cur != null) {
        cur = cur.next;
        length++;
    }

    // 列表长度减去减去倒数第K个结点即为正数第几个结点
    int step = length - cnt;
  
    // 从头开始遍历step次即可
    while (step > 0 && head != null) {
        head = head.next;
        step--;
    }

    return head;
}
