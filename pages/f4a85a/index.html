    # 1.LRU实现

    设计并实现最近最少使用(LRU) 缓存的数据结构. 它应该支持以下操作: get 和 put.

    • get(key): 如果键存在于缓存中, 则获取键的值(总是正数), 否则返回 -1.
    • put(key, value): 如果键不存在, 请设置或插⼊值. 当缓存达到其最大容量时, 它应该在插入新项目之前,
      使最近最少使用的项目无效.

    参考力扣题目：LRU缓存(146-中等)🌸💯🔋

    示例:

    LRUCache cache = new LRUCache(2);	// 设置最大容量为2
cache.put(1, 1);
cache.put(2, 2);
cache.get(1); // 返回1
cache.put(3, 3); // 去除key2
cache.get(2); // 返回-1(未找到key2)
cache.get(3); // 返回 3
cache.put(4, 4); // 去除key1
cache.get(1); // 返回-1(未找到key1)
cache.get(3); // 返回3
cache.get(4); // 返回4

    # 双向链表+哈希表实现

    采用这两种数据结构的组合, get 操作就可以在 O(1) 时间复杂度内完成了. 由于 put 操作要删除的节点⼀般是尾部节点, 所以可以用⼀个变量 tail 时刻记录尾部节点的位置, 这样 put 操作也可以在 O(1) 时间内完成了.

    ​​

    这里实现就是需要先来一个结点类.

    class Node {
    int key;
    int value;
    Node prev;
    Node next;

    public Node(int key, int value) {
        this.key = key;
        this.value = value;
    }
}

    实现如下，看看注释即可，这里有一些公共操作，比如把结点放到首部，所以会抽出一些公共方法。

    class LRUCache {
    // 记录最大大小
    private int maxSize;
    // 存储key到结点的映射
    private Map<Integer, Node> map = new HashMap<>();

    // 双向链表的头尾指针
    private Node head = new Node(Integer.MAX_VALUE, Integer.MAX_VALUE);
    private Node tail = new Node(Integer.MAX_VALUE, Integer.MAX_VALUE);

    public LRUCache(int capacity) {
        this.maxSize = capacity;
        // 初始化头尾指针
        head.next = tail;
        tail.prev = head;
    }

    public int get(int key) {
        // 不存在缓存值
        if (!map.containsKey(key)) {
            return -1;
        }
        Node curNode = map.get(key);
        // 移动当前结点为队头结点
        refreshNodeToHead(curNode);
        return curNode.value;
    }

    public void put(int key, int value) {
        // 结点已经存在, 刷新结点值并刷新结点位置到队头
        if (map.containsKey(key)) {
            // 更新结点值
            map.get(key).value = value;
            // 使结点成为队头结点
            refreshNodeToHead(map.get(key));
            return;
        }

        // 新增一个结点并放在队头
        Node newNode = new Node(key, value);
        // 注意map中同样需要记录该结点
        map.put(key, newNode);
        insert2Head(newNode);

        // 如果元素个数超了 直接删除队尾结点
        if (map.size() > maxSize) {
            removeLastNode();
        }
    }

    // 删除队尾结点
    private void removeLastNode() {
        map.remove(tail.prev.key);
        tail.prev = tail.prev.prev;
        tail.prev.next = tail;
    }

    // 插入结点到最前面成为队头结点
    private void insert2Head(Node node) {
        node.next = head.next;
        head.next.prev = node;
        head.next = node;
        node.prev = head;
    }

    // 刷新结点到最前面成为队头结点
    private void refreshNodeToHead(Node node) {
        // 将结点从中间去掉
        node.prev.next = node.next;
        node.next.prev = node.prev;
        // 把结点移动到最前面
        insert2Head(node);
    }
}

    # 1.实现生产者与消费者模型
    # (1)基于wait和notify实现

    这是基于 JDK 的 wait(), notify() 方法实现的。

    可以将生产者和消费者需要的方法写在公共类中. 也就是创建一个资源类.

    static class Storage {
    // 最大容量
    private static final int CAPACITY = 5;
    // 缓存
    private LinkedList<String> linkedList = new LinkedList<>();

    // 生产者方法
    public void produce() throws Exception {
        // 注意这里是对资源linkedList加锁(不能对this加锁)
        synchronized (linkedList) {
            // 容量已满则等待
            while (CAPACITY == linkedList.size()) {
                System.out.println("【生产者】: 仓库已满, 暂时不能执行生产任务!");
                // 生产者线程等待!!!!
                linkedList.wait();
            }
            // 生产一个产品
            linkedList.push("李四");
            System.out.println("【生产者】: 生产了一个产品, 现仓储量为:" + linkedList.size());
            // 通知消费者进行消费
            linkedList.notifyAll();
        }
    }

    // 消费者方法
    public void consume() throws Exception {
        // JDK的notifyAll()解除那些在该对象上调用wait()方法的线程的阻塞状态.
        synchronized (linkedList) {
            // 多线程判断中使用while, 不要使用if否则会出现虚假唤醒问题
            while (linkedList.isEmpty()) {
                System.out.println("【消费者】: 仓库无货, 暂时不能执行消费任务!");
                // 消费者线程等待!!!!
                linkedList.wait();
            }

            // 有产品则消费
            linkedList.pop();
            System.out.println("【消费者】: 消费了一个产品. 现仓储量为:" + linkedList.size());

            // 通知生产者线程进行生产
            linkedList.notifyAll();
        }
    }
}

    # (2)基于await与signalAll实现

    这是基于 JUC 中的 Lock 下面的 await() 和 signal() 方法实现。

    将公共的属性和方法放在 Resouce 类中, 在资源类中使用 Lock 中的 lock() 进行加锁, 控制并发操作. 使用 await() 方法阻塞线程, 使用 signalAll() 唤醒线程.

    资源类:

    static class Resouce {
    private final int MAX_CAPACITY = 3;
    private final int MIN_CAPACITY = 0;
    private int number = 0;
    // 显式锁
    private final Lock lock = new ReentrantLock();
    private final Condition condition = lock.newCondition();

    // 生产方法
    public void product() {
        try {
            // 显式加锁
            lock.lock();
            // 如果生产的数量大于最大值则阻塞
            while (number >= MAX_CAPACITY) {
                System.out.println("【生产者】: 当前仓库满了, 停止生产");
                // 阻塞等待
                condition.await();
            }
            number++;
            System.out.println("【生产者】: 生产了一个产品\t【现仓储量为】:" + number);
            // 生产完毕唤醒所有等待的线程
            condition.signalAll();
        } catch (InterruptedException e) {
            e.printStackTrace();
        } finally {
            // 释放锁
            lock.unlock();
        }
    }

    // 消费方法
    public void consumer() {
        try {
            // 加锁
            lock.lock();
            // 没有产品可以消费则阻塞等待
            while (number <= MIN_CAPACITY) {
                System.out.println("【消费者】: 当前没有产品可以消费");
                // 阻塞等待
                condition.await();
            }
            number--;
            System.out.println("【消费者】: 消费了一个产品\t【现仓储量为】:" + number);
            // 消费完毕唤醒所有等待的线程
            condition.signalAll();
        } catch (InterruptedException e) {
            e.printStackTrace();
        } finally {
            // 释放锁
            lock.unlock();
        }
    }
}

    # (3)基于BlockingQueue实现

    生产者

    static class ProducerThread implements Runnable {
    // 阻塞队列
    private BlockingQueue queue;
    private volatile boolean flag = true;

    // 原子计数器
    private static AtomicInteger count = new AtomicInteger();

    public ProducerThread(BlockingQueue queue) {
        this.queue = queue;
    }

    @Override
    public void run() {
        try {
            System.out.println("生产线程启动...");
            while (flag) {
                System.out.println("正在生产数据....");
                String data = count.incrementAndGet() + "";
              
                // 将数据存入阻塞队列中
                boolean offer = queue.offer(data, 2, TimeUnit.SECONDS);
              
                if (offer) {
                    System.out.println("生产者,存入" + data + "到队列中, 成功.");
                } else {
                    System.out.println("生产者,存入" + data + "到队列中, 失败.");
                }
              
                Thread.sleep(1000);
            }
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            System.out.println("生产者退出线程");
        }
    }

    public void stopThread() {
        this.flag = false;
    }
}

    消费者

    static class ConsumerThread implements Runnable {
    // 阻塞队列
    private BlockingQueue<String> queue;
    private volatile boolean flag = true;

    public ConsumerThread(BlockingQueue<String> queue) {
        this.queue = queue;
    }

    @Override
    public void run() {
        System.out.println("消费线程启动...");
        try {
            while (flag) {
                System.out.println("消费者, 正在从队列中获取数据...");
                // 从阻塞队列获取元素
                String data = queue.poll(2, TimeUnit.SECONDS);
                if (data != null) {
                    System.out.println("消费者拿到队列中的数据data: " + data);
                    Thread.sleep(1000);
                } else {
                    System.out.println("消费者, 超过2秒未获取到数据...");
                    flag = false;
                }
            }
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            System.out.println("消费者退出线程...");
        }
    }
}

    BlockingQueue<String> queue = new LinkedBlockingQueue<>(10);
    // 往生产者和消费者同时阻塞队列

    Thread t1 = new Thread(producerThread1);
    Thread t2 = new Thread(producerThread2);
    Thread c1 = new Thread(consumerThread1);
    t1.start();
    t2.start();
    c1.start();
    // 执行2s后, 生产者不再生产
    Thread.sleep(2 * 1000);
    producerThread1.stopThread();
    producerThread2.stopThread();
}

    # (4)基于普通队列实现

    基于显示锁的生产者与消费者模型. 注意只有一个仓库, 生产者和消费者都往仓库里面放东西.

    static class Storage {

    // 仓库最大存储量
    private static final int MAX_SIZE = 10;
    // 仓库存储的队列
    private Queue<String> queue = new LinkedList<>();
    // 锁
    private final Lock lock = new ReentrantLock();
    // 仓库满的条件变量
    private final Condition fullCondition = lock.newCondition();
    // 仓库空的条件变量
    private final Condition emptyCondition = lock.newCondition();

    // 生产者
    public void produce(String good) {
        try {
            // 获得锁
            lock.lock();
            while (queue.size() >= MAX_SIZE) {
                System.out.println("仓库已满");
                // 仓库满了生产者等待,满条件等待
                fullCondition.await();
            }
            // 生产者添加货物
            queue.offer(good);
            System.out.println("生产者:" + Thread.currentThread().getName() + "生产产品,库存:" + queue.size());
            // 解锁空条件
            emptyCondition.signalAll();
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            lock.unlock();
        }
    }

    // 消费者方法
    public void consume() {
        try {
            // 获得锁
            lock.lock();
            while (queue.size() == 0) {
                System.out.println("仓库为空");
                // 达成空条件,等待
                emptyCondition.await();
            }
            // 从队列弹出元素消费
            queue.poll();
            System.out.println("消费者:" + Thread.currentThread().getName() + "消费产品,库存:" + queue.size());
            // 释放掉满条件
            fullCondition.signalAll();
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            lock.unlock();
        }
    }

    // 消费者线程
    static class Producer implements Runnable {
        private Storage storage;

        public Producer(Storage storage) {
            this.storage = storage;
        }

        @Override
        public void run() {
            while (true) {
                try {
                    Thread.sleep(1000);
                    storage.produce("123");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }
    }

    // 消费者线程
    static class Consumer implements Runnable {
        private Storage storage;

        public Consumer(Storage storage) {
            this.storage = storage;
        }

        @Override
        public void run() {
            while (true) {
                try {
                    Thread.sleep(3000);
                    storage.consume();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }
    }

    public static void main(String[] args) {
        // 注意这里只有一个仓库
        Storage storage = new Storage();

        // 生产者和消费者都往仓库里面放东西,所以传入同一个仓库实例
        Thread p1 = new Thread(new Producer(storage));
        Thread p2 = new Thread(new Producer(storage));
        Thread p3 = new Thread(new Producer(storage));

        Thread c1 = new Thread(new Consumer(storage));
        Thread c2 = new Thread(new Consumer(storage));
        Thread c3 = new Thread(new Consumer(storage));

        p1.start();
        p2.start();
        p3.start();
        c1.start();
        c2.start();
        c3.start();
    }
}

    # 2.两个线程打印奇数偶数
    # (1)synchronized加锁

    这里基于 synchronized 实现.

    // 计数器
private static int count = 0;
// 设置一个对象锁
private final static Object lock = new Object();

static class TurningRunner implements Runnable {
    @Override
    public void run() {
        while (count <= 10) {
            // 获取锁
            synchronized (lock) {
                // 拿到锁就打印
                System.out.println(count);
                count++;
                // 唤醒另一个线程
                lock.notifyAll();
                try {
                    // 如果任务还没有结束, 则让出当前的锁并休眠
                    if (count <= 10) {
                        lock.wait();
                    }
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }
    }
}

public static void main(String[] args) throws InterruptedException {
    // 初始化2个线程
    Thread a = new Thread(new TurningRunner(), "偶数");
    Thread b = new Thread(new TurningRunner(), "奇数");
    a.start();
    b.start();
}

    # 3.三个线程交替打印ABC

    使用 ReentrantLock 与 Condition 配合. 初始化三个线程，A 打印完唤醒 B，B 打印完唤醒 C，C 打印完唤醒 A。同时使用一个 CountDownLatch 来统计打印了多少轮(如果强制要求前面必须打完多少轮就应该使用这个)。

    static class Solution1 {
    private static int count = 0;
    // 定义全局锁
    private static ReentrantLock lock = new ReentrantLock();
    // 通过一个锁生成三个condition
    private static Condition conditionA = lock.newCondition();
    private static Condition conditionB = lock.newCondition();
    private static Condition conditionC = lock.newCondition();

    // 栅栏,用于控制打印多少轮(重复多少次)
    private static CountDownLatch countDownLatch = new CountDownLatch(10);
  
    static Runnable runnable1 = new Runnable() {
        @Override
        public void run() {
            for (int i = 0; i < 10; i++) {
                try {
                    // 加锁
                    lock.lock();
                    // 如果不是A则等待
                    if (count % 3 != 0) {
                        conditionA.await();
                    }
                    System.out.print("A");
                    count++;
                    // 打印完A则唤醒B
                    conditionB.signal();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    lock.unlock();
                }
            }
        }
    };

    static Runnable runnable2 = new Runnable() {
        @Override
        public void run() {
            for (int i = 0; i < 10; i++) {
                try {
                    // 加锁
                    lock.lock();
                    // 如果不是B则等待
                    if (count % 3 != 1) {
                        conditionB.await();
                    }
                    System.out.print("B");
                    count++;
                    // 打印完B则唤醒C
                    conditionC.signal();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    lock.unlock();
                }
            }
        }
    };

    static Runnable runnable3 = new Runnable() {
        @Override
        public void run() {
            for (int i = 0; i < 10; i++) {
                try {
                    // 加锁
                    lock.lock();
                    if (count % 3 != 2) {
                        conditionC.await();
                    }
                    System.out.print("C");
                    count++;
                    // 打印完C则唤醒A
                    conditionA.signal();
                    // 完成一轮则减一
                    countDownLatch.countDown();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    lock.unlock();
                }
            }
        }
    };

  
    // 测试代码
    public static void main(String[] args) {
        // 创建三个线程
        Thread thread1 = new Thread(runnable1);
        Thread thread2 = new Thread(runnable2);
        Thread thread3 = new Thread(runnable3);
        thread1.setPriority(Thread.MAX_PRIORITY);
        thread1.start();
        thread2.start();
        thread3.start();
        try {
            // 栅栏在此处等待完成
            countDownLatch.await();
            System.out.println("\n完成");
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }

}

    # 1.覆写equals方法

    写出完美 equals 方法的建议(源自 JavaCore):

    • 检测 this 与 otherObject 是否引用同一对象(地址相同).
    • 检测 otherObject 是否为 null(null 就肯定不是了).
    • 比较 this 与 otherObject 是否属于同一个类(类型不同也不是).
    • 将 otherObject 类型转换为相应的类类型变量.
    • 现在开始对所有需要比较的域依次进行比较. 使用 == 比较基本类型域, 使用 equals 比较对象域. 这里就需要看对不同类自己的情况写.
    // 以下是Employee类的equals方法
public boolean equals(Object otherObject){
     // 检测this与otherObject是否引用同一对象
    if(this <mark> otherObject) return true;   
    // 检测otherObject是否为null
    if(otherObject </mark> null) return false;   
    // 比较this与otherObject是否属于同一种类型
    if(this.getClass() != otherObject.geClass()) return false;
    // 将otherObject转换为相应的类类型变量: 类型转换
    Employee other = (Employee) otherObject;  
    // 现在开始对所有需要比较的域进行比较. 使用 <mark> 比较基本类型域, 使用equals比较对象域. 
    return Object.equals(name, other.name) && salary </mark> other.salary && Objects.equals(hireDay, other.hireDay);
}
