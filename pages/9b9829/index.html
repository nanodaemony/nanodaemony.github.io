    # 快慢双指针交互法

    使用快慢双指针，每次快指针都会移动一步，如果快指针遇到非 0 的值，则与慢指针的位置交互，并移动慢指针，这样快指针到末尾即处理完成。

    // [0,1,0,3,12]
public void moveZeroes(int[] nums) {
    if (nums <mark> null || nums.length </mark> 0) {
        return;
    }

    // 快慢双指针
    int fast = 0;
    int slow = 0;
    while (fast < nums.length) {
        // 如果快指针不为0则与慢指针交互
        if (nums[fast] != 0) {
            swap(nums, slow, fast);
            slow++;
        }
        // 每次都会移动一下快指针
        fast++;
    }
}

private void swap(int[] nums, int i, int j) {
    int temp = nums[i];
    nums[i] = nums[j];
    nums[j] = temp;
}

    # 快慢双指针法

    使用快慢双指针进行遍历, 快指针遇到等于 val 的元素直接则直接跳过, 否则将快指针的元素赋值给慢指针, 并移动慢指针.

    整个过程如下:

    本题代码如下:

    public int removeElement(int[] nums, int deleteVal) {
    if (nums == null) {
        return 0;
    }

    // 由于第一个元素可能就是删除元素, 因此快指针需要从0开始
    int slow = 0, fast = 0;
    while (fast < nums.length) {
        if (nums[fast] != deleteVal) {
            nums[slow] = nums[fast];
            slow++;
        }
        // 快指针一直向前走
        fast++;
    }
    return slow + 1;
}

    # 快慢双指针法

    数组是有序的, 那么重复的元素一定会相邻. 要求删除重复元素, 实际上就是将不重复的元素移到数组的左侧.

    使用快慢双指针的流程如下:

    比较 slow 和 fast 位置的元素是否相等. 如果相等, fast 后移 1 位, 相当于跳过重复元素. 如果不相等, 将 fast 位置的元素复制到 slow + 1 位置上(先将 slow + 1), slow 后移一位, fast 后移 1 位. 重复上述过程, 直到 fast 到达数组末尾. 返回 slow + 1, 即为新数组长度.

    动图示例:

    ​​

    代码如下:

    public int removeDuplicates(int[] nums) {
    if (nums <mark> null || nums.length </mark> 0) return 0;

    // 快慢双指针
    int slow = 0, fast = 1;

    // 1 1 2 2 3 3 3 4 4 5 6 7
    while (fast < nums.length) {
        // 如果快慢指针指向的值相等, 快指针继续向前走
        if (nums[fast] == nums[slow]) {
            fast++;
            // 如果快慢指针指向的值不相等, 慢指针向前走一步, 并将快指针的值赋给慢指针
        } else {
            // 慢指针先向前移动一次, 不然会覆盖掉slow之前指向的位置的元素
            slow++;
            nums[slow] = nums[fast];
            fast++;
        }
    }
    // slow是索引, 需要+1才是长度
    return slow + 1;
}

    # 快慢双指针法

    本题思路与 删除排序数组中的重复项(26-简单) 类似, 也是使用双指针, 只是需要每个元素保留两次.

    前两个元素是一定不会动的, 所以快慢指针起始指向第二, 三个元素. 之后不断移动快指针, 如果快指针的元素值与慢指针的前两个元素都相同, 则说明慢指针的元素已经出现了两次, 此时直接移动快指针即可. 如果不相同, 则把快指针位置的元素复制到慢指针处, 也就相当于把重复的元素抹掉了.

    ​​

    最后数组后面剩余的元素(比如最后以 2333333333 结尾, 这里只需要保留 "233")是不用管的, 因为方法只需要返回慢指针的长度即可.

    代码如下:

    public int removeDuplicates(int[] nums) {
    if (nums == null) {
        return 0;
    }

    if (nums.length <= 2) {
        return nums.length;
    }

    // 第一个元素一定不会动, 快慢指针均指向第二, 三个元素
    int slow = 1;
    int fast = 2;

    while (fast < nums.length) {
        // 如果快指针的元素与慢指针及其之前的元素都相同, 此时说明出现次数超过2了, 则快指针继续向前移动
        if (nums[fast] <mark> nums[slow] && nums[fast] </mark> nums[slow - 1]) {
            fast++;
            // 否则将快指针处的元素复制到慢指针处
        } else {
            // 慢指针先向前移动一次, 不然会覆盖掉slow之前指向的位置的元素
            slow++;
            // 将快指针的值赋给慢指针
            nums[slow] = nums[fast];
            fast++;
        }
    }

    // slow是索引, 需要+1才是长度
    return slow + 1;
}

    # 逆向双指针法

    本题不能对两个数组合并后排序, 这样时间复杂度比较高.

    直接使用两个指针, 分别指向两个数组有效位置的末尾, 然后逐一比较元素大小, 较大的元素填充到 nums1 的最后面即可, 也就是从右往左进行逆向双指针遍历. 整个过程直到其中一个数组合并完为止, 最后如果其中一个还没合并完全, 则再单独合并这个数组即可.

    代码如下:

    public void merge(int[] nums1, int m, int[] nums2, int n) {
    if (nums1 <mark> null || nums2 </mark> null) {
        return;
    }

    // 两个指针指向两个数组的末尾
    int idx1 = m - 1;
    int idx2 = n - 1;

    // 结果指针指向nums1的末尾, 从后往前填充结果
    int resIdx = nums1.length - 1;

    // 注意: 从后往前合并, 退出循环的条件是其中一个数组已经合并完
    while (idx1 >= 0 && idx2 >= 0) {
        if (nums1[idx1] >= nums2[idx2]) {
            nums1[resIdx--] = nums1[idx1--];
        } else {
            nums1[resIdx--] = nums2[idx2--];
        }
        // 结果指针都需要移动
        resIdx--;
    }

    // 如果两个数组还有剩余, 则直接填充到nums1中
    while (idx1 >= 0) {
        nums1[resIdx--] = nums1[idx1--];
    }
    while (idx2 >= 0) {
        nums1[resIdx--] = nums2[idx2--];
    }
}

    # 排序+双指针法

    这个题排序很关键! 直接用系统的排序算法即可, Arrays.sort() . 先将数组排序, 然后遍历数组, 作为第一个数, 并在这个数的右半部分区间使用双指针进行查找, 需要注意去重.

    过程图解:

    具体流程如下:

    • 首先对数组进行排序, 排序后固定一个数 nums[i] , 再使用左右指针指向 nums[i] 后面的两端, 数字分别为 nums[left] 和 nums[right], 计算三个数的和 sum 判断是否满足为 0, 满足则添加进结果集.
    • 如果 nums[i] 大于 0, 则三数之和必然无法等于 0, 结束循环.
    • 如果 nums[i] == nums[i - 1], 则说明该数字重复, 会导致结果重复, 所以应该跳过.
    • 当 sum 0 时, nums[left] nums[left + 1] 则会导致结果重复, 应该跳过, left++.
    • 当 sum 0 时, nums[right] nums[right − 1] 则会导致结果重复, 应该跳过, right--.
    • 时间复杂度: O(n^2), n 为数组长度.
    public static List<List<Integer>> threeSum(int[] nums) {
    if (Objects.isNull(nums) || nums.length < 3) {
        return Collections.emptyList();
    }

    List<List<Integer>> resList = new ArrayList<>();

    // 对原始数组排序
    Arrays.sort(nums);

    // 遍历整个数组
    for (int i = 0; i < nums.length; i++) {

        // 如果当前数字大于0, 因为已经排序好, 所以后面不可能有三个数加和等于 0, 直接终止循环
        if (nums[i] > 0) {
            break;
        }

        // 当前的值等于前一个元素的值, 那么得到的结果将会和前一次相同, 这里进行去重
        if (i > 0 && nums[i] == nums[i - 1]) {
            continue;
        }

        // 当前元素右边部分的左右指针
        int left = i + 1;
        int right = nums.length - 1;

        // 终止条件是左右指针相遇
        while (left < right) {
            // 三数求和
            int sum = nums[i] + nums[left] + nums[right];
            // 和为0则加入结果集
            if (sum <mark> 0) {
                resList.add(Arrays.asList(nums[i], nums[left], nums[right]));
                // 去重
                while (left < right && nums[left] </mark> nums[left + 1]) {
                    left++;
                }
                // 去重
                while (left < right && nums[right] == nums[right - 1]) {
                    right--;
                }
                // 同时移动两个指针, 因为已经找到了一个解, 继续找下一个解
                left++;
                right--;
                // 和小于0则左指针右移
            } else if (sum < 0) {
                left++;
                // 和大于0则右指针左移
            } else {
                right--;
            }
        }
    }

    return resList;
}

    # 排序+双指针法

    四数之和, 和 三数之和(15-中等) 是一个思路, 都是使用双指针法, 基本解法就是在三数之和的基础上再套一层 for 循环, 并且进行去重.

    但有一些细节需要注意, 例如: 不要判断 nums[k] > target​ 就返回了, 三数之和可以通过 nums[i] > 0​ 就返回了, 因为 0 已经是确定的数了, 四数之和这道题目 target 是任意值. 比如: 数组是 [-4, -3, -2, -1]​, target​ 是 -10​, 不能因为 -4 > -10​ 而跳过. 但是依旧可以去做剪枝, 逻辑变成 nums[i] > target && (nums[i] >= 0 || target >= 0)​ 就可以了.

    public List<List<Integer>> fourSum(int[] nums, int target) {

        if (i > 0 && nums[i] == nums[i - 1]) {

            if (sum > target) {
                right--;
            } else if (sum < target) {
                left++;
            }
        }
    }
    return ans;
}

    # 双指针遍历法

    将这个题目给的特殊的数组当作一个链表来看，数组的下标就是指向元素的指针，把数组的元素也看作指针。如 0 是指针，指向 nums[0]，而 nums[0] 也是指针，指向 nums[nums[0]].

    假设有这样一个样例：[1,2,3,4,5,6,7,8,9,5]。如果按照上面的循环下去就会得到这样一个链表的路径：1 2 3 4 5 [6 7 8 9] [6 7 8 9] [6 7 8 9]... 这样就有了一个环，也就是 6 7 8 9。point 会一直在环中循环的前进。所以位置 5 就是进入环的元素，这样这个题目就变成了求带环链表的第一个入口结点的问题了！

    解法就类似了，设置两个一快(fast)一慢(slow)两个指针，一个每次走两步，一个每次走一步，直到两者相遇。然后慢指针重置，再同时走，后面相遇的地方就是入环的第一个结点，也就是重复的数字了。

    public int findDuplicate(int[] nums) {
    int slow = 0, fast = 0;
    // 快慢指针, 快指针走两步, 慢指针走一步
    do {
        slow = nums[slow];
        fast = nums[nums[fast]];
    } while (slow != fast);

    // 慢指针回到起点, 然后同时走, 相遇时即为环的入口
    slow = 0;

    while (slow != fast) {
        slow = nums[slow];
        fast = nums[fast];
    }

    return slow;
}

    # 左右双指针移动短板法

    注意: 每次移动的是较短的那边的柱子.

    ​​

    在每个状态下, 无论长板或短板向中间收窄一格, 都会导致水槽底边宽度 −1 变短:

    • 若向内 移动短板, 水槽的短板 min(h[i],h[j]) 可能变大, 因此下个水槽的面积 可能增大.
    • 若向内 移动长板, 水槽的短板 min(h[i],h[j]) 不变或变小, 因此下个水槽的面积 一定变小.

    因此, 初始化双指针分列水槽左右两端, 循环每轮将短板向内移动一格, 并更新面积最大值, 直到两指针相遇时跳出; 即可获得最大面积.

    public int maxArea(int[] nums) {
    if (nums == null || nums.length < 2) return 0;
  
    // 左右双指针
    int left = 0, right = nums.length - 1;
    // 暂存最大面积
    int maxArea = Integer.MIN_VALUE;
  
    // 左右指针未相遇, 这里不用等于是因为等于时面积为0
    while (left < right) {
        // 计算当前面积: 宽度为右指针减左指针, 高度为左右指针中的最小值
        maxArea = Math.max(maxArea, (right - left) * Math.min(nums[left], nums[right]));
  
        // 重要: 每次移动高度较小的指针
        if (nums[left] < nums[right]) {
            left++;
        } else {
            right--;
        }
    }
    return maxArea;
}

        } else {
            res[resIndex] = nums[right] * nums[right];
            right--;
        }
        resIndex--;
    }
    return res;

    public int[] exchange(int[] nums) {

    int left = 0;
    int right = nums.length - 1;
  
    // 如果两个指针相遇, 则不需要处理 因此退出条件是左指针小于右指针
    while (left < right) {

    nums[left] = nums[right];

    char[] chars = s.toCharArray();

    // 每次i增加2k, 并翻转2k这个区间的字符
    for (int i = 0; i < chars.length; i += 2 * k) {
        // 寻找需要反转的区间的左右范围

        while (left < right) {
            swap(chars, left++, right--);

    return new String(chars);

    public boolean validPalindrome(String str) {
    if (str == null) {
        return false;
    }
    if (str.length() <= 1) {
        return true;
    }

    // 左右双指针
    int left = 0;
    int right = str.length() - 1;

    while (left <= right) {
        // 如果两个字符相同则逼近指针
        if (str.charAt(left) == str.charAt(right)) {
            left++;
            right--;

            return isPalindrome(str, left + 1, right) || isPalindrome(str, left, right - 1);

    }
    return true;

        if (str.charAt(i++) != str.charAt(j--)) {
            return false;
        }

    return true;

public int trap(int[] heights) {
    if (heights <mark> null || heights.length </mark> 0) {
        return 0;

    // 获取最大高度
    int maxHeight = getMaxHeight(heights);

    int res = 0;
    for (int i = 1; i <= maxHeight; i++) {

private int getLevelCount(int[] heights, int level) {
    int leftBorder = 0;
    int rightBorder = heights.length - 1;

    // 找到左右边界
    while (leftBorder <= heights.length && heights[leftBorder] < level) {
        leftBorder++;
    }

    while (rightBorder >= 0 && heights[rightBorder] < level) {
        rightBorder--;
    }

    if (leftBorder >= rightBorder) {
        return 0;

    // 计算这一层的雨水量
    int count = 0;
    for (int i = leftBorder; i <= rightBorder; i++) {

    return count;

    for (int i = 0; i < heights.length; i++) {
        max = Math.max(max, heights[i]);

            res = res + Math.max(0, rightMax - height[right]);
            right--;
        }
    }

    return res;

    # 滑动窗口

    暴力法是一个 for 循环滑动窗口的起始位置, 一个 for 循环为滑动窗口的终止位置, 用两个 for 循环实现不断搜索区间的过程.

    那么滑动窗口如何用一个 for 循环来完成这个操作呢. 首先要思考如果用一个 for 循环, 那么应该表示滑动窗口的起始位置, 还是终止位置. 如果只用一个 for 循环来表示滑动窗口的起始位置, 那么如何遍历剩下的终止位置? 此时难免再次陷入暴力解法的怪圈. 所以只用一个 for 循环, 那么这个循环的索引, 一定是表示滑动窗口的​终止位置.

    那么问题来了, 滑动窗口的起始位置如何移动呢? 举个例子, s=7, 数组是 2, 3, 1, 2, 4, 3, 来看一下查找的过程:

    最后找到 4, 3 是最短距离.

    可以发现滑动窗口也可以理解为双指针法的一种! 只不过这种解法更像是一个窗口的移动, 所以叫做滑动窗口更适合一些. 在本题中实现滑动窗口, 主要确定如下三点:

    • 窗口内是什么?
    • 如何移动窗口的起始位置?
    • 如何移动窗口的结束位置?

    窗口就是满足其和 ≥ s 的长度最小的连续子数组.

    窗口的起始位置如何移动: 如果当前窗口的值大于等于 s 了, 窗口就要向前移动了(也就是该缩小了) .

    窗口的结束位置如何移动: 窗口的结束位置就是遍历数组的指针, 也就是 for 循环里的索引.

    可以发现滑动窗口的精妙之处在于根据当前子序列和大小的情况, 不断调节子序列的起始位置. 从而将 O(n^2)暴力解法降为 O(n).

    代码如下:

    public int minSubArrayLen(int s, int[] nums) {
    // left和right表示窗口的左右边界
    int left = 0;
    int right = 0;
    // 记录窗口内的和
    int windowSum = 0;
    // 记录最小窗口大小
    int minWindow = Integer.MAX_VALUE;
    // 不断移动窗口右边界
    while (right < nums.length) {
        // 计算窗口内的和
        windowSum += nums[right];
        // 如果当前窗口内的元素和大于等于s则持续移动左边界, 减小窗口大小
        while (windowSum >= s) {
            // 不断更新最小值
            minWindow = Math.min(minWindow, right - left + 1);
            windowSum -= nums[left];
            left++;
        }
        right++;
    }
    return minWindow == Integer.MAX_VALUE ? 0 : minWindow;
}

    # 滑动窗口法

    设连续正整数序列的左边界 i 和右边界 j, 则可构建滑动窗口从左向右滑动. 循环中持续判断滑动窗口内元素和与目标值 target 的大小关系, 若相等则记录结果, 若大于 target 则移动左边界 i 以减小窗口内的元素和, 若小于 target 则移动右边界 j 以增大窗口内的元素和.

    参考 (opens new window):

    代码如下:

    public int[][] fileCombination(int target) {
    int left = 1;  // 滑动窗口的左边界(整数从1开始)
    int right = 1; // 滑动窗口的右边界
    int windowSum = 0; // 滑动窗口中数字的和
    List<int[]> resList = new ArrayList<>();

    while (right < target) {
        // 计算窗口内的和
        windowSum += right;
        // 如果当前窗口内的元素和大于等于target则持续移动左边界, 减小窗口大小
        while (windowSum >= target) {
            // 记录一次结果
            if (windowSum == target) {
                int[] res = new int[right - left + 1];
                for (int i = left; i <= right; i++) {
                    res[i - left] = i;
                }
                resList.add(res);
            }

            // 不断更新窗口左边界
            windowSum -= left;
            left++;
        }

        // 移动窗口右边界
        right++;
    }

    return resList.toArray(new int[resList.size()][]);
}

    # 滑动窗口法

    维护一个滑动窗口, 窗口内的都是没有重复的字符, 去尽可能的扩大窗口的大小, 窗口不停的向右滑动.

    • 如果当前遍历到的字符从未出现过, 那么直接扩大右边界.
    • 如果当前遍历到的字符出现过, 则缩小窗口(左边索引向右移动), 然后继续观察当前遍历到的字符.

    代码如下：

    public int lengthOfLongestSubstring(String s) {
    // 滑动窗口左右指针
    int left = 0, right = 0;
    // 记录窗口中字符出现的次数
    int[] windowCnt = new int[128];
    // 记录最长长度
    int maxUniqueLen = 0;

    while (right < s.length()) {
        // 右边界字符
        char rightChar = s.charAt(right);
        windowCnt[rightChar]++;
        // 移动窗口右边界
        right++;
        // 如果滑动窗口中出现重复字符, 则开始移动左边界缩小窗口
        while (windowCnt[rightChar] > 1) {
            char leftChar = s.charAt(left);
            windowCnt[leftChar]--;
            left++;
        }
        // 更新结果值
        maxUniqueLen = Math.max(maxUniqueLen, right - left);
    }
    return maxUniqueLen;
}

    Map<Character, Integer> dic = new HashMap<>();

        dic.put(s.charAt(j), j);
        // 滚动更新最大的结果
        res = Math.max(res, j - i);
    }
    return res;

    # 滑动窗口法

    因为字符串中的字符全是小写字母，可以用长度为 26 的数组记录字母出现的次数。

    首先统计字符串 p 中每个字符出现的次数，然后统计 s 的前 p.length() 个子串中每个字符出现的次数。滑动窗口从 p.length() 的长度开始，每次移动一个字符，移动后需要根据情况，减少左边界移除的字符个数，增加右边界新增的字符的个数。每次移动后都进行个数的对比，匹配则找到一个异位词的索引。

    注意寻找的是异位词，所以这个滑动窗口是一个固定大小的窗口，其大小为 p.length() 。

    // s = "cbaebabacd", p = "abc"
public List<Integer> findAnagrams(String s, String p) {
    int n = s.length(), m = p.length();
    List<Integer> res = new ArrayList<>();
    if (n < m) {
        return res;
    }

    // 记录p中每个字符出现的次数
    int[] pCnt = new int[26];
    // 记录s中每个字符出现的次数(注意这里只统计了前m个字符)
    int[] sCnt = new int[26];

    for (int i = 0; i < m; i++) {
        pCnt[p.charAt(i) - 'a']++;
        sCnt[s.charAt(i) - 'a']++;
    }

    // 如果初始时两个数组相等则加入结果中
    if (Arrays.equals(sCnt, pCnt)) {
        res.add(0);
    }

    // 滑动窗口, 从m开始遍历即可, 对比的是 [m - p.length(), m] 这个滑动区间的字符个数
    for (int i = m; i < n; i++) {
        // 由于是滑动窗口, 需要将窗口左边界的字符个数减少一个, 右边界的字符个数增加一个
        sCnt[s.charAt(i - m) - 'a']--;
        sCnt[s.charAt(i) - 'a']++;

        // 对比两个数组是否相等, 相等则说明是异位词
        if (Arrays.equals(sCnt, pCnt)) {
            res.add(i - m + 1);
        }
    }

    return res;
}

    # 滑动窗口法

    枚举 s 子串的右端点 right（子串最后一个字母的下标），如果子串涵盖 t，就不断移动左端点 left 直到不涵盖为止。在移动过程中更新最短子串的左右端点。

    具体来说：

    • 初始化 ansLeft=−1, ansRight=m，用来记录最短子串的左右端点，其中 m 是 s 的长度。

    • 用一个哈希表（或者数组）cntT 统计 t 中每个字母的出现次数。

    • 初始化 left = 0，以及一个空哈希表（或者数组）cntS，用来统计 s 子串中每个字母的出现次数。

    • 遍历 s，设当前枚举的子串右端点为 right，把 s[right] 的出现次数加一。

    • 遍历 cntS 中的每个字母及其出现次数，如果出现次数都大于等于 cntT 中的字母出现次数：

      • 如果 right − left < ansRight − ansLeft，说明找到了更短的子串，更新 ansLeft = left, ansRight = right。
      • 把 s[left] 的出现次数减一。
      • 左端点右移，即 left 加一。
      • 重复上述三步，直到 cntS 有字母的出现次数小于 cntT 中该字母的出现次数为止。

    • 最后，如果 ansLeft < 0，说明没有找到符合要求的子串，返回空字符串，否则返回下标 ansLeft 到下标 ansRight 之间的子串。

    public String minWindow(String s, String t) {
    char[] chars = s.toCharArray();
    int length = chars.length;
    int ansLeft = -1;
    int ansRight = length;
    // s 子串字母的出现次数
    int[] cntS = new int[128];
    // t 中字母的出现次数
    int[] cntT = new int[128]; 
    for (char c : t.toCharArray()) {
        cntT[c]++;
    }

    int left = 0;
    // 移动子串右端点
    for (int right = 0; right < length; right++) {
        // 右端点字母移入子串
        cntS[chars[right]]++;
        // 校验涵盖, 如果覆盖看看是不是最短的情况 是则缓存起来
        while (checkIsCovered(cntS, cntT)) {
            // 找到更短的子串
            if (right - left < ansRight - ansLeft) {
                // 记录此时的左右端点
                ansLeft = left; 
                ansRight = right;
            }
            // 左端点字母移出子串
            cntS[chars[left]]--; 
            left++;
        }
    }
  
    return ansLeft < 0 ? "" : s.substring(ansLeft, ansRight + 1);
}

private boolean checkIsCovered(int[] cntS, int[] cntT) {
    for (int i = 'A'; i <= 'Z'; i++) {
        if (cntS[i] < cntT[i]) {
            return false;
        }
    }
    for (int i = 'a'; i <= 'z'; i++) {
        if (cntS[i] < cntT[i]) {
            return false;
        }
    }
    return true;
}
