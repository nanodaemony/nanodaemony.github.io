## 部署脚本

停止与删除镜像：

```shell
sudo docker stop littlegrid-mysql littlegrid-redis littlegrid-backend littlegrid-frontend 2>/dev/null
sudo docker rm littlegrid-mysql littlegrid-redis littlegrid-backend littlegrid-frontend 2>/dev/null
```

在部署服务器执行：

```
sudo ./deploy.sh update
```

首次构建仍会下载依赖，但后续构建会直接使用缓存，大幅加速。如果遇到依赖问题需要重置缓存：

```bash
./deploy.sh clean-cache
```

查看服务端日志：

```bash
sudo ./logs.sh app
```

## 服务端日志查看

```bash
已创建 logs.sh 日志查看脚本。
使用方式
在服务器上部署后，运行：
# 添加执行权限
chmod +x logs.sh

# 查看帮助
./logs.sh help
命令列表
┌────────────────────────┬────────────────────────┐
│          命令          │          说明           │
├────────────────────────┼────────────────────────┤
│ ./logs.sh app          │ 实时查看应用日志          │
├────────────────────────┼────────────────────────┤
│ ./logs.sh error        │ 实时查看错误日志         │
├────────────────────────┼────────────────────────┤
│ ./logs.sh all          │ 同时查看应用和错误日志     │
├────────────────────────┼────────────────────────┤
│ ./logs.sh trace abc123 │ 按 TraceId 查找日志      │
├────────────────────────┼────────────────────────┤
│ ./logs.sh search 登录  │ 搜索关键词               │
├────────────────────────┼────────────────────────┤
│ ./logs.sh today        │ 查看今天的请求日志        │
├────────────────────────┼────────────────────────┤
│ ./logs.sh docker       │ 查看 Docker 容器日志     │
├────────────────────────┼────────────────────────┤
│ ./logs.sh ls           │ 列出日志文件及大小        │
└────────────────────────┴────────────────────────┘

示例

# 实时监控日志
./logs.sh app

# 排查特定请求
./logs.sh trace a1b2c3d4

# 搜索登录相关日志
./logs.sh search 登录

# 查看今天所有请求
./logs.sh today
```
