## 元素查找题目

### 搜索二维矩阵(74-中等) 🔑💯

给定一个二维数组, 具有如下特征：

- 每行中的整数从左到右按非严格递增顺序排列。
- 每行的第一个整数大于前一行的最后一个整数。

给定一个数，判断这个数是否在该二维数组中。

<img src="/zimg/yuque-1777868290106-1.png" style="width: 470px">

```
输入: matrix = [[1,3,5,7],[10,11,16,20],[23,30,34,60]], target = 30
输出: true
```

[https://leetcode.cn/problems/search-a-2d-matrix/description/](https://leetcode.cn/problems/search-a-2d-matrix/description/)

**解题思路**

要求时间复杂度 `O(M + N)`，空间复杂度 `O(1)`。其中 M 为行数，N 为 列数。由于矩阵是有排序特点的，所以不用按照普通的遍历方法来查找，而是要学会利用这个特点。

**从右上角开始往下查找，找到第一个大于目标值的行，说明目标值就可能在这行，然后再从右往左逐一查找即可**。

![](/zimg/yuque-1777868290117-2.png)

代码如下：

```
public boolean searchMatrix(int[][] matrix, int target) {
    if (matrix == null || matrix.length == 0 || matrix[0].length == 0) {
        return false;
    }
    // 行数
    int cow = matrix.length;
    // 列数
    int col = matrix[0].length;

    // 从右上角开始往下找
    for (int i = 0; i < cow; i++) {
        if (matrix[i][col - 1] < target) {
            continue;
        }
        // 找到一个比目标值大的行则开始从右往左找
        for (int j = col - 1; j >= 0; j--) {
            if (matrix[i][j] == target) {
                return true;
            }
        }
    }

    return false;
}
```

这个题的另一个解法如下，**写法与下一题的 搜索二维矩阵 II 一模一样**！

```
public boolean searchMatrix(int[][] matrix, int target) {
    if (matrix == null) {
        return false;
    }

    // 起始点为右上角的元素
    int i = 0;
    int j = matrix[0].length - 1;

    while (true) {
        // 越界则未找到
        if (i >= matrix.length || j < 0) {
            break;
        }
        // 找到元素
        if (matrix[i][j] == target) {
            return true;
            // 如果当前元素比目标小 说明当前格子这一层都比目标小, 直接多加一层
        } else if (matrix[i][j] < target) {
            i++;
            // 如果当前元素比目标大 说明当前格子这一列都比目标大, 直接减少一列
        } else {
            j--;
        }
    }

    return false;
}
```

**二分法**

注意：由于本题的矩阵在全局**打平之后是一个有序的数组**，所以**可以用\*\***二分法\***\*来进行快速处理**！

```
public boolean searchMatrix(int[][] matrix, int target) {
    int m = matrix.length;
    int n = matrix[0].length;
    int left = 0;
    int right = m * n - 1;

    while (left <= right) {
        // 找到中点
        int mid = left + (right - left) / 2;
        int midValue = matrix[mid / n][mid % n];

        if (midValue == target) {
            return true;
        } else if (midValue < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    return false;
}
```

编写一个高效的算法来搜索 `m x n` 矩阵 matrix 中的一个目标值 target。该矩阵具有以下特性：

### 数组中大于一半的元素(169-简单) 🔑💯

给定一个大小为 n 的数组，找到其中的多数元素。多数元素是指在数组中出现次数大于 `n / 2` 的元素。可以假设数组是非空的，并且给定的数组总是存在多数元素。

```
输入: [3,2,3] 输出: 3
输入: [2,2,1,1,1,2,2] 输出: 2
```

[https://leetcode-cn.com/problems/majority-element/](https://leetcode-cn.com/problems/majority-element/)

**排序法**

排序：排序之后出现超过数组长度一半的元素一定在中间位置。当然这样写就不能满足题意了，因为时间复杂度超标了。

```
public int majorityElement(int[] nums) {
    Arrays.sort(nums);
    return nums[nums.length / 2];
}
```

**遍历计数法**

摩尔投票法：时间复杂度为 `O(N)`。使用 cnt 来统计一个元素出现的次数，当遍历到的元素和统计元素相等时，令 `cnt++`，否则令 `cnt--`。如果前面遍历了 i 个元素，且 `cnt == 0`，说明前 i 个元素没有 majority，或者有 majority，但是出现的次数少于 `i / 2`，因为如果多于 `i / 2` 的话 cnt 就一定不会为 0。这样就可以更换新的元素来统计了。此时剩下的 `n - i` 个元素中，majority 的数目依然多于 `(n - i) / 2`，因此继续查找就能找出 majority。

<img src="/zimg/yuque-1777868290216-5.png" style="width: 623px">

动图如下：

<img src="/zimg/yuque-1777868290237-6.gif" style="width: 616px">

代码如下：

```
public int majorityElement(int[] nums) {
    int majority = 0, cnt = 0;

    // 遍历数组
    for (int num : nums) {
        // 票数为0时则直接更新majority为当前的值
        if (cnt == 0) {
            majority = num;
        }
        // 如果相同则投票++,否则投票--
        if (majority == num) {
            cnt++;
        } else {
            cnt--;
        }
    }
    return majority;
}
```

## 二分法题目

### 二分法总结

#### 二分查找框架

```
int binarySearch(int[] nums, int target) {
    int left = 0, right = ...;

    while(...) {
        int mid = left + (right - left) / 2;
        if (nums[mid] == target) {
            ...
        } else if (nums[mid] < target) {
            left = ...
        } else if (nums[mid] > target) {
            right = ...
        }
    }
    return ...;
}
```

分析二分查找的一个技巧是：不要出现 else，而是把所有情况用 else if 写清楚，这样可以清楚地展现所有细节。

框架中 `...` 标记的部分，就是可能出现细节问题的地方，当见到一个二分查找的代码时，首先注意这几个地方。后文用实例分析这些地方能有什么样的变化。

另外声明一下，计算 mid 时需要防止溢出，代码中 `left + (right - left) / 2` 就和 `(left + right) / 2` 的结果相同，但是有效防止了 left 和 right 太大直接相加导致溢出。

#### 基本二分查找(寻找一个数)

这个场景是最简单的，即**在一个不存在重复元素的升序数组中搜索一个数，如果存在返回其索引**，否则返回 -1。

```
int binarySearch(int[] nums, int target) {
    int left = 0, right = nums.length - 1;

    // 搜索区间为左右闭区间[left, right]
    while(left <= right) {
        // 计算中值
        int mid = left + (right - left) / 2;
        // 如果中值小于目标值, 说明目标值一定在mid位置的右边 因此更新左边界为mid+1
        if (nums[mid] < target) {
            left = mid + 1;
            // 如果中值大于目标值, 说明目标值一定在mid位置的左边 因此更新右边界为mid-1
        } else if (nums[mid] > target) {
            right = mid - 1;
            // 找到目标值直接返回
        } else if(nums[mid] == target) {
            return mid;
        }
    }

    // 退出while循环的条件是left > right了, 肯定不存在了, 直接返回-1
    return -1;
}
```

1. **为什么 while 循环的条件中是** `**<=**`**, 而不是** `**<**`**?**

答：因为初始化 right 的赋值是 `nums.length - 1`，即最后一个元素的索引，而不是 `nums.length`。

这二者可能出现在不同功能的二分查找中，区别是：前者相当于两端都闭区间 `[left, right]`，后者相当于左闭右开区间 `[left, right)`，因为索引大小为 `nums.length` 是越界的。

这个算法中使用的是前者 `[left, right]` 两端都闭的区间，这个区间其实就是每次进行搜索的区间。

什么时候应该停止搜索呢？当然找到了目标值的时候可以终止：

```
if(nums[mid] == target)
    return mid;
```

但如果没找到，就需要 while 循环终止，然后返回 -1。那 while 循环什么时候应该终止？搜索区间为空的时候应该终止，意味着没得找了，就等于没找到。

`while(left <= right)` 的终止条件是 `left == right + 1`，写成区间的形式就是 `[right + 1, right]`，或者带个具体的数字进去 `[3, 2]`，可见这时候区间为空，因为没有数字既大于等于 3 又小于等于 2 的吧。所以这时候 while 循环终止是正确的，直接返回 -1 即可。

`while(left < right)` 的终止条件是 `left == right`，写成区间的形式就是 `[left, right]`，或者带个具体的数字进去 `[2, 2]`，这时候区间非空，还有一个数 2，但此时 while 循环终止了。也就是说这区间 `[2, 2]` 被漏掉了，索引 2 没有被搜索，如果这时候直接返回 -1 就是错误的。

当然如果非要用 `while(left < right)` 也可以，既然已经知道了出错的原因，就打个补丁好了：

```
//...
while(left < right) {
    // ...
}
return nums[left] == target ? left : -1;
```

2. **为什么** `**left = mid + 1, right = mid - 1**`**? 而看有的代码是** `**right = mid**` **或者** `**left = mid**`**，没有这些加加减减，到底怎么回事?**

答：这也是二分查找的一个难点，不过只要能理解前面的内容，就能够很容易判断。刚才明确了「**搜索区间**」这个概念，而且本算法的搜索区间是两端都闭的，即 `[left, right]`。那么当发现索引 mid 不是要找的 target 时，下一步应该去搜索哪里呢？

当然是去搜索 `[left, mid - 1]` 或者 `[mid + 1, right]` 对不对？因为 mid 已经搜索过，应该从搜索区间中去除。

3. **此算法有什么缺陷？**

比如说有序数组 `nums = [1,2,2,2,3], target = 2`，此算法返回的索引是 2。没错。但是如果想得到 target 的左侧边界，即索引 1，或者想得到 target 的右侧边界，即索引 3，这样的话此算法是无法处理的。

下面就来讨论这两种二分查找的算法。

#### 寻找左侧边界的二分查找

**即在一个存在重复元素的非递减数组中寻找一个数字最开始出现的位置**。

以下是最常见的代码形式，其中的标记是需要注意的细节：

```
int leftBound(int[] nums, int target) {
    if (nums.length == 0) return -1;
    int left = 0;
    int right = nums.length; // 注意

    while (left < right) { // 注意
        int mid = (left + right) / 2;
        if (nums[mid] == target) {
            right = mid;
        } else if (nums[mid] < target) {
            left = mid + 1;
        } else if (nums[mid] > target) {
            right = mid; // 注意
        }
    }
    return left;
}
```

1. **为什么 while 中是** `**<**` **而不是** `**<=**`**?**

答：用相同的方法分析，因为 `right = nums.length` 而不是 `nums.length - 1`。因此每次循环的「搜索区间」是 `[left, right)` 左闭右开。`while(left < right)` 终止的条件是 `left == right`，此时搜索区间 `[left, left)` 为空，所以可以正确终止。

这里先说一个搜索左右边界和上面这个算法的一个区别：前面普通算法的 `right` 不是 `nums.length - 1` 吗，为啥这里非要写成 `nums.length` 使得「搜索区间」变成左闭右开呢？

因为对于搜索左右侧边界的二分查找，这种写法比较普遍，就拿这种写法举例了，保证以后遇到这类代码可以理解。你非要用两端都闭的写法反而更简单，会在后面写相关的代码，把三种二分搜索都用一种两端都闭的写法统一起来，耐心往后看就行了。

2. **为什么没有返回 -1 的操作？如果 nums 中不存在 target 这个值怎么办？**

答：因为要一步一步来，先理解一下这个「左侧边界」有什么特殊含义：

<img src="/zimg/yuque-1777868290280-7.jpeg" style="width: 467px">

对于这个数组，算法会返回 1。这个 1 的含义可以这样解读：nums 中小于 2 的元素有 1 个。

比如对于有序数组 `nums = [2,3,5,7], target = 1`，算法会返回 0，含义是：nums 中小于 1 的元素有 0 个。

再比如说 `nums = [2,3,5,7], target = 8`，算法会返回 4，含义是：nums 中小于 8 的元素有 4 个。

综上可以看出，函数的返回值(即 left 变量的值)取值区间是闭区间 `[0, nums.length]`，所以简单添加两行代码就能在正确的时候 `return -1`：

```
while (left < right) {
    //...
}
// target 比所有数都大
if (left == nums.length) return -1;
// 类似之前算法的处理方式
return nums[left] == target ? left : -1;
```

3. **为什么** `**left = mid + 1,right = mid**` **和之前的算法不一样？**

答：这个很好解释，因为「搜索区间」是 `[left, right)` 左闭右开，所以当 `nums[mid]` 被检测之后，下一步的搜索区间应该去掉 mid 分割成两个区间，即 `[left, mid)` 或 `[mid + 1, right)`。

4. **为什么该算法能够搜索左侧边界?**

答：关键在于对于 `nums[mid] == target` 这种情况的处理：

```
if (nums[mid] == target)
    right = mid;
```

可见找到 target 时不要立即返回，而是缩小「搜索区间」的上界 right，在区间 `[left, mid)` 中继续搜索，即不断向左收缩，达到锁定左侧边界的目的。

5. **为什么返回 left 而不是 right？**

答：都是一样的, 因为 while 终止的条件是 `left == right`。

6. **能不能想办法把 right 变成** `**nums.length - 1**`**，也就是继续使用两边都闭的「搜索区间」？这样就可以和第一种二分搜索在某种程度上统一起来了**。

答：当然可以，只要明白了「搜索区间」这个概念，就能有效避免漏掉元素，随便怎么改都行。下面严格根据逻辑来修改：

因为非要让搜索区间两端都闭，所以 right 应该初始化为 `nums.length - 1`，while 的终止条件应该是 `left == right + 1`，也就是其中应该用 `<=`：

```
int leftBound(int[] nums, int target) {
    // 搜索区间为 [left, right]
    int left = 0, right = nums.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        // if else ...
    }
```

因为搜索区间是两端都闭的，且现在是搜索左侧边界，所以 left 和 right 的更新逻辑如下：

```
if (nums[mid] < target) {
    // 搜索区间变为 [mid+1, right]
    left = mid + 1;
} else if (nums[mid] > target) {
    // 搜索区间变为 [left, mid-1]
    right = mid - 1;
} else if (nums[mid] == target) {
    // 收缩右侧边界
    right = mid - 1;
}
```

由于 while 的退出条件是 `left == right + 1`，所以当 target 比 nums 中所有元素都大时，会存在以下情况使得索引越界：

<img src="/zimg/yuque-1777868290289-8.jpeg" style="width: 489px">

因此最后返回结果的代码应该检查越界情况：

```
if (left >= nums.length || nums[left] != target)
    return -1;
return left;
```

至此完整代码如下：

```
// 左闭右闭区间寻找左边界
int leftBound(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    // 左闭右闭的搜索区间
    while (left <= right) {
        int mid = left + (right - left) / 2;
        // 如果中值小于目标值, 说明目标值一定在mid位置的右边 因此更新左边界为mid+1
        if (nums[mid] < target) {
            left = mid + 1;
            // 如果中值大于目标值, 说明目标值一定在mid位置的左边 因此更新右边界为mid-1
        } else if (nums[mid] > target) {
            right = mid - 1;
            // 与目标值相同. 注意别返回, 需要更新搜索区间的right来逐渐找出左侧边界
        } else if (nums[mid] == target) {
            right = mid - 1;
        }
    }
    // while循环退出条件是left > right, 此时left可能越界
    // 最后要检查 left 越界的情况
    if (left >= nums.length || nums[left] != target) {
        return -1;
    }

    return left;
}
```

这样就和第一种二分搜索算法统一了，都是两端都闭的「搜索区间」，而且最后返回的也是 left 变量的值。

#### 寻找右侧边界的二分查找

**即在一个存在重复元素的非递减数组中寻找一个数字最后出现的位置**。

类似寻找左侧边界的算法，这里也会提供两种写法，还是先写常见的左闭右开的写法，只有两处和搜索左侧边界不同，已标注：

```
int rightBound(int[] nums, int target) {
    if (nums.length == 0) return -1;
    int left = 0, right = nums.length;

    while (left < right) {
        int mid = (left + right) / 2;
        if (nums[mid] == target) {
            left = mid + 1; // 注意
        } else if (nums[mid] < target) {
            left = mid + 1;
        } else if (nums[mid] > target) {
            right = mid;
        }
    }
    return left - 1; // 注意
}
```

1. **为什么这个算法能够找到右侧边界？**

答：类似地, 关键点还是这里：

```
if (nums[mid] == target) {
    left = mid + 1;
```

当 `nums[mid] == target` 时，不要立即返回，而是增大「搜索区间」的下界 left，使得区间不断向右收缩，达到锁定右侧边界的目的。

2. **为什么最后返回** `**left - 1**` **而不像左侧边界的函数返回 left？而且我觉得这里既然是搜索右侧边界, 应该返回 right 才对。**

答：首先 while 循环的终止条件是 `left == right`，所以 left 和 right 是一样的，你非要体现右侧的特点，返回 right - 1 好了。至于为什么要减一，这是搜索右侧边界的一个特殊点，关键在这个条件判断：

```
if (nums[mid] == target) {
    left = mid + 1;
    // 这样想: mid = left - 1
```

<img src="/zimg/yuque-1777868290304-9.jpeg" style="width: 498px">

因为对 left 的更新必须是 `left = mid + 1`，就是说 while 循环结束时，`nums[left]` 一定不等于 target 了，而 `nums[left - 1]` 可能是 target。

至于为什么 left 的更新必须是 `left = mid + 1`，同左侧边界搜索，就不再赘述。

3. **为什么没有返回 -1 的操作？如果 nums 中不存在 target 这个值, 怎么办？**

答：类似之前的左侧边界搜索，因为 while 的终止条件是 `left == right`，就是说 left 的取值范围是 `[0, nums.length]`，所以可以添加两行代码，正确地返回 -1：

```
while (left < right) {
    // ...
}
if (left == 0) return -1;
return nums[left-1] == target ? (left-1) : -1;
```

4. 是否也可以把这个算法的「搜索区间」也统一成两端都闭的形式呢？这样这三个写法就完全统一了。

答：当然可以，类似搜索左侧边界的统一写法，其实只要改两个地方就行了：

```
// 左闭右闭区间寻找右边界
int rightBound(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    // 左闭右闭的搜索区间
    while (left <= right) {
        int mid = left + (right - left) / 2;
        // 如果中值小于目标值, 说明目标值一定在mid位置的右边 因此更新左边界为mid+1
        if (nums[mid] < target) {
            left = mid + 1;
            // 如果中值大于目标值, 说明目标值一定在mid位置的左边 因此更新右边界为mid-1
        } else if (nums[mid] > target) {
            right = mid - 1;
            // 与目标值相同. 注意别返回, 需要更新搜索区间的left来逐渐找出右侧边界
        } else if (nums[mid] == target) {
            left = mid + 1;
        }
    }
    // 最后要检查 right 越界的情况
    if (right < 0 || nums[right] != target) {
        return -1;
    }
    return right;
}
```

### 二分查找(704-简单)

给定一个 n 个**不重复元素有序**的(升序)整型数组 nums 和一个目标值 target，写一个函数搜索 nums 中的 target，如果目标值存在返回下标，否则返回 -1。

```
示例 1: 输入: nums = [-1,0,3,5,9,12], target = 9   输出: 4   解释: 9 出现在 nums 中并且下标为 4
示例 2: 输入: nums = [-1,0,3,5,9,12], target = 2   输出: -1  解释: 2 不存在 nums 中因此返回 -1
```

[https://leetcode.cn/problems/binary-search/description/](https://leetcode.cn/problems/binary-search/description/)

**二分法(左闭右闭)**

这道题目的前提是数组为有序数组，同时题目还强调数组中**无重复元素**，因为一旦有重复元素，使用二分查找法返回的元素下标可能不是唯一的，这些都是使用基本二分法的前提条件，当看到题目描述满足如上条件的时候，可要想一想是不是可以用二分法了。

二分查找涉及的很多的边界条件，逻辑比较简单，但就是写不好。例如到底是 `while(left < right)` 还是 `while(left <= right)`，到底是 `right = mid`，还是要 `right = mid - 1` 呢？二分法经常写乱，主要是因为对区间的定义没有想清楚，区间的定义就是不变量。要在二分查找的过程中保持不变量，就是在 while 寻找中每一次边界的处理都要坚持根据区间的定义来操作，这就是循环不变量规则。

下面定义 target 是在一个在左闭右闭的搜索区间里，也就是 `[left, right]` (这个非常重要)。

区间的定义这就决定了二分法的代码应该如何写，因为定义 target 在 `[left, right]` 区间，所以有如下两点：

- `while (left <= right)` 要使用 `<=`，因为 `left == right` 是有意义的，所以使用 `<=`。
- `if (nums[mid] > target)` right 要赋值为 `mid - 1`，因为当前这个 `nums[mid]` 一定不是 target，那么接下来要查找的左区间结束下标位置就是 `mid - 1`。

例如在数组：`1,2,3,4,7,9,10,11` 中查找元素 3，如图所示：

![](/zimg/yuque-1777868290326-11.png)

不好定义区间的话，画个图就非常明确了。代码如下：

```
public int search(int[] nums, int target) {
    int left = 0, right = nums.length - 1;

    // 搜索区间为左右闭区间[left, right]
    while (left <= right) {
        // 计算中值
        int mid = left + (right - left) / 2;

        // 找到目标值直接返回
        if (nums[mid] == target) {
            return mid;
            // 如果中值小于目标值, 说明目标值一定在mid位置的右边 因此更新左边界为mid+1
        } else if (nums[mid] < target) {
            left = mid + 1;
            // 如果中值大于目标值, 说明目标值一定在mid位置的左边 因此更新右边界为mid-1
        } else if (nums[mid] > target) {
            right = mid - 1;
        }
    }

    // 退出while循环的条件是left > right了, 肯定不存在了, 直接返回-1
    return -1;
}
```

### 搜索插入位置(35-简单) 💯

给定一个排序数组和一个目标值，在数组中找到目标值，并返回其索引。如果目标值不存在于数组中，返回它将会被按顺序插入的位置。

```
示例 1: 输入: nums = [1,3,5,6], target = 5   输出: 2    找到元素返回元素当前位置
示例 2: 输入: nums = [1,3,5,6], target = 2   输出: 1    未找到元素返回插入的位置
示例 3: 输入: nums = [1,3,5,6], target = 7   输出: 4    未找到元素返回插入的位置
```

[https://leetcode.cn/problems/search-insert-position/description/](https://leetcode.cn/problems/search-insert-position/description/)

**二分法**

本题就是基础二分法的变化版本。区别在于二分查找找不到元素返回 -1，这里**找不到元素返回 left 索引即可**。因为循环 `while (left <= right)` 退出条件是 `left > right`，因此退出循环时 left 就是需要插入的位置。

<img src="/zimg/yuque-1777868290351-12.gif" style="width: 667px">

代码如下：

```
public int searchInsert(int[] nums, int target) {
    if (nums == null || nums.length == 0) return -1;

    // nums = [1,3,5,6,7], target = 4
    int left = 0, right = nums.length - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        // 找到元素了直接返回索引
        if (nums[mid] == target) {
            return mid;
            // 如果中值小于目标值, 说明目标值一定在mid位置的右边 因此更新左边界为mid+1
        } else if (nums[mid] < target) {
            left = mid + 1;
            // 如果中值大于目标值, 说明目标值一定在mid位置的左边 因此更新右边界为mid-1
        } else if (nums[mid] > target) {
            right = mid - 1;
        }
    }
    // 退出while循环的条件是left > right了, 有可能存在溢出的问题, 所以判断一下
    if (left > nums.length - 1) return nums.length;

    // 只有最后这里与二分查找不同, 返回left即可
    return left;
}
```

### 大于给定字符的最小字符(744-简单)

给定一个有序的字符数组 letters 和一个字符 target，要求找出 letters 中大于 target 的最小字符，如果找不到就返回第 1 个字符。

```
Input: letters = ["c", "f", "j"]  target = "d"  Output: "f"
Input: letters = ["c", "f", "j"] target = "k"  Output: "c"(找不到返回第一个字符)
```

[https://leetcode.cn/problems/find-smallest-letter-greater-than-target/description/](https://leetcode.cn/problems/find-smallest-letter-greater-than-target/description/)

**二分法**

依然是二分查找。

```
public char nextGreatestLetter(char[] letters, char target) {
    // 左右双指针
    int left = 0, right = letters.length - 1;

    // 循环退出条件
    // 搜索区间为左右闭区间[left, right]
    while (left <= right) {
        int mid = left + (right - left) / 2;
        // mid小于等于target时, 说明 mid 之前的元素都小于等于 target, 移动左边界到mid+1的位置
        if (letters[mid] <= target) {
            left = mid + 1;
        } else {
            // 当 mid 大于 target 时直接移动 right
            right = mid - 1;
        }
    }

    if (left >= letters.length) {
        // 如果 left 超过了数组的长度, 说明没有比 target 大的元素, 返回第一个元素
        return letters[0];
    }

    // 如果找不到返回第一个元素
    return letters[left];
}
```

```
int leftBound(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    // 左闭右闭的搜索区间
    while (left <= right) {
        int mid = left + (right - left) / 2;
        // 如果中值小于目标值, 说明目标值一定在mid位置的右边 因此更新左边界为mid+1
        if (nums[mid] < target) {
            left = mid + 1;
            // 如果中值大于目标值, 说明目标值一定在mid位置的左边 因此更新右边界为mid-1
        } else if (nums[mid] > target) {
            right = mid - 1;
            // 与目标值相同. 注意别返回, 需要更新搜索区间的right来逐渐找出左侧边界
        } else if (nums[mid] == target) {
            right = mid - 1;
        }
    }
    // while循环退出条件是left > right, 此时left可能越界
    // 最后要检查 left 越界的情况
    if (left >= nums.length || nums[left] != target) {
        return -1;
    }

    return left;
}

// 左闭右闭区间寻找右边界
int rightBound(int[] nums, int target) {
    int left = 0, right = nums.length - 1;
    // 左闭右闭的搜索区间
    while (left <= right) {
        int mid = left + (right - left) / 2;
        // 如果中值小于目标值, 说明目标值一定在mid位置的右边 因此更新左边界为mid+1
        if (nums[mid] < target) {
            left = mid + 1;
            // 如果中值大于目标值, 说明目标值一定在mid位置的左边 因此更新右边界为mid-1
        } else if (nums[mid] > target) {
            right = mid - 1;
            // 与目标值相同. 注意别返回, 需要更新搜索区间的left来逐渐找出右侧边界
        } else if (nums[mid] == target) {
            left = mid + 1;
        }
    }
    // 最后要检查 right 越界的情况
    if (right < 0 || nums[right] != target) {
        return -1;
    }
    return right;
}
```

一个长度为 `n-1` 的递增排序数组中的所有数字都是唯一的，并且每个数字都在范围 `0～n-1` 之内。在范围 `0～n-1` 内的 n 个数字中有且只有一个数字不在该数组中，请找出这个数字。


[https://leetcode-cn.com/problems/que-shi-de-shu-zi-lcof](https://leetcode-cn.com/problems/que-shi-de-shu-zi-lcof)

**题解思路**

排序数组中的搜索问题，首先想到 二分法 解决。根据题意，数组可以按照以下规则划分为两部分。

- 左子数组：`nums[i] = i`。
- 右子数组：`nums[i] != i`。

缺失的数字等于 "右子数组的首位元素" 对应的索引，因此考虑使用二分法查找 "右子数组的首位元素"。

过程参考：

代码如下：

```
public int takeAttendance(int[] nums) {
    if (nums == null || nums.length == 0) {
        return -1;
    }

    // [left, right]的左右闭区间
    int left = 0, right = nums.length - 1;

    // 循环退出条件是left > right
    while (left <= right) {
        int mid = left + (right - left) / 2;
        // 如果中值等于mid, 说明缺失的数字在mid的右边
        if (nums[mid] == mid) {
            left = mid + 1;
        } else {
            // 缺失的数字在mid的左边
            right = mid - 1;
        }
    }

    // 最后看看left是否越过了数组的右边界
    if (left >= nums.length) {
        return nums.length;
    }

    return left;
}
```

### 旋转排序数组中的最小值(153-中等) 🔑💯

把一个不存在重复元素的升序数组进行旋转，找出旋转数组的最小值。

```
输入: nums = [3,4,5,1,2]      输出: 1
输入: nums = [4,5,6,7,0,1,2]  输出: 0
```

[https://leetcode.cn/problems/find-minimum-in-rotated-sorted-array/description/](https://leetcode.cn/problems/find-minimum-in-rotated-sorted-array/description/)

**二分法**

关键在于确定对半分得到的两个数组哪一个是旋转数组，哪一个是递增数组。很容易知道**递增数组的第一个元素一定小于等于最后一个元素**。

- 这里 while 里面取 `left < right` 即可，因为 `left == right` 时，就代表只有一个元素了，此时就算找到了。
- 取中值后，对比 `nums[left]` 与 `nums[right]` 值，如果 `nums[left] < nums[right]`，则说明右半部分是递增的，最小值在左边。此时把 right 指向 mid，注意这里有可能 mid 本身就是最小值，所以不能取 `mid - 1`。否则左边是递增的，由于 left 到 mid 是递增的，所以 mid 一定不是最小值，所以 left 指向 `mid + 1`。

![](/zimg/yuque-1777868291162-17.png)

代码如下：

```
public int findMin(int[] nums) {
    if (nums.length == 0) {
        return 0;
    }

    int left = 0, right = nums.length - 1;
    // 5 6 7 8 9 1 2 3 4
    // 二分查找搜索
    while (left < right) {
        // 取中值
        int mid = left + (right - left) / 2;
        // 如果mid小于right的值, 说明右边是递增的
        if (nums[mid] < nums[right]) {
            // 最小值在左边, 此时更新right到mid的位置 注意这里不是mid-1 因为mid自己可能就是最小值
            right = mid;
        } else {	// 说明mid左边是递增的
            // 这里说明目标值一定在mid的右边, 更新左索引到mid+1
            left = mid + 1;
        }
    }

    // 指针相遇, 返回这个值
    return nums[left];
}
```

注意：如果数组元素允许重复(力扣题目：154)，会出现一个特殊的情况：`nums[l] == nums[m] == nums[h]`，此时无法确定解在哪个区间，需要切换到顺序查找。例如对于数组 `{1, 1, 1, 0, 1}`，l、m 和 h 指向的数都为 1，此时无法知道最小数字 0 在哪个区间。

```
public int minNumberInRotateArray2(int[] nums) {
    if (nums.length == 0) return 0;
    int left = 0, right = nums.length - 1;
    while (left < right) {
        int mid = left + (right - left) / 2;
        // 左右中值都一样
        if (nums[left] == nums[mid] && nums[mid] == nums[right]) {
            // 此时从左到右顺序查找
            return minNumber(nums, left, right);
            // 中间小于右边
        } else if (nums[mid] <= nums[right]) {
            right = mid;
            // 中间小于左边
        } else {
            left = mid + 1;
        }
    }
    return nums[left];
}

private int minNumber(int[] nums, int left, int right) {
    for (int i = left; i < right; i++) {
        if (nums[i] > nums[i + 1]) {
            return nums[i + 1];
        }
    }
    return nums[left];
}
```

### 寻找旋转排序数组中的值(33-中等) 💯

把一个**不存在重复元素的升序数组进行旋转**，并在旋转数组中找出指定的值。

```
输入: nums = [4,5,6,7,0,1,2], target = 0  输出: 4
输入: nums = [4,5,6,7,0,1,2], target = 3   输出: -1
输入: nums = [1], target = 0   输出: -1
```

[https://leetcode.cn/problems/search-in-rotated-sorted-array/description/](https://leetcode.cn/problems/search-in-rotated-sorted-array/description/)

**二分法**

此题与 [旋转排序数组中的最小值(153-中等)](https://www.yuque.com/daerwendecha/qwuv92/ik6lslhlbzchwpsx#S1Uvl) 非常类似。但是本题是找指定的值，由于发生了旋转，所以在判断左边还是右边有序之后，还需要一些补充的逻辑。

由于题目说数字了无重复，举个例子：`1 2 3 4 5 6 7`。可以大致分为两类：

- 第一类 `2 3 4 5 6 7 1` 这种，也就是 `nums[start] <= nums[mid]`。此例子中就是 `2 <= 5`。这种情况下，左半部分有序。因此如果 `nums[start] <= target < nums[mid]`，则继续在左半部分找，否则去右半部分找。第二类 `6 7 1 2 3 4 5`。
- 这种也就是 `nums[start] > nums[mid]`。此例子中就是 `6 > 2`。这种情况下，右半部分有序。因此如果 `nums[mid] < target <= nums[end]`，则在右半部分找，否则去左半部分找。

代码如下：

```
public int search(int[] nums, int target) {
    if (nums == null || nums.length == 0) {
        return -1;
    }
    int left = 0;
    int right = nums.length - 1;

    while (left <= right) {
        int mid = left + (right - left) / 2;
        // 找到目标值直接返回
        if (nums[mid] == target) {
            return mid;
        }

        // 说明右边部分有序
        if (nums[mid] < nums[right]) {
            // 说明目标值在右边部分, 更新左边界
            if (nums[mid] < target && target <= nums[right]) {
                left = mid + 1;
                // 否则更新右边界
            } else {
                right = mid - 1;
            }
            // 说明左边部分有序
        } else {
            // 说明目标值在左边部分, 更新右边界
            if (nums[mid] > target && target >= nums[left]) {
                right = mid - 1;
                // 否则更新左边界
            } else {
                left = mid + 1;
            }
        }
    }
    return -1;
}
```

### 求平方根(69-简单) 💯

给你一个非负整数 x，计算并返回 x 的算术平方根。由于返回类型是整数，结果只保留整数部分，小数部分将被舍去。注意：不允许使用任何内置指数函数和算符。

```
输入：x = 4   输出：2
输入：x = 8   输出：2
解释：8 的算术平方根是 2.82842..., 由于返回类型是整数，小数部分将被舍去。
```

[https://leetcode.cn/problems/sqrtx/description](https://leetcode.cn/problems/sqrtx/description)

**二分查找法**

由于 x 平方根的整数部分 ans 是满足 `k * k ≤ x` 的最大 k 值，因此**可以对 k 进行二分查找**，从而得到答案。

二分查找的下界为 0，上界可以粗略地设定为 x。在二分查找的每一步中，只需要比较中间元素 mid 的平方与 x 的大小关系，并通过比较的结果调整上下界的范围。由于所有的运算都是整数运算，不会存在误差，因此在得到最终的答案 res 后，也就不需要再去尝试 `res + 1` 了。

要点：

- **使用左右双指针**。
- `mid <= x / mid` **使用除法而不是乘法来避免溢出**。
- 使用 `result` 来滚动记录结果值。

```
public int mySqrt(int x) {
    int left = 1;
    int right = x;

    // 滚动更新结果值
    int result = 0;

    // 左右闭区间
    while (left <= right) {
        int mid = left + (right - left) / 2;

        // 使用除法而不是乘法来避免溢出
        if (mid <= x / mid) {
            // 不断更新结果值
            result = mid;
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }

    return result;
}
```

### 计算指定精度的平方根

一种求带小数指定精度平方根的写法，注意 right 边界是 1，因为**需要处理类似 0.01 的平方根结果为 0.1 的情况**。

```
public float mySqrt(float num, float e) {
    // 结果为小数
    float res = 0;

    if (num > 0 && e > 0) {
        float left = 0;
        float right = Math.max(num, 1);  // 解决num<1时的问题!!!!!

        while (left < right) {
            float mid = (left + right) / 2;

            if (mid * mid < num - e) {
                left = mid;
            } else if (mid * mid > num + e) {
                right = mid;
            } else {
                res = mid;
                break;
            }
        }
    }

    return res;
}
```

### 两数平方和(633-中等)

判断一个数是否为两个数的平方和。

```
输入：c = 5   输出：true   解释：1 * 1 + 2 * 2 = 5
```

[https://leetcode.cn/problems/sum-of-square-numbers/description/](https://leetcode.cn/problems/sum-of-square-numbers/description/)

**平方根双向逼近法**

找到这个数的平方根开始不断逼近求。

由于是求平方根之和，所以其中最大的一个数也就是可能为目标值的平方根。因此可以计算此目标值的平方根 right，然后使用双指针逐渐从 `left = 0` 逼近计算目标值。

注意这里为了避免值溢出，不能计算 `left * left + right * right` 的结果再与 num 比较，而是直接拿 num 减去其中一个的值再与另一个比较。

```
public boolean judgeSquareSum(int num) {
    // 由于是平方数之和, 所以可能的最大的范围就是num的平方根
    int left = 0, right = (int) Math.sqrt(num);

    while (left <= right) {

        // 找到目标值
        if (left * left == num - right * right) {
            return true;
            // 如果平方和大于目标值, 则右指针左移
        } else if (left * left > num - right * right) {
            right--;
            // 如果平方和小于目标值, 则左指针右移
        } else {
            left++;
        }
    }
    return false;
}
```

### 有序矩阵的第 K 小元素(378-中等)

给定一个 `n x n` 的矩阵，其中每行和每列元素均按升序排序，找到**矩阵中第 k 小的元素**。请注意是排序后的第 k 小元素，而不是第 k 个不同的元素。

```
matrix = [
[ 1,  5,  9],
[10, 11, 13],
[12, 13, 15]
],  k = 8,  返回 13.
```

[https://leetcode.cn/problems/kth-smallest-element-in-a-sorted-matrix/description/](https://leetcode.cn/problems/kth-smallest-element-in-a-sorted-matrix/description/)

此题比 搜索二维矩阵(240-中等) 的搜索某个元素要困难一些。

**二分法**

有序的可以考虑二分法。对于矩阵来说，有序的区间范围就是矩阵的左上角和右下角，也就是 left 和 right。这里写一个辅助方法，**用于统计矩阵中小于等于 mid 的元素个数，注意这里统计时也能利用有序的特性进行逐层统计**。然后根据元素个数来缩小二分的区间。最后返回区间左边即可。

```
public int kthSmallest(int[][] matrix, int k) {
    // 矩阵的左上和右下角, 也就是最小值和最大值
    int left = matrix[0][0];
    int right = matrix[matrix.length - 1][matrix[0].length - 1];

    // 循环退出条件
    // 搜索区间为左右闭区间[left, right]
    while (left <= right) {
        // 计算中间值
        int mid = left + (right - left) / 2;
        // 统计小于等于 mid 的元素个数
        int count = countLessOrEqual(matrix, mid);

        // 注意这里并没有直接返回 mid，而是根据 count 和 k 的大小关系来调整搜索区间
        // 因为矩阵需要两个索引才能确定位置, 可能没有元素等于 mid，所以就不判断 count 和 k 相等的情况

        if (count < k) {
            // 第 k 小元素在右半部分 [mid+1, right]
            left = mid + 1;
        } else {
            // 第 k 小元素在左半部分 [left, mid-1]
            right = mid - 1;
        }
    }

    // 最终 left 就是第 k 小元素
    return left;
}

// 统计矩阵中小于等于target的元素个数
private int countLessOrEqual(int[][] matrix, int target) {
    int length = matrix.length;
    int count = 0;
    int row = length - 1;
    // 从左下角开始
    int col = 0;

    while (row >= 0 && col < length) {
        if (matrix[row][col] <= target) {
            // 当前列中小于等于 target 的元素个数
            count += row + 1;
            col++; // 向右移动
        } else {
            row--; // 向上移动
        }
    }

    return count;
}
```

### 有序数组中的单一元素(540-中等)

给定一个只包含整数的有序数组，每个元素都会出现两次，唯有一个数只会出现一次，找出这个数。要求时间复杂度 `o(lgn)`。

```
输入: [1,1,2,3,3,4,4,8,8]   输出: 2
```

[https://leetcode.cn/problems/single-element-in-a-sorted-array/description/](https://leetcode.cn/problems/single-element-in-a-sorted-array/description/)

**二分法**

可以用异或法，但是这样复杂度就是 `O(N)` 了。由于是有序数组，所以可以用二分。

二分时，对于计算的 `mid` 值，需要确保其为一个偶数，这样才能通过比较 `mid` 和 `mid + 1` 位置的值是否相同，来决定往哪边去缩小区间。如果 `nums[mid] == nums[mid + 1]`，说明 `[left, mid + 1]` 这个区间**内的所有元素都出现了两次**，此时需要往右缩小区间，否则则往左缩小区间。

代码如下：

```
public int singleNonDuplicate(int[] nums) {
    int left = 0, right = nums.length - 1;

    // 循环退出条件
    // 搜索区间为左右闭区间[left, right]
    while (left <= right) {
        int mid = left + (right - left) / 2;

        // 确保mid是一个偶数, 这样就能判断mid与mid+1位置的元素是否相同
        if (mid % 2 == 1) {
            mid--;
        }

        // 检查 mid 和 mid + 1 是否相等
        if (mid < nums.length - 1 && nums[mid] == nums[mid + 1]) {
            // 如果相等, 说明只出现一次的元素在右侧的区间 [mid + 2, right]
            left = mid + 2;
        } else {
            // 如果不相等，说明只出现一次的元素在左侧的区间 [left, mid - 1] 或 mid 就是目标
            right = mid - 1;
        }
    }

    // 最终 left 指向的就是只出现一次的元素
    return nums[left];
}
```
