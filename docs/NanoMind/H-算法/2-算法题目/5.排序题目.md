### 扑克牌顺子(LCR-186-简单) 🔑

五张牌，其中大小鬼为癞子，牌面为 0。判断这五张牌是否能组成顺子。

![](/zimg/yuque-1777880829592-1.png)

[https://leetcode.cn/problems/bu-ke-pai-zhong-de-shun-zi-lcof/description/](https://leetcode.cn/problems/bu-ke-pai-zhong-de-shun-zi-lcof/description/)

**排序法**

首先统计癞子牌的张数，然后排序，然后遍历看实际需要多少个癞子牌才能达标。

```
public boolean checkDynasty(int[] nums) {
    if (nums.length < 5) return false;
    // 从小到大排序
    Arrays.sort(nums);
    // 统计癞子数量
    int king = 0;
    for (int num : nums) {
        if (num == 0) king++;
    }

    // 统计需要多少个癞子
    int gap = 0;

    // 使用癞子去补全不连续的顺子, 这里从癞子数开始遍历
    for (int i = king; i < nums.length - 1; i++) {
        // 如果前后牌相同, 直接不行
        if (nums[i] == nums[i + 1]) {
            return false;
        }
        // 如果前后不一致则计算需要多少个King才能补齐
        gap = gap + (nums[i + 1] - nums[i] - 1);
    }

    // 最后判断king数量是否超标
    return king >= gap;
}
```

### H 数(274-中等) 💯

根据维基百科上 h 指数的定义：h 代表 "高引用次数"，一名科研人员的 h 指数是指至少发表了 h 篇论文，并且至少有 h 篇论文被引用次数大于等于 h。如果 h 有多种可能的值，h 指数是其中最大的那个。

给一个整数数组 citations，其中 `citations[i]` 表示研究者的第 i 篇论文被引用的次数。计算并返回该研究者的 h 指数。

```
示例 1: 输入: citations = [3,0,6,1,5] 输出: 3
解释: 给定数组表示研究者总共有 5 篇论文, 每篇论文相应的被引用了 3, 0, 6, 1, 5 次. 由于研究者有 3 篇论文每篇 至少 被引用了 3 次, 其余两篇论文每篇被引用 不多于 3 次, 所以她的 h 指数是 3.
示例 2: 输入: citations = [1,3,1]  输出: 1
```

[https://leetcode.cn/problems/h-index/description/](https://leetcode.cn/problems/h-index/description/)

**排序法**

这个题本质上就是看有多少大于当前引用次数的文章，所以对数组进行排序之后，天然就具备了这个能力。

排序后从左往右遍历，逐一把比当前文章引用数更多的文章数量(`citations.length - i`)和 当前文章引用数量(`citations[i]`) 进行对比，如果前者大则符合 h 指数的条件，此时更新 h 指数即可。

![](/zimg/yuque-1777880829605-2.png)

代码如下：

```
public int hIndex(int[] citations) {
    if (citations == null) {
        return 0;
    }

    // 从大到小排序
    Arrays.sort(citations);

    int h = 0;
    // 挨着遍历, 找到最大的h
    for (int i = 0; i < citations.length; i++) {
        // 如果当前的引用数大于等于剩余的文章数, 则认为找到了一个H指数
        if (citations[i] >= citations.length - i) {
            // 更新最大的H指数
            h = Math.max(h, citations.length - i);
        }
    }

    return h;
}
```

**字符串排序法**

        return "";

        nums[i] = numbers[i] + "";

    Arrays.sort(nums, (s1, s2) -> (s1 + s2).compareTo(s2 + s1));

    for (String str : nums) {
        res.append(str);
    }

    return res.toString();
}

### 下一个排列(31-中等) 💯

整数数组的一个排列就是将其所有成员以序列或线性顺序排列。例如 `arr = [1,2,3]`，以下这些都可以视作 arr 的排列：`[1,2,3]、[1,3,2]、[3,1,2]、[2,3,1]`。

整数数组的 下一个排列 是指其整数的下一个字典序更大的排列。更正式地，如果数组的所有排列根据其字典顺序从小到大排列在一个容器中，那么数组的 下一个排列 就是在这个有序容器中排在它后面的那个排列。如果不存在下一个更大的排列，那么这个数组必须重排为字典序最小的排列，即其元素按升序排列。

给你一个整数数组 nums，找出 nums 的下一个排列。必须原地修改，只允许使用额外常数空间。

```
输入：nums = [1,2,3]   输出：[1,3,2]
输入：nums = [3,2,1]   输出：[1,2,3]
输入：nums = [1,1,5]   输出：[1,5,1]
```

[https://leetcode.cn/problems/next-permutation/description](https://leetcode.cn/problems/next-permutation/description)

**两遍扫描排序法**

下一个排列总是比当前排列要大，除非该排列已经是最大的排列。整体是希望找到一种方法，能够找到一个大于当前序列的新序列，且变大的幅度尽可能小。

具体地流程：

- 将一个左边的「较小数」与一个右边的「较大数」交换，以能够让当前排列变大，从而得到下一个排列。
- 同时要让这个「较小数」尽量靠右，而「较大数」尽可能小。当交换完成后，「较大数」右边的数需要按照升序重新排列。这样可以在保证新排列大于原来排列的情况下，使变大的幅度尽可能小。

以排列 `[4,5,2,6,3,1]` 为例：能找到的符合条件的一对「较小数」与「较大数」的组合为 2 与 3，满足「较小数」尽量靠右，而「较大数」尽可能小。当 2 和 3 完成交换后排列变为 `[4,5,3,6,2,1]`，此时可以重排「较小数」右边的序列，序列变为 `[4,5,3,1,2,6]`。这里的重新排序，其实就是反转一下，因为之前是降序的，反转后就变成了升序。

![](/zimg/yuque-1777880829648-4.gif)

代码如下：

```
public void nextPermutation(int[] nums) {
    if (nums == null || nums.length == 0) {
        return;
    }

    int firstIndex = -1;
    // 从后往前找到第一个升序的位置
    for (int i = nums.length - 2; i >= 0; i--) {
        if (nums[i] < nums[i + 1]) {
            firstIndex = i;
            break;
        }
    }
    // 如果没有找到升序的位置, 说明整个数组是降序的, 直接翻转整个数组即可
    if (firstIndex == -1) {
        reverse(nums, 0, nums.length - 1);
        return;
    }

    int secondIndex = -1;
    // 从后往前找到第一个比firstIndex位置大的数
    for (int i = nums.length - 1; i >= 0; i--) {
        if (nums[i] > nums[firstIndex]) {
            secondIndex = i;
            break;
        }
    }
    // 交换两个位置的数
    swap(nums, firstIndex, secondIndex);
    // 翻转firstIndex后面的数(也就是原来的降序变成升序)
    reverse(nums, firstIndex + 1, nums.length - 1);
}

private void reverse(int[] nums, int i, int j) {
    while (i < j) {
        swap(nums, i++, j--);
    }
}

private void swap(int[] nums, int i, int i1) {
    int tmp = nums[i];
    nums[i] = nums[i1];
    nums[i1] = tmp;
}
```

## 排序算法题目

### ⭐️ 快速排序

给你一个整数数组 nums，请将该数组升序排列。必须在不使用任何内置函数的情况下解决问题，时间复杂度为 `O(nlog(n))`，并且空间复杂度尽可能小。

[https://leetcode.cn/problems/sort-an-array/description/](https://leetcode.cn/problems/sort-an-array/description/)

**快速排序法**

从一个数组中随机选出一个数 N，通过一趟排序将数组分割成三个部分，小于 N 的区域，等于 N 的区域，大于 N 的区域，然后再按照此方法对小于区的和大于区分别递归进行，从而达到整个数据变成有序数组。这里的写法是三向切分快速排序，可以应对大量数据重复的情况。

![](/zimg/yuque-1777880829711-5.png)

算法流程参考：[https://blog.csdn.net/u010452388/article/details/81218540](https://blog.csdn.net/u010452388/article/details/81218540)

下面的写法是**三向切分快速排序**，可以应对大量数据重复的情况。面试可用。**这个代码可以作为模板**！

```
public int[] sort(int[] nums) {
    if (nums == null || nums.length <= 1) {
        return nums;
    }
    quickSort(nums, 0, nums.length - 1);
    return nums;
}

private void quickSort(int[] nums, int left, int right) {
    // 递归终止条件: 两个指针相遇
    if (left >= right) {
        return;
    }

    int lt = left;       // 小于基准的区域的右边界
    int gt = right;      // 大于基准的区域的左边界

    int pivot = nums[left];  // 选择第一个元素作为基准(简单实现, 一般可以随机取数值)
    int i = left;    // 从pivot开始, 当前正在检查的元素

    // 注意: 这里是 i <= gt, 因为是这两个指针在逐渐逼近
    while (i <= gt) {
        // 小于pivot：如果当前元素小于基准, 则交换到小于区域的右边界
        if (nums[i] < pivot) {
            swap(nums, lt, i);
            lt++;
            i++;
            // 大于pivot：如果当前元素大于基准, 则交换到大于区域的左边界
            // 这里仅移动右指针(因为交换后的元素需要重新检查)
        } else if (nums[i] > pivot) {
            swap(nums, i, gt);
            gt--;
            // 等于pivot：只移动当前的指针
        } else {
            i++;
        }
    }
    // 现在数组分为三个闭区间
    // nums[left..lt - 1] < pivot
    // nums[lt..gt] == pivot
    // nums[gt + 1..right] > pivot

    // 下面处理的都是闭区间
    // 这里只对大于pivot部分和小于pivot部分再排序了 等于的区间就不管了!!
    quickSort(nums, left, lt - 1);
    quickSort(nums, gt + 1, right);
}

private void swap(int[] nums, int i, int j) {
    int temp = nums[i];
    nums[i] = nums[j];
    nums[j] = temp;
}
```

### 数组中第 K 个最大元素(215-中等) 💯

找到数组中第 k 大的元素。

[https://leetcode.cn/problems/kth-largest-element-in-an-array/description/](https://leetcode.cn/problems/kth-largest-element-in-an-array/description/)

快速选择和堆排序都用于可以求解 Kth Element 和 TopK Elements 问题。

**堆排序法**

使用小顶堆并保持堆的大小最大为 K，这样堆中维护的就是前 K 个最大的元素，由于是**小顶堆**，所以堆顶就是第 K 大的元素。

```
public int findKthLargest(int[] nums, int k) {
    // 小顶堆, 顶部元素是最小的
    PriorityQueue<Integer> minHeap = new PriorityQueue<>();
    for (int num : nums) {
        // 所有元素入堆
        minHeap.add(num);
        // 维护堆的大小为k
        if (minHeap.size() > k) {
            minHeap.poll();
        }
    }
    // 直接获取堆顶即为第k大的
    return minHeap.peek();
}
```

**快速选择法**

快速排序的核心包括 "哨兵划分" 和 "递归"。

- 哨兵划分：以数组某个元素(一般选取首元素)为基准数，将所有小于基准数的元素移动至其左边，大于基准数的元素移动至其右边。
- 递归：对 左子数组 和 右子数组 递归执行 哨兵划分，直至子数组长度为 1 时终止递归，即可完成对整个数组的排序。

下图展示了数组 `[2,4,1,0,3,5]` 的快速排序流程。

![](/zimg/yuque-1777880829722-6.png)

三向切分快速选择的代码如下（使用快排的模板）：

```
public int findKthLargest(int[] nums, int k) {
    if (nums == null || nums.length == 0 || k < 1 || k > nums.length) {
        throw new IllegalArgumentException("Invalid input");
    }

    // 转换为找第 (nums.length - k) 小的元素
    // 因为第k大就是升序排序后的第nums.length - k个元素
    return quickSelect(nums, 0, nums.length - 1, nums.length - k);
}

private int quickSelect(int[] nums, int left, int right, int k) {
    // Base case
    if (left == right) {
        return nums[left];
    }

    int lt = left;       // 小于基准的区域的右边界
    int gt = right;      // 大于基准的区域的左边界
    int pivot = nums[left];  // 选择第一个元素作为基准
    int i = left;        // 当前正在检查的元素

    while (i <= gt) {
        if (nums[i] < pivot) {
            swap(nums, lt, i);
            lt++;
            i++;
        } else if (nums[i] > pivot) {
            swap(nums, i, gt);
            gt--;
        } else {
            i++;
        }
    }

    // 现在数组分为三个部分(闭区间)：
    // nums[left..lt-1] < pivot
    // nums[lt..gt] == pivot
    // nums[gt+1..right] > pivot

    // 如果k在等于pivot的范围内, 找到了直接返回pivot!!!!
    if (k >= lt && k <= gt) {
        return pivot;
        // 如果k在小于pivot的范围内, 递归处理左半部分
    } else if (k < lt) {
        return quickSelect(nums, left, lt - 1, k);
        // 否则递归处理右半部分
    } else {
        return quickSelect(nums, gt + 1, right, k);
    }
}

private void swap(int[] nums, int i, int j) {
    int temp = nums[i];
    nums[i] = nums[j];
    nums[j] = temp;
}
```

另一种类似使用额外空间但是更好理解的代码如下：

```
public int findKthLargest(int[] nums, int k) {
    List<Integer> numList = new ArrayList<>();
    for (int num : nums) {
        numList.add(num);
    }
    return quickSelect(numList, k);
}

private int quickSelect(List<Integer> nums, int k) {
    // 随机选择基准数
    Random rand = new Random();
    int pivot = nums.get(rand.nextInt(nums.size()));
    // 将大于, 小于, 等于 pivot 的元素划分至 big, small, equal列表中
    List<Integer> small = new ArrayList<>();
    List<Integer> equal = new ArrayList<>();
    List<Integer> big = new ArrayList<>();
    for (int num : nums) {
        if (num > pivot) {
            big.add(num);
        } else if (num < pivot) {
            small.add(num);
        } else {
            equal.add(num);
        }
    }

    // 第 k 大元素在 big 中, 继续递归划分
    if (k <= big.size()) {
        return quickSelect(big, k);
    }

    // 第 k 大元素在 small 中, 递归划分
    if (nums.size() - small.size() < k) {
        return quickSelect(small, k - nums.size() + small.size());
    }

    // 第 k 大元素在 equal 中, 直接返回 pivot
    return pivot;
}
```

### 数组中最小的 K 个数(LCR-159-中等) 🔑

输入 n 个整数，找出其中最小的 K 个数。例如输入 `4,5,1,6,2,7,3,8` 这 8 个数字，则最小的 4 个数字是 `1,2,3,4`。

[https://leetcode.cn/problems/zui-xiao-de-kge-shu-lcof/description/](https://leetcode.cn/problems/zui-xiao-de-kge-shu-lcof/description/)

**堆排序法**

使用大顶堆维护元素，这样堆顶的元素就是最大的，堆内元素都比堆顶小，这样就是最小的 K 个数了。

维护一个大小为 K 的最小堆过程如下：在添加一个元素之后，如果大顶堆的大小大于 K，那么需要将大顶堆的堆顶元素去除。

```
public int[] inventoryManagement(int[] nums, int k) {
    if (k == 0 || nums.length == 0) {
        return new int[0];
    }
    // 默认是小根堆, 实现大根堆需要重写一下比较器.
    Queue<Integer> bigHeap = new PriorityQueue<>((v1, v2) -> v2 - v1);
    for (int num : nums) {
        bigHeap.add(num);
        // 如果堆的大小超过了k, 就弹出一个元素
        if (bigHeap.size() > k) {
            bigHeap.poll();
        }
    }

    // 返回堆中的元素
    int[] res = new int[bigHeap.size()];
    int idx = 0;
    for (int num : bigHeap) {
        res[idx++] = num;
    }
    return res;
}
```

**快速排序法**

直接使用**快速排序**对整个数组排序，最后得到前 K 个小的元素。

**⭐\*\***️ 快速选择法\*\*

题目只要求返回最小的 cnt 个数，对这 cnt 个数的顺序并没有要求。因此只需要**将数组划分为最小的 cnt 个数 和其他数字两部分即可，而快速排序的哨兵划分可完成此目标**。

根据快速排序原理，如果某次哨兵划分后 基准数正好是第 `cnt + 1` 小的数字，那么此时基准数左边的所有数字便是题目所求的 最小的 cnt 个数。

根据此思路，考虑在每次哨兵划分后，判断基准数在数组中的索引是否等于 cnt，若 true 则直接返回此时数组的前 cnt 个数字即可。注意这里**不需要处理相等的情况**。

```
public int[] getLeastNumbers(int[] nums, int k) {
    if (nums == null || nums.length == 0 || k <= 0) {
        return new int[0];
    }
    if (k >= nums.length) {
        return nums.clone();
    }

    // 使用快速选择算法找到前k小的数
    quickSelect(nums, 0, nums.length - 1, k);

    // 返回前k个元素(不需要排序)
    int[] result = new int[k];
    System.arraycopy(nums, 0, result, 0, k);

    return result;
}

private void quickSelect(int[] nums, int left, int right, int k) {
    // Base case
    if (left >= right) {
        return;
    }

    int lt = left;       // 小于基准的区域的右边界
    int gt = right;      // 大于基准的区域的左边界
    int pivot = nums[left];  // 选择第一个元素作为基准
    int i = left;        // 当前正在检查的元素

    while (i <= gt) {
        if (nums[i] < pivot) {
            swap(nums, lt, i);
            lt++;
            i++;
        } else if (nums[i] > pivot) {
            swap(nums, i, gt);
            gt--;
        } else {
            i++;
        }
    }

    // 现在数组分为三个部分(闭区间)：
    // nums[left..lt-1] < pivot
    // nums[lt..gt] == pivot
    // nums[gt+1..right] > pivot

    if (k < lt) {
        // 只需要在左边部分继续排序
        quickSelect(nums, left, lt - 1, k);
    } else if (k > gt) {
        // 需要在右边部分继续排序
        quickSelect(nums, gt + 1, right, k);
    }
    // 如果k在等于pivot的范围内，则已经找到前k小的数!!!! 所以这里直接退出了
}

private void swap(int[] nums, int i, int j) {
    int temp = nums[i];
    nums[i] = nums[j];

```

[https://leetcode.cn/problems/sort-colors/description/](https://leetcode.cn/problems/sort-colors/description/)

**快速排序法**

本题也是 "荷兰国旗" 问题。它其实是三向切分快速排序的一种变种，在三向切分快速排序中，每次切分都将数组分成三个区间：小于切分元素，等于切分元素，大于切分元素，而该算法是将数组分成三个区间：等于红色，等于白色，等于蓝色。

代码如下，也是类似**三向切分快速排序**的写法：

```
public void sortColors(int[] nums) {
    if (nums == null || nums.length <= 1) {
        return;
    }

    int left = 0;          // 0 的右边界(相当于0)
    int right = nums.length - 1; // 2 的左边界(相当于2)
    int i = 0;             // 当前遍历指针(相当于1)

    while (i <= right) {
        if (nums[i] == 0) {
            // 当前是0, 交换到左区, 并同时移动左指针和当前指针
            swap(nums, left, i);
            left++;
            i++;
        } else if (nums[i] == 2) {
            // 当前是2, 交换到右区, 这里仅移动右指针(因为交换后的元素需要重新检查)
            swap(nums, i, right);
            right--;
        } else {
            // 当前是1, 直接移动当前指针
            i++;
        }
    }
}

private void swap(int[] nums, int i, int j) {
    int temp = nums[i];
    nums[i] = nums[j];
    nums[j] = temp;
}

### 前 K 个高频元素(347-中等) 💯

给定一个非空的整数数组。返回其中出现频率前 k 高的元素。

```
输入: nums = [1,1,1,2,2,3], k = 2   输出: [1,2]
```

[https://leetcode.cn/problems/top-k-frequent-elements/description/](https://leetcode.cn/problems/top-k-frequent-elements/description/)

**优先级队列(堆)**

主要涉及到如下三块内容：要统计元素出现频率，对频率排序，找出前 K 个高频元素。

首先统计元素出现的频率，这一类的问题可以使用 map 来进行统计。然后是对频率进行排序，这里可以使用优先级队列。

使用小顶堆还是大顶堆？要用小顶堆，因为要统计最大前 k 个元素，只有小顶堆每次将最小的元素弹出。最后小顶堆里积累的才是前 k 个最大元素。

寻找前 k 个最大元素流程如图所示：(图中的频率只有三个，所以正好构成一个大小为 3 的小顶堆，如果频率更多一些，则用这个小顶堆进行扫描)。

![](/zimg/yuque-1777880829768-8.png)

使用 map 记录每个元素出现的频率。构造一个小顶堆，逐一将元素及其频率信息入堆中。当堆的元素等于 K 个时，每次入堆需要判断堆顶的频次与当前的频次的大小，如果大于堆顶的频次，则需要弹出堆顶，并将当前元素入堆，也就是整体上一直保持堆的元素个数不超过 K。

代码如下：

```
public int[] topKFrequent(int[] nums, int k) {
    // 统计每个元素出现的次数
    Map<Integer, Integer> num2Frequency = new HashMap<>();
    for (int num : nums) {
        num2Frequency.put(num, num2Frequency.getOrDefault(num, 0) + 1);
    }

    // 在堆中存储二元组(num, cnt), cnt表示元素值num的出现次数
    // 出现次数按从队头到队尾的顺序是从小到大排, 出现次数最低的在队头(相当于小顶堆)
    PriorityQueue<int[]> minHeap = new PriorityQueue<>((o1, o2) -> o1[1] - o2[1]);

    // 小顶堆只需要维持k个元素有序
    for (Map.Entry<Integer, Integer> entry : num2Frequency.entrySet()) {
        // 元素加入堆中
        minHeap.add(new int[]{entry.getKey(), entry.getValue()});
        if (minHeap.size() > k) {
            // 堆内元素个数大于 k 个时, 弹出堆顶元素
            minHeap.poll();
        }
    }

    int[] res = new int[k];
    // 依次弹出堆中所有元素, 由于是小顶堆所以先弹出的是出现次数少的
    for (int i = k - 1; i >= 0; i--) {
        res[i] = minHeap.poll()[0];
    }

    return res;
}
```

**桶排序**

设置若干个桶，每个桶存储出现频率相同的数，并且桶的下标代表桶中数出现的频率，即第 i 个桶中存储的数出现的频率为 i。

把数都放到桶之后，从后向前遍历桶，最先得到的 k 个数就是出现频率最多的的 k 个数。

![](/zimg/yuque-1777880829798-9.png)

代码如下：

```
public int[] topKFrequent(int[] nums, int k) {
    // 统计每个元素出现的次数
    Map<Integer, Integer> num2Frequency = new HashMap<>();
    for (int num : nums) {
        num2Frequency.put(num, num2Frequency.getOrDefault(num, 0) + 1);
    }

    // 构建桶, 把出现频率相同的元素放在同一个桶里
    List<Integer>[] buckets = new ArrayList[nums.length + 1];
    for (Map.Entry<Integer, Integer> entry : num2Frequency.entrySet()) {
        int frequency = entry.getValue();
        if (buckets[frequency] == null) {
            buckets[frequency] = new ArrayList<>();
        }
        buckets[frequency].add(entry.getKey());
    }

    // 从后往前遍历桶(频次高的在后面), 依次取出前k个高频元素
    List<Integer> topK = new ArrayList<>();
    for (int i = buckets.length - 1; i >= 0 && topK.size() < k; i--) {
        if (buckets[i] != null) {
            if (buckets[i].size() <= (k - topK.size())) {
                topK.addAll(buckets[i]);
            } else {
                topK.addAll(buckets[i].subList(0, k - topK.size()));
            }
        }
    }

    // 将 List<Integer> 转换为 int[]
    int[] result = new int[topK.size()];
    for (int i = 0; i < topK.size(); i++) {
        result[i] = topK.get(i);
    }

    return result;
}
```

**堆排序法**

也是统计每个字符出现的频率，然后全部加入到一个大顶堆中，这样堆顶元素是出现频率最高的。这里堆中也需要加入一个 `int[]` 对象，其中第一个元素为字符，第二个元素为该字符出现的次数。所以大顶堆的排序比较器为 `(o1, o2) -> o2[1] - o1[1]`。

```
public String frequencySort(String s) {
    // 先统计每个字符出现的次数
    Map<Character, Integer> char2Frequency = new HashMap<>();
    for (char num : s.toCharArray()) {
        char2Frequency.put(num, char2Frequency.getOrDefault(num, 0) + 1);
    }

    // 维护一个大顶堆 堆顶元素是出现频率最高的
    // 在堆中存储二元组(num, cnt), cnt表示元素值num的出现次数
    PriorityQueue<int[]> maxHeap = new PriorityQueue<>((o1, o2) -> o2[1] - o1[1]);

    // 所有元素加入堆中
    for (Map.Entry<Character, Integer> entry : char2Frequency.entrySet()) {
        maxHeap.add(new int[]{entry.getKey(), entry.getValue()});
    }

    StringBuilder builder = new StringBuilder();
    // 依次弹出堆中所有元素, 由于是大顶堆所以先弹出的是出现次数多的
    while (!maxHeap.isEmpty()) {
        int[] entry = maxHeap.poll();
        for (int i = 0; i < entry[1]; i++) {
            builder.append((char) entry[0]);
        }
    }

    return builder.toString();
}
```

**桶排序**

与 [前 K 个高频元素(347-中等)](https://www.yuque.com/daerwendecha/qwuv92/ugmcbfc89pfz82wz#FBGFk) 的思路基本一致，也是统计各个字符的出现频次并排序，然后按照出现次数从大到小构造字符串。

```
public String frequencySort(String s) {
    // 统计每个字符出现的频次
    Map<Character, Integer> num2Frequency = new HashMap<>();
    for (char c : s.toCharArray()) {
        num2Frequency.put(c, num2Frequency.getOrDefault(c, 0) + 1);
    }

    List<Character>[] buckets = new ArrayList[s.length() + 1];
    for (char c : num2Frequency.keySet()) {
        int f = num2Frequency.get(c);
        if (buckets[f] == null) {
            buckets[f] = new ArrayList<>();
        }
        buckets[f].add(c);
    }

    StringBuilder builder = new StringBuilder();

    // 从后往前遍历桶, 这样可以保证频次高的字符在前面
    for (int i = buckets.length - 1; i >= 0; i--) {
        if (buckets[i] == null) {
            continue;
        }
        for (char c : buckets[i]) {
            for (int j = 0; j < i; j++) {
                builder.append(c);
            }
        }
    }

    return builder.toString();
}

### 数据流的中位数(295-困难) 💯🔑

中位数是有序整数列表中的中间值。如果列表的大小是偶数，则没有中间值，中位数是两个中间值的平均值。如果从数据流中读出奇数个数值，那么中位数就是所有数值排序之后位于中间的数值。如果从数据流中读出偶数个数值，那么中位数就是所有数值排序之后中间两个数的平均值。

实现 `MedianFinder` 类：

- `MedianFinder()` 初始化 `MedianFinder` 对象。
- `void addNum(int num)` 将数据流中的整数 num 添加到数据结构中。
- `double findMedian()` 返回到目前为止所有元素的中位数。

[https://leetcode.cn/problems/find-median-from-data-stream/description/](https://leetcode.cn/problems/find-median-from-data-stream/description/)

**解题思路**

**建立一个小顶堆 minHeap 保存较大的一半元素，一个大顶堆 maxHeap 保存较小的一半元素。这里可能出现奇数个元素，因此设定 minHeap 可以比 maxHeap 多一个元素**。

`addNum(num)`：

- 当 `m == n`(即 N 为偶数)：需向 minHeap 添加一个元素。实现方法：将新元素 num 插入至 maxHeap，再将 maxHeap 堆顶元素插入至 minHeap。
- 当 `m != n`(即 N 为奇数)：需向 maxHeap 添加一个元素。实现方法：将新元素 num 插入至 minHeap，再将 minHeap 堆顶元素插入至 maxHeap。

`findMedian()`：

- 当 `m == n`(即 N 为偶数)：则中位数为 `(minHeap 的堆顶元素 + maxHeap 的堆顶元素) / 2`。
- 当 `m != n`(即 N 为奇数)：则中位数为 `minHeap 的堆顶元素`(因为 minHeap 多一个元素)。

![](/zimg/yuque-1777880829819-10.png)

代码如下：

```
class MedianFinder {

    private Queue<Integer> minHeap;
    private Queue<Integer> maxHeap;

    public MedianFinder() {
        // 小顶堆 保存较大的一半数据(假设小顶堆可以多放一个数据)
        minHeap = new PriorityQueue<>();
        // 大顶堆 保存较小的一半数据
        maxHeap = new PriorityQueue<>((x, y) -> (y - x));
    }

    public void addNum(int num) {
        // 两个堆元素个数不相等时, 先将num放入小顶堆, 然后将小顶堆的最小值弹出放入大顶堆
        if (minHeap.size() != maxHeap.size()) {
            minHeap.add(num);
            int aMin = minHeap.poll(); // 弹出小顶堆的最小值放到大顶堆中
            maxHeap.add(aMin);
        } else {
            // 小顶堆可以多放一个数据
            maxHeap.add(num);
            int bMax = maxHeap.poll(); // 弹出大顶堆的最大值放到小顶堆中
            minHeap.add(bMax);
        }
    }

    public double findMedian() {
        return minHeap.size() != maxHeap.size() ? minHeap.peek() : (minHeap.peek() + maxHeap.peek()) / 2.0;
    }
}
```

### 数组中的逆序对(困难) 🔑

在数组中的两个数字，如果前面一个数字大于后面的数字，则这两个数字组成一个逆序对。输入一个数组，求出这个数组中的逆序对的总数。

```
输入: record = [9, 7, 5, 4, 6]   输出: 8
解释: 逆序对为 (9, 7), (9, 5), (9, 4), (9, 6), (7, 5), (7, 4), (7, 6), (5, 4).
```

[https://leetcode.cn/problems/shu-zu-zhong-de-ni-xu-dui-lcof/description/](https://leetcode.cn/problems/shu-zu-zhong-de-ni-xu-dui-lcof/description/)

**归并排序法**

归并排序与逆序对是息息相关的。归并排序体现了 “分而治之” 的算法思想，具体为：

- 分：不断将数组从中点位置划分开(即二分法)，将整个数组的排序问题转化为子数组的排序问题。
- 治：划分到子数组长度为 1 时，开始向上合并，不断将较短排序数组合并为较长排序数组，直至合并至原数组时完成排序。

示例过程：

![](/zimg/yuque-1777880829835-11.png)

合并阶段本质上是合并两个排序数组的过程，而每当遇到左子数组当前元素 > 右子数组当前元素时，意味着「左子数组当前元素 至 末尾元素」与「右子数组当前元素」构成了若干「逆序对」。因此，考虑在归并排序的合并阶段统计「逆序对」数量，完成归并排序时，也随之完成所有逆序对的统计。

```
private int[] record, tmp;

public int reversePairs(int[] record) {
    this.record = record;
    tmp = new int[record.length];
    return mergeSort(0, record.length - 1);
}

private int mergeSort(int left, int right) {
    // 终止条件
    if (left >= right) {
        return 0;
    }

    // 递归划分
    int mid = (left + right) / 2;
    int res = mergeSort(left, mid) + mergeSort(mid + 1, right);

    // 合并阶段
    int i = left, j = mid + 1;

    for (int k = left; k <= right; k++) {
        tmp[k] = record[k];
    }

    for (int k = left; k <= right; k++) {
        if (i == mid + 1) {
            record[k] = tmp[j++];
        } else if (j == right + 1 || tmp[i] <= tmp[j]) {
            record[k] = tmp[i++];
        } else {
            record[k] = tmp[j++];
            // 统计逆序对数量
            res += mid - i + 1;
        }
    }

    return res;
}
```

### 最多可以参加的会议数(1353-中等) 💯

给一个数组 events，其中 `events[i] = [startDayi, endDayi]`，表示会议 i 开始于 `startDayi`，结束于 `endDayi`。可以在满足 `startDayi <= d <= endDayi` 中的任意一天 d 参加会议 i。在任意一天 d 中只能参加一场会议，返回可以参加的最大会议数目。

![](/zimg/yuque-1777880829854-12.png)

```
输入：events = [[1,2],[2,3],[3,4]]   输出：3
解释：可以参加所有的三个会议。安排会议的一种方案如上图。
第 1 天参加第一个会议。
第 2 天参加第二个会议。
第 3 天参加第三个会议。
```

[https://leetcode.cn/problems/maximum-number-of-events-that-can-be-attended/description/](https://leetcode.cn/problems/maximum-number-of-events-that-can-be-attended/description/)

**堆排序法**

首先**根据会议的开始时间将所有会议排序，目的是先处理开始时间早的会议**。

**然后使用一个优先队列（最小堆）来存储会议的结束时间，确保每次从队列中取出的都是最早结束的会议。这有助于在时间冲突时，尽量选取能参加的会议**。

主循环：当还有未处理的会议或者优先队列中还有未结束的会议进行以下操作：

- 如果队列为空，则将当前会议的结束时间加入队列，并将 day 设置为当前会议的开始时间。
- 将所有在 day 及之前开始的会议的结束时间加入优先队列。
- 检查队列中最早结束的会议是否在当前天或之后结束，如果是，则可以参加这个会议，并将 day 增加一天，同时 res 计数增加一。
- 无论是否参加会议，都移除优先队列中最早结束的会议。

代码如下：

```
public int maxMeetings(int[][] meetings) {
    // 按照每个会议的开始时间(meetings[i][0])对会议进行排序
    Arrays.sort(meetings, Comparator.comparingInt(a -> a[0]));

    // 优先队列用来存储会议的结束时间，最小堆（默认行为）确保最先处理最早结束的会议
    PriorityQueue<Integer> queue = new PriorityQueue<>();

    // day 表示当前处理的天数，id 表示当前处理的会议，res 表示参加的会议数
    int day = 0, id = 0, res = 0;

    // 当会议列表还有会议或优先队列不为空时继续执行
    while (id < meetings.length || !queue.isEmpty()) {
        // 如果队列为空，则将当前会议的结束时间加入队列，并将 day 设置为当前会议的开始时间
        if (queue.isEmpty()) {
            queue.add(meetings[id][1]);
            day = meetings[id++][0];
        }

        // 将所有在当前天 day 或之前开始的会议的结束时间加入优先队列
        while (id < meetings.length && meetings[id][0] <= day) {
            queue.add(meetings[id++][1]);
        }

        // 如果队列中最早的结束时间不小于当前天数，则可以参加这个会议
        if (queue.peek() >= day) {
            day++; // 参加会议，天数向后推进一天
            res++; // 参加会议数加一
        }

        // 无论是否参加会议，都移除优先队列中的最早结束时间
        queue.poll();
    }

    // 返回最多可以参加的会议数
    return res;
}
```

### 查找和最小的 K 对数(373-中等) 💯

给定两个以 非递减顺序排列 的整数数组 nums1 和 nums2，以及一个整数 k。定义一对值 `(u,v)`，其中第一个元素来自 nums1，第二个元素来自 nums2。请找到和最小的 k 个数对。

```

解释: 返回序列中的前 3 对数：[1,2],[1,4],[1,6],[7,2],[7,4],[11,2],[7,6],[11,4],[11,6]

```

[https://leetcode.cn/problems/find-k-pairs-with-smallest-sums/description](https://leetcode.cn/problems/find-k-pairs-with-smallest-sums/description)

**堆排序法**

**使用一个大顶堆，这样堆内就只存储最小的 k 个元素，堆顶的就是最大的元素**。然后遍历两个数组，构造出数对，并决定是否往堆里面加入。如果堆中元素小于 k 则直接加入堆中。否则将当前数对与堆顶的数对进行对比，如果当前数对更大，由于是排序的数组，则直接终止比较即可。否则将当前数对加入堆中，并弹出堆顶元素即可。如果数组无序，可以先排个序再按照此法处理。

```
public List<List<Integer>> kSmallestPairs(int[] nums1, int[] nums2, int k) {
    // 构造大顶堆(核心就是排序的方法!!!)
    Queue<List<Integer>> minHeap = new PriorityQueue<>((a, b) -> b.get(0) + b.get(1) - a.get(0) - a.get(1));

    // 遍历两个数组入堆
    for (int num1 : nums1) {
        for (int num2 : nums2) {
            // 构造一个数对
            minHeap.add(Arrays.asList(num1, num2));

            if (minHeap.size() > k) {
                // 弹出堆顶元素
                minHeap.poll();
                // 剪枝操作: 因为数组升序, 如果当前数对的和已经大于等于堆顶元素的和, 则不需要再继续比较
                List<Integer> top = minHeap.peek();
                if (num1 + num2 >= top.get(0) + top.get(1)) {
                    break;
                }
            }
        }
    }

    // 遍历堆元素, 提取结果集
    List<List<Integer>> resList = new ArrayList<>();
    while (!minHeap.isEmpty()) {
        List<Integer> tmp = minHeap.poll();
        resList.add(new ArrayList<>(tmp));
    }

    return resList;
}
```
