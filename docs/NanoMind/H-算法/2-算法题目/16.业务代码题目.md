## 多线程

### ⭐ 两个线程分别打印奇偶数

使用 `Object` 的 `wait()` 和 `notify()` 方法实现。

```
public class PrintOddAndEven {

    private static final Object lock = new Object();
    private static int number = 1;
    private static final int MAX_NUMBER = 100;

    public static void main(String[] args) {
        // 创建两个线程
        Thread oddThread = new Thread(new OddPrinter());
        Thread evenThread = new Thread(new EvenPrinter());

        oddThread.start();
        evenThread.start();
    }

    // 写一个实现了Runnable接口的类
    static class OddPrinter implements Runnable {
        // 覆写run方法
        @Override
        public void run() {
            while (number <= MAX_NUMBER) {
                // 加锁
                synchronized (lock) {
                    if (number % 2 == 0) {
                        try {
                            // 进行等待
                            lock.wait();
                        } catch (InterruptedException e) {
                            e.printStackTrace();
                        }
                    }
                    // 打印奇数
                    System.out.println(number);
                    number++;
                    // 通知其他线程抢锁
                    lock.notify();
                }
            }
        }
    }

    static class EvenPrinter implements Runnable {
        @Override
        public void run() {
            while (number <= MAX_NUMBER) {
                // 加锁
                synchronized (lock) {
                    if (number % 2 == 1) {
                        try {
                            lock.wait();
                        } catch (InterruptedException e) {
                            e.printStackTrace();
                        }
                    }
                    System.out.println(number);
                    number++;
                    lock.notify();
                }
            }
        }
    }

}
```

也可以使用 `ReentrantLock` + `Condition` 实现。

```
public class SequentialPrinterWithLock {
    private static final Lock lock = new ReentrantLock();
    // 一个锁创建两个Condition
    private static final Condition oddCondition = lock.newCondition();
    private static final Condition evenCondition = lock.newCondition();
    private static int number = 1;
    private static final int MAX_NUMBER = 100;

    public static void main(String[] args) {
        Thread oddThread = new Thread(new OddPrinter());
        Thread evenThread = new Thread(new EvenPrinter());

        oddThread.start();
        evenThread.start();
    }

    static class OddPrinter implements Runnable {
        @Override
        public void run() {
            while (number < MAX_NUMBER) {
                // 显式加锁
                lock.lock();
                try {
                    if (number % 2 == 0) {
                        // 条件等待
                        oddCondition.await();
                    }

                    System.out.println(number);
                    number++;
                    // 通知另一个线程开放
                    evenCondition.signal();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    lock.unlock();
                }
            }
        }
    }

    static class EvenPrinter implements Runnable {
        @Override
        public void run() {
            while (number < MAX_NUMBER) {
                lock.lock();
                try {
                    if (number % 2 != 0) {
                        evenCondition.await();
                    }

                    System.out.println(number);
                    number++;
                    oddCondition.signal();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                } finally {
                    lock.unlock();
                }
            }
        }
    }
}
```

### ⭐ 三个线程分别打印 123 直到 100

三个线程果断选择用 `ReentrantLock` + `Condition` 实现。其实就是给每个线程分配一个 Condition，如果这个线程需要等待，则调用对应 Condition 的 `await()` 方法，如果需要唤醒某个线程则调用对应 Condition 的 `signal()` 方法。

```
public class PrintOneTwoThreeCondition {

    private static final int WORKER_COUNT = 3;
    private static int countIndex = 0;
    private static final ReentrantLock lock = new ReentrantLock();
    // Condition列表：一个线程一个Condition
    private static final List<Condition> conditions = new ArrayList<>();

    public static void main(String[] args) {
        for (int i = 0; i < WORKER_COUNT; i++) {
            // 为每一个线程分配一个condition
            Condition condition = lock.newCondition();
            conditions.add(condition);
            // 开启线程
            new Thread(new Printer(i)).start();
        }
    }

    // 打印类实现Runnable接口
    static class Printer implements Runnable {

        // 传入当前需要打印的索引
        int index;

        public Printer(int numIndex) {
            this.index = numIndex;
        }

        @Override
        public void run() {
            while (true) {
                // 加锁
                lock.lock();
                try {
                    // 如果当前线程不满足打印条件, 则当前线程等待!!
                    if (countIndex % 3 != index) {
                        conditions.get(index).await();
                    }

                    // 已经打印完了唤醒下一个线程保证正常退出
                    if (countIndex > 100) {
                        signalNext();
                        // 退出循环 线程运行结束
                        return;
                    }

                    System.out.println(countIndex);
                    // 计数器+1
                    countIndex++;
                    // 通知下一个线程干活!!!
                    signalNext();
                } catch (Exception e) {
                    e.printStackTrace();
                } finally {
                    lock.unlock();
                }
            }
        }

        private void signalNext() {
            // 找到下一个需要唤醒的索引-这里都是通过Condition的索引来做的
            int nextIndex = (index + 1) % conditions.size();
            conditions.get(nextIndex).signal();
        }
    }

}
```

### 五个线程 A 先执行完成后再并发执行 BCD 最后执行 E

使用两个 `CountDownLatch`。

```
public class CountDownABCDE {

    private static CountDownLatch latchA = new CountDownLatch(1);      // 控制bcd等待a完成
    private static CountDownLatch latchBCD = new CountDownLatch(3);    // 控制e等待bcd完成

    public static void main(String[] args) throws InterruptedException {
        // 线程A
        Thread a = new Thread(() -> {
            System.out.println("A start");
            // do something...
            System.out.println("A done");
            // 通知bcd线程可以开始了
            latchA.countDown();
        });

        // 线程BCD, 逻辑参考：BCDRunnable
        Thread b = new Thread(new BCDRunnable("B"));
        Thread c = new Thread(new BCDRunnable("C"));
        Thread d = new Thread(new BCDRunnable("D"));

        // 线程E
        Thread e = new Thread(() -> {
            try {
                // 等bcd全部完成
                latchBCD.await();
                System.out.println("E start");
                // do something...
                System.out.println("E done");
            } catch (InterruptedException ex) {
                ex.printStackTrace();
            }
        });

        // 启动所有线程
        a.start();
        b.start();
        c.start();
        d.start();
        e.start();
    }

    // BCD线程的执行逻辑
    static class BCDRunnable implements Runnable {

        private String name;
        public BCDRunnable(String name) {
            this.name = name;
        }

        @Override
        public void run() {
            // 都等待A完成
            try {
                latchA.await();
            } catch (InterruptedException e) {
                throw new RuntimeException(e);
            }

            System.out.println(name + " start");
            // do something...
            System.out.println(name + " end");
            // B、C、D完成：减少计数
            latchBCD.countDown();
        }
    }

}
```

### 10 个线程模拟赛马，所有马就绪后才能开跑，所有马到达终点后裁判宣布赛马成绩

```
public class HorseRace {
    // 用于存储赛马成绩
    private static List<String> results = Collections.synchronizedList(new ArrayList<>());

    // 用于控制所有马同时开跑
    private static CountDownLatch startLatch = new CountDownLatch(1);

    public static void main(String[] args) {
        // 初始化十匹马并启动线程
        for (int i = 1; i <= 10; i++) {
            new Thread(new Horse("Horse " + i)).start();
        }

        // 所有马就绪后开跑
        try {
            System.out.println("All horses ready. Race starts now!");
            // 控制所有马一起开跑
            startLatch.countDown();

        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    // 用于确保所有马到达终点后再宣布成绩
    private static CyclicBarrier finishBarrier = new CyclicBarrier(10, new Runnable() {
        @Override
        public void run() {
            // 裁判宣布成绩
            System.out.println("比赛完成. 成绩如下: ");
            for (String result : results) {
                System.out.println(result);
            }
        }
    });

    static class Horse implements Runnable {
        private String name;

        public Horse(String name) {
            this.name = name;
        }

        @Override
        public void run() {
            try {
                // 马就绪, 等待开跑信号也就是CountDownLatch执行完成
                startLatch.await();

                // 马开始跑(模拟跑的时间)
                long raceTime = (long) (Math.random() * 100);
                Thread.sleep(raceTime);

                // 马到达终点
                results.add(name + " finished in " + raceTime + " ms");

                // 等待其他马到达终点
                finishBarrier.await();
            } catch (Exception e) {
                e.printStackTrace();
            }
        }
    }
}
```

### ⭐️ 并发调三个方法，只要有一个成功就立即成功，否则等都失败才失败

使用 `CompletionService` 实现。

```
public class TaskAllSuccess {

    public static void main(String[] args) {
        System.out.println(isTaskSuccess());
    }

    private static final List<String> taskNames = Arrays.asList("task1", "task2", "task3", "task4", "task5");

    private static boolean isTaskSuccess() {
        // 创建一个线程池
        ScheduledExecutorService executorService = new ScheduledThreadPoolExecutor(
            20, new BasicThreadFactory.Builder().namingPattern("multi-black-list-decision-%d").build());

        // 定义一个CompletionService, 返回值为boolean类型
        CompletionService<Boolean> completionService = new ExecutorCompletionService<>(executorService);

        // 把要执行的任务提交给completionService
        for (String taskName : taskNames) {
            completionService.submit(new Task(taskName));
        }

        try {
            int tasks = taskNames.size();
            // 再循环中不断尝试get返回结构
            while (tasks > 0) {
                // 获取结果
                Future<Boolean> future = completionService.take();
                boolean result = future.get();
                // 拿到一个结果后就判断是否为true(只要有一个为true直接返回true)
                if (result) {
                    return true;
                }

                tasks--;
            }
            // 都执行完之后, 没有true, 则最终返回一次false
            return false;
        } catch (InterruptedException | ExecutionException e) {
            return false;
        }
    }

    static class Task implements Callable<Boolean> {
        private String name;

        public Task(String name) {
            this.name = name;
        }

        @Override
        public Boolean call() throws Exception {
            return name.equals("task1");
        }
    }

}
```

### 并发调 ABC 三个方法，完成后再执行最后一个 D 方法

```
public class CompletableFutureDemo {

    // 模拟异步服务调用
    public static CompletableFuture<String> serviceA() {
        return CompletableFuture.supplyAsync(() -> {
            try {
                Thread.sleep(1000); // 模拟耗时操作
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            return "Result from ServiceA";
        });
    }

    public static CompletableFuture<String> serviceB() {
        return CompletableFuture.supplyAsync(() -> {
            try {
                Thread.sleep(1500); // 模拟耗时操作
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            return "Result from ServiceB";
        });
    }

    public static CompletableFuture<String> serviceC() {
        return CompletableFuture.supplyAsync(() -> {
            try {
                Thread.sleep(800); // 模拟耗时操作
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            return "Result from ServiceC";
        });
    }

    public static void serviceD(String combinedResult) {
        System.out.println("All services completed! Combined results: " + combinedResult);
        System.out.println("Now executing ServiceD...");
        // 执行 serviceD 的逻辑
    }

    public static void main(String[] args) throws ExecutionException, InterruptedException {
        // 1. 并行调用 serviceA、serviceB、serviceC
        CompletableFuture<String> futureA = serviceA();
        CompletableFuture<String> futureB = serviceB();
        CompletableFuture<String> futureC = serviceC();

        // 2. 使用 allOf 等待所有任务完成
        CompletableFuture<Void> allFutures = CompletableFuture.allOf(futureA, futureB, futureC);

        // 3. 在所有任务完成后，执行 serviceD
        CompletableFuture<String> combinedFuture = allFutures.thenApply(v -> {
            // 获取各个任务的结果
            String resultA = futureA.join();
            String resultB = futureB.join();
            String resultC = futureC.join();

            // 合并结果（这里简单拼接，实际可按业务需求处理）
            return resultA + ", " + resultB + ", " + resultC;
        });

        // 4. 执行 serviceD
        combinedFuture.thenAccept(CompletableFutureDemo::serviceD);

        // 5. 阻塞等待所有任务完成（仅用于演示，实际生产环境可能不需要）
        combinedFuture.get();
    }
}
```

### 实现生产者与消费者模型

**基于 wait 和 notify 实现**

这是基于 JDK 的 `wait()` 和 `notify()` 方法实现的。

可以将生产者和消费者需要的方法写在公共类中。也就是创建一个资源类。

```
static class Storage {
    // 最大容量
    private static final int CAPACITY = 5;
    // 缓存
    private LinkedList<String> linkedList = new LinkedList<>();

    // 生产者方法
    public void produce() throws Exception {
        // 注意这里是对资源linkedList加锁(不能对this加锁)
        synchronized (linkedList) {
            // 容量已满则等待
            while (CAPACITY == linkedList.size()) {
                System.out.println("【生产者】: 仓库已满, 暂时不能执行生产任务!");
                // 生产者线程等待!!!!
                linkedList.wait();
            }
            // 生产一个产品
            linkedList.push("李四");
            System.out.println("【生产者】: 生产了一个产品, 现仓储量为:" + linkedList.size());
            // 通知消费者进行消费
            linkedList.notifyAll();
        }
    }

    // 消费者方法
    public void consume() throws Exception {
        // JDK的notifyAll()解除那些在该对象上调用wait()方法的线程的阻塞状态.
        synchronized (linkedList) {
            // 多线程判断中使用while, 不要使用if否则会出现虚假唤醒问题
            while (linkedList.isEmpty()) {
                System.out.println("【消费者】: 仓库无货, 暂时不能执行消费任务!");
                // 消费者线程等待!!!!
                linkedList.wait();
            }

            // 有产品则消费
            linkedList.pop();
            System.out.println("【消费者】: 消费了一个产品. 现仓储量为:" + linkedList.size());

            // 通知生产者线程进行生产
            linkedList.notifyAll();
        }
    }
}
```

**基于 await 与 signalAll 实现**

这是基于 JUC 中的 `Lock` 下面的 `await()` 和 `signal()` 方法实现。

将公共的属性和方法放在 `Resouce` 类中，在资源类中使用 `Lock` 中的 `lock()` 进行加锁，控制并发操作。使用 `await()` 方法阻塞线程，使用 `signalAll()` 唤醒线程。

资源类：

```
static class Resouce {
    private final int MAX_CAPACITY = 3;
    private final int MIN_CAPACITY = 0;
    private int number = 0;
    // 显式锁
    private final Lock lock = new ReentrantLock();
    private final Condition condition = lock.newCondition();

    // 生产方法
    public void product() {
        try {
            // 显式加锁
            lock.lock();
            // 如果生产的数量大于最大值则阻塞
            while (number >= MAX_CAPACITY) {
                System.out.println("【生产者】: 当前仓库满了, 停止生产");
                // 阻塞等待
                condition.await();
            }
            number++;
            System.out.println("【生产者】: 生产了一个产品\t【现仓储量为】:" + number);
            // 生产完毕唤醒所有等待的线程
            condition.signalAll();
        } catch (InterruptedException e) {
            e.printStackTrace();
        } finally {
            // 释放锁
            lock.unlock();
        }
    }

    // 消费方法
    public void consumer() {
        try {
            // 加锁
            lock.lock();
            // 没有产品可以消费则阻塞等待
            while (number <= MIN_CAPACITY) {
                System.out.println("【消费者】: 当前没有产品可以消费");
                // 阻塞等待
                condition.await();
            }
            number--;
            System.out.println("【消费者】: 消费了一个产品\t【现仓储量为】:" + number);
            // 消费完毕唤醒所有等待的线程
            condition.signalAll();
        } catch (InterruptedException e) {
            e.printStackTrace();
        } finally {
            // 释放锁
            lock.unlock();
        }
    }
}
```

```
static class ProducerThread implements Runnable {
    // 阻塞队列
    private BlockingQueue queue;
    private volatile boolean flag = true;

    // 原子计数器
    private static AtomicInteger count = new AtomicInteger();

    public ProducerThread(BlockingQueue queue) {
        this.queue = queue;
    }

    @Override
    public void run() {
        try {
            System.out.println("生产线程启动...");
            while (flag) {
                System.out.println("正在生产数据....");
                String data = count.incrementAndGet() + "";

                // 将数据存入阻塞队列中
                boolean offer = queue.offer(data, 2, TimeUnit.SECONDS);

                if (offer) {
                    System.out.println("生产者,存入" + data + "到队列中, 成功.");
                } else {
                    System.out.println("生产者,存入" + data + "到队列中, 失败.");
                }

                Thread.sleep(1000);
            }
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            System.out.println("生产者退出线程");
        }
    }

    public void stopThread() {
        this.flag = false;
    }
}
```

```
static class ConsumerThread implements Runnable {
    // 阻塞队列
    private BlockingQueue<String> queue;
    private volatile boolean flag = true;

    public ConsumerThread(BlockingQueue<String> queue) {
        this.queue = queue;
    }

    @Override
    public void run() {
        System.out.println("消费线程启动...");
        try {
            while (flag) {
                System.out.println("消费者, 正在从队列中获取数据...");
                // 从阻塞队列获取元素
                String data = queue.poll(2, TimeUnit.SECONDS);
                if (data != null) {
                    System.out.println("消费者拿到队列中的数据data: " + data);
                    Thread.sleep(1000);
                } else {
                    System.out.println("消费者, 超过2秒未获取到数据...");
                    flag = false;
                }
            }
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            System.out.println("消费者退出线程...");
        }
    }
}
```

**基于普通队列实现**

基于**显示锁的生产者与消费者模型**。注意只有一个仓库，生产者和消费者都往仓库里面放东西。

```
static class Storage {

    // 仓库最大存储量
    private static final int MAX_SIZE = 10;
    // 仓库存储的队列
    private Queue<String> queue = new LinkedList<>();
    // 锁
    private final Lock lock = new ReentrantLock();
    // 仓库满的条件变量
    private final Condition fullCondition = lock.newCondition();
    // 仓库空的条件变量
    private final Condition emptyCondition = lock.newCondition();

    // 生产者
    public void produce(String good) {
        try {
            // 获得锁
            lock.lock();
            while (queue.size() >= MAX_SIZE) {
                System.out.println("仓库已满");
                // 仓库满了生产者等待,满条件等待
                fullCondition.await();
            }
            // 生产者添加货物
            queue.offer(good);
            System.out.println("生产者:" + Thread.currentThread().getName() + "生产产品,库存:" + queue.size());
            // 解锁空条件
            emptyCondition.signalAll();
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            lock.unlock();
        }
    }

    // 消费者方法
    public void consume() {
        try {
            // 获得锁
            lock.lock();
            while (queue.size() == 0) {
                System.out.println("仓库为空");
                // 达成空条件,等待
                emptyCondition.await();
            }
            // 从队列弹出元素消费
            queue.poll();
            System.out.println("消费者:" + Thread.currentThread().getName() + "消费产品,库存:" + queue.size());
            // 释放掉满条件
            fullCondition.signalAll();
        } catch (Exception e) {
            e.printStackTrace();
        } finally {
            lock.unlock();
        }
    }

    // 消费者线程
    static class Producer implements Runnable {
        private Storage storage;

        public Producer(Storage storage) {
            this.storage = storage;
        }

        @Override
        public void run() {
            while (true) {
                try {
                    Thread.sleep(1000);
                    storage.produce("123");
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }
    }

    // 消费者线程
    static class Consumer implements Runnable {
        private Storage storage;

        public Consumer(Storage storage) {
            this.storage = storage;
        }

        @Override
        public void run() {
            while (true) {
                try {
                    Thread.sleep(3000);
                    storage.consume();
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
        }
    }

    public static void main(String[] args) {
        // 注意这里只有一个仓库
        Storage storage = new Storage();

        // 生产者和消费者都往仓库里面放东西,所以传入同一个仓库实例
        Thread p1 = new Thread(new Producer(storage));
        Thread p2 = new Thread(new Producer(storage));
        Thread p3 = new Thread(new Producer(storage));

        Thread c1 = new Thread(new Consumer(storage));
        Thread c2 = new Thread(new Consumer(storage));
        Thread c3 = new Thread(new Consumer(storage));

        p1.start();
        p2.start();
        p3.start();
        c1.start();
        c2.start();
        c3.start();
    }
}
```

### 写一个单例方法

**双重校验锁**实现：

```
public class Singleton {
    // 私有构造方法
    private Singleton() {}
    // volatile关键字保证可见性并禁止指令重排序
    private volatile static Singleton singleton;

    // 方法上不用加锁了
    public static Singleton getSingleton() {
        if (singleton == null) {
            // 初始化时加锁对类对象加锁
            synchronized (Singleton.class) {
                if (singleton == null) {
                    singleton = new Singleton();
                }
            }
        }
        return singleton;
    }
}
```

**为什么加** `**synchronized**`**？**

防止重复创建对象。不加锁可能导致两个线程都判断 singleton 为 null，导致两次创建对象。

**为什么加** `**volatile**`**？**

1. **解决内存可见性问题**。多线程环境下，线程 A 修改 `singleton` 后，线程 B 可能看不到最新的值（**由于 CPU 缓存不一致问题**）。如果不加的情况下线程 A 初始化 `singleton`并退出同步块。线程 B 读取 `singleton`，但由于缓存不一致，可能仍然看到 `null`，导致**重复初始化**。
2. **防止指令重排序问题**。`singleton = new Singleton();` 这行代码在 JVM 中并不是一个原子操作，它实际上分为 3 个步骤：**分配内存空间**（`memory = allocate()`）、**初始化对象**（`ctorInstance(memory)`，调用构造函数）、**将对象的引用指向分配的内存**（`instance = memory`）。由于 **JVM 的指令重排序优化**，步骤 **2 和 3 可能会被交换顺序**，导致这个过程可能变成**先赋值再进行初始化**。问题场景变成：线程 A 进入 `getSingleton()`，发现 `singleton == null`，于是进入同步块。线程 A 执行 `singleton = new Singleton()`，但由于指令重排序，`singleton` 先被赋值（非 `null`），**但对象还未初始化完成**。线程 B 进入 `getSingleton()`，发现 `singleton != null`（因为**线程 A 已经赋值**），**直接返回未完全初始化的对象，导致程序出错**。

### 覆写 equals 方法

写出完美 `equals` 方法的建议(源自 JavaCore)：

- 检测 `this` 与 `otherObject` 是否引用同一对象(地址相同)。
- 检测 `otherObject` 是否为 `null`(`null` 就肯定不是了)。
- 比较 `this` 与 `otherObject` 是否属于同一个类(类型不同也不是)。
- 将 `otherObject` 类型转换为相应的类类型变量。
- 现在开始对所有需要比较的域依次进行比较。使用 `==` 比较基本类型域，使用 `equals` 比较对象域。这里就需要看对不同类自己的情况写。

```
// 以下是Employee类的equals方法
public boolean equals(Object otherObject){
     // 检测this与otherObject是否引用同一对象
    if(this == otherObject) return true;
    // 检测otherObject是否为null
    if(otherObject == null) return false;
    // 比较this与otherObject是否属于同一种类型
    if(this。getClass() ！= otherObject。geClass()) return false;
    // 将otherObject转换为相应的类类型变量：类型转换
    Employee other = (Employee) otherObject;
    // 现在开始对所有需要比较的域进行比较。使用 == 比较基本类型域，使用equals比较对象域。
    return Object。equals(name，other。name) && salary == other。salary && Objects。equals(hireDay，other。hireDay);
}
```
