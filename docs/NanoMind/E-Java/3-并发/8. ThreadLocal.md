## 基础

### 定义

ThreadLocal 即**线程本地变量，\*\***每个线程都有同一个变量的独有拷贝**，如多个线程同时操作的是同一个 ThreadLocal 对象，**但每个线程都有自己独立的值，各个线程对于变量的访问和修改不会影响其他的线程\*\*。

ThreadLocal 用一种**存储变量与线程绑定**的方式，在**每个线程中用自己的 ThreadLocalMap 安全隔离变量**，为解决多线程程序的并发问题提供了一种新的思路，比如为每个线程创建一个独立的数据库连接。所以 ThreadLocal 是一种采用无同步思想解决线程安全问题的方法。ThreadLocal 从理论上讲并不是用来解决多线程并发问题的，因为根本不存在多线程竞争。这是一种**空间换时间**的策略。

因为是线程绑定的，所以在很多场景也被用来实现**线程参数传递**，如 Spring 的 RequestContextHolder。也因为**每个线程拥有自己唯一的 ThreadLocalMap，所以 ThreadLocalMap 是天然线程安全的**。

### 基本方法与使用示例

ThreadLocal 的四个方法：

1. **initialValue**：返回此线程局部变量的**初始值**。
2. **get**：返回**当前线程副本中的值**。首次调用时会创建并初始化副本。
3. **set**：设置**当前线程副本的值为指定值**；许多应用依赖 `initialValue()` 方法设置值。
4. **remove**：**移除此线程局部变量的值**。

```
public class ThreadLocalExample {
    public static void main(String[] args) {

        // 定义一个ThreadLocal对象, 多个线程都能使用
        ThreadLocal threadLocal = new ThreadLocal();

        Thread thread1 = new Thread(() -> {
            threadLocal.set(1);
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
            System.out.println(threadLocal.get());
            threadLocal.remove();	// 移除值
        });

        Thread thread2 = new Thread(() -> {
            threadLocal.set(2);
            threadLocal.remove();	// 移除值
        });

        thread1.start();
        thread2.start();
    }
}
```

### 使用场景

常见使用场景：

- **存储上下文信息**。被广泛用在 Spring 等框架中。可以存储上下文信息，避免在不同代码之间来回传递，简化代码。比如从 Controller 的参数可以传递到很深的方法中，这样可以避免每个方法都加一个入参。比如用户身份信息存储，存储鉴权通过之后的身份信息。
- **实现线程安全**。例如对于 `DateFormat/SimpleDateFormat` 是非线程安全的，实现安全的一种方式是使用锁，另一种方式是每次都创建一个新的对象，还有一种方式就是使用 ThreadLocal，每个线程使用自己的 DateFormat，就不存在线程安全问题了。
- **TraceId 存储**。分布式链路追踪中**存储与本次请求相关的 traceId 信息**。

主要就俩作用：

- **解决并发问题**。
- **在线程中传递数据**，在同一个线程执行过程中，ThreadLocal 的数据一直在，所以可以在前面把数据放到 ThreadLocal 中，在后面取出来用，就可以**避免要把这些数据一直通过参数传递**。

## 核心原理

### ThreadLocal 如何实现线程隔离

ThreadLocal 是线程本地变量，每个线程拥有该变量的独立副本，线程间互不干扰。采用 **空间换时间** 的策略，通过线程绑定变量解决并发问题。

**每个线程都有自己的 ThreadLocalMap，ThreadLocalMap 是 ThreadLocal 的内部静态类，用于存储线程的变量副本，Key 为 ThreadLocal 实例，Value 为存储的值**。

比如下图，**A 和 B 两个线程，各自有一个自己的 ThreadLocalMap，ThreadLocalMap 里面都存了三个相同的 ThreadLocal 对象，只不过这三个 ThreadLocal 对象会各自维护自己的 Value 值，也就是 A B 线程使用的 Random 对象其实是不同的**，也就不会有并发问题了。

**每个线程内部维护一个 ThreadLocalMap，Key 是 ThreadLocal 实例，Value 是存储的值**。线程通过自己的 ThreadLocalMap 访问数据，互不干扰。

### 内存泄露问题

ThreadLocal 内存泄漏的核心问题在于 ThreadLocalMap 的**键值引用关系 和 线程生命周期 的不匹配**。

ThreadLocalMap 的 Entry 结构是导致泄漏的关键：

- **键(Key)**：**弱引用(WeakReference)指向 ThreadLocal 对象**。
- **值(Value)**：**强引用存储实际数据**。

泄漏发生的具体过程：

1. ThreadLocal 被回收：当 ThreadLocal 实例失去强引用(如设置为 null)后，由于**键是弱引用，会在 GC 时被回收**。
2. Entry 键变为 null：**此时 Entry 中的** `**key=null**`**，但 value 仍然强引用着数据对象**。
3. 无法访问这些值：**虽然值还存在，但因为没有 key，无法通过正常 API 访问到这些值**。
4. 线程长期存活：如果线程是线程池中的工作线程，**会一直存活，导致这些 "孤儿" 值永远无法释放**。

这个问题在**线程池场景**下更严重：

- 线程会被复用，生命周期很长。
- **每次任务执行都可能设置新的 ThreadLocal 值**。
- **如果不清理，ThreadLocalMap 会不断积累无用的 Entry**。

Thread 对象如果一直在被使用，比如**在线程池中被重复使用，那么从这条引用链就一直在，那么就会导致 ThreadLocalMap 无法被回收**。

**解决方法(最佳实践)**：**每次使用完 ThreadLocal 实例，都调用它的 remove() 方法，清除 Entry 中的数据**。调用 `remove()` 方法最佳时机是**线程运行结束之前的 finally 代码块**中调用，这样能完全避免操作不当导致的内存泄漏，这种主动清理的方式比惰性删除有效。**对于日常的接口开发而言，由于使用的就是 Tomcat 的线程池**，所以就需要注意！

调用 `remove()` 方法会：通过当前 ThreadLocal 实例找到对应的 Entry，**显式地将 Entry 的 value 置为 null**，将整个 Entry 从 ThreadLocalMap 中移除。这样就能**彻底断开 value 的强引用**，允许 GC 回收对象。

## InheritableThreadLocal 与 TransmittableThreadLocal

**父子线程之间有可能存在传递数据/共享数据的需求**，使用 ThreadLocal 是不行的，因为 ThreadLocal 保存的是每个**线程自己**的副本数据。

**因此可以使用 InheritableThreadLocal 来实现父子线程之间的数据传递。其基本原理就是\*\***创建线程时，会看看父线程中是否有 InheritableThreadLocal，如果有则把它复制到子线程的 InheritableThreadLocal 中\*\*。

**InheritableThreadLocal 是用于父子线程之间参数传递的**，但这种方式有一个问题，那就是**必须要是在主线程中手动创建的子线程才可以**，而现在池化技术非常普遍了，**很多时候线程都是通过线程池进行创建和复用的**，这时候 InheritableThreadLocal 就不行了。

**TransmittableThreadLocal** 是阿里开源的一个方案，这个类继承并加强 InheritableThreadLocal 类。用来实现线程之间的参数传递。

**TransmittableThreadLocal 的一般使用场景：**

1. **分布式跟踪系统或全链路压测(即链路打标)**
2. **日志收集记录系统上下文**
3. **Session 级 Cache**
4. **应用容器或上层框架跨应用代码给下层 SDK 传递信息**

**TransmittableThreadLocal 使用示例：**

简单的父子线程之间参数传递：

```
TransmittableThreadLocal<String> context = new TransmittableThreadLocal<>();
// 在父线程中设置
context.set("value-set-in-parent");
// 在子线程中可以读取，值是"value-set-in-parent"
String value = context.get();
```

在线程池中使用：

```
ExecutorService executorService = ...
// 额外的处理，生成修饰了的对象executorService
executorService = TtlExecutors.getTtlExecutorService(executorService);
TransmittableThreadLocal<String> context = new TransmittableThreadLocal<>();

// 在父线程中设置
context.set("value-set-in-parent");

Runnable task = new RunnableTask();
Callable call = new CallableTask();
executorService.submit(task);
executorService.submit(call);

// Task或是Call中可以读取，值是"value-set-in-parent"
String value = context.get();
```
