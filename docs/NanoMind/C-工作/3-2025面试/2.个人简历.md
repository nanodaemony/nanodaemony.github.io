#### 小猿学练机项目(核心领域负责人/多项目主 R/获得年度 A 绩效) 2023.05-至今

**项目描述**：小猿学练机项目的核心产品是自研的**智能设备**，如墨水屏、安卓平板等。设备搭载了众多学习与练习相关的 APP，给用户提供诸多学习功能场景，如同步练习、AI 精准学、学情诊断等。

**工作内容**：

- **全流程参与系统业务架构调整与重构落地**。参与了基于 DDD 思想的领域建模与业务架构调整工作，将复杂业务拆分为 "练习"、"学习"、“基建” 等核心领域，解决原有架构因 4 年迭代导致的业务逻辑混乱、系统耦合严重、开发效率与质量低下的问题。主要负责了 “练习” 域的系统设计与重构。采用 DDD 思想解决练习场景下业务架构的梳理与建模，将练习域分为 “练习记录”、“批改”、“渲染” 等子领域，采用领域事件（如练习行为事件）的方式实现复杂业务系统之间的解耦，并对服务进行了职责边界划分。同时负责新系统架构开发落地、老系统数据与逻辑迁移、服务治理、新系统压测与性能优化、稳定性保障等工作。重构后开发效率显著提升，子领域内中规模需求平均开发交付周期从 2 周缩短至 1 周，练习流程服务可用性从 2 个 9 提升到 3 个 9 以上。
- **基于服务端进行结构化题目图片渲染优化**。针对墨水屏设备性能较低，存在题目本地渲染（将题目富文本信息渲染为图片）速度较慢的问题，采用基于 Node 进行渲染的服务端渲染优化方案。为保护 Node 渲染服务，使用 MySQL 持久化渲染结果，并使用本地缓存 Caffeine + 分布式 Redis 集群的二级缓存方式实现结果缓存。整体实现 95% 的缓存命中率，从出题到客户端渲染完成的平均耗时由 460ms 降低至 220ms。采用分库分表实现 10 亿级题目渲染图片的存储，并通过 Canal 监听 Binlog 实现题目变更时的实时预渲染。为预防热点数据击穿，实施全量题目及新增热点题目的预渲染策略。当渲染服务异常时自动降级为客户端渲染模式。
- **设计实现设备维修系统**。主导设计了硬件设备故障寄修的全流程维修系统，构建基于状态机的维修工单流转机制，通过标准化状态同步接口（如维修单同步、检测结果同步、支付状态同步等）实现与多家第三方维修工厂的高效对接，实现新增工厂接入开发成本在 1 周内。系统对接了支付中台、物流、短信等系统，并采用 OAuth2 的 client_credentials 模式保障第三方工厂系统的授权与安全调用。采用 RocketMQ 延迟消息取消用户未支付的维修单。用户成功支付维修费用后，通过消息重试+补偿日志表+补偿 JOB 的方式实现本系统与第三方系统的维修订单状态的一致性同步，并通过每周对账机制确保多系统间维修数据一致，系统间状态一致率超 99.9%。
- **实现本地缓存监控组件**。开发本地缓存监控组件，实现对 Guava Cache 和 Caffeine 本地缓存的统一监控与指标采集。组件采用线程安全设计，通过定时自动采集包括缓存命中率、QPS、加载耗时、驱逐数量等多项核心指标，并异步上报至 Prometheus，同时结合 Grafana 实现监控可视化与异常告警。同时利用该组件进行本地缓存治理，发现并解决了 10+ 服务本地缓存的命中率低以及缓存加载时间长的问题。

#### 小猿口算项目(核心研发/多项目主 R) 2021.07-2023.04

**项目描述**：小猿口算是一个学习辅导相关的 APP，基于拍照作业批改、在线练习等核心功能辅助学生学习，APP 日活用户近千万。

**工作内容**：

- **核心拍照检查接口压测与优化**。对拍照检查接口进行全链路压测，通过压测结果进行分析性能瓶颈并进行接口优化。针对压测发现的 CPU 使用率高、GC 频繁、依赖的上游服务响应超时（存在性能瓶颈）等问题，使用快慢接口分服务隔离部署并调整慢批改接口的 Tomcat 线程池参数，并采用预先上传图片的方案代替原有的接口直传 MultipartFile 文件方案，降低请求链路对于图片处理的负载。引入基于 Apache Common Pool 的内存池（64K 对齐），实现对服务内图片内存分配的池化，降低 GC 频率。为应对上游服务超时问题，对接口新增单机限流以应对突发流量，同时引入模型批改超时熔断、视觉识别超时自动切换模型（自研 OCR 降级为 Google OCR）等降级机制，以防止上游服务超时引发雪崩。优化后服务 P95 响应时间从 2.15s 优化至 1.50s，减少了 80% 长尾请求；GC 时长稳定，Full GC 频率降低。
- **负责冬奥集福卡运营活动(近千万用户参与)**。活动包含用户做任务收集冬奥卡碎片、抢万能卡、合成金卡等玩法，成功集齐碎片并合成金卡后可获得实物奖品。抢万能卡活动实现了基于 Redis + Lua 脚本的万能卡库存防超卖方案，通过原子操作确保并发抢卡（峰值 QPS 过万）下库存的准确性，同时使用本地缓存记录库存清空状态，拦截后续的无效请求。用户合成金卡成功后，采用本地消息表 + 消息队列 + 补偿任务等机制实现合卡与创建实物奖品订单的最终一致性。同时建立了相关的业务监控指标，提升活动期间系统的可用性和数据一致性。
- **实现亿级图片转换与 CDN 缓存**。通过搭建图片 CDN 源站服务统一管理图片处理逻辑和缓存策略，解决原始图片直接暴露的安全风险和处理逻辑分散问题。系统根据不同业务场景使用不同策略对资源图片进行实时处理，包括压缩、格式转换、倾斜矫正和水印添加等流程，转换后图片通过多级缓存（CDN 边缘节点 + Redis）以加速访问。通过图片处理过程降级机制与双 CDN 灾备方案（阿里云+腾讯云）保障服务高可用。通过夜间全球 CDN 预热策略，使海外用户图片 P95 加载时长降低 60ms。
