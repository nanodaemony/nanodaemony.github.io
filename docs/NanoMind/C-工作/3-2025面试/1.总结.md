## 项目总结

项目非常重要。一般都会**问自己觉得最 🐂、映像最深刻的项目**。一个好的项目可以吹的东西很多，所以要提前准备项目。

这次的简历写了 7 个项目，其实整体下来，问的最多的就重构的项目，顺带问了题目渲染项目和维修相关项目。口算的项目也只问了第一个。项目应该不需要太多，可以考虑下项目之间的关联性。比如我的重构项目，主要提到四件事：基于 DDD 进行领域建模、进行系统重构的代码实现、老系统的逻辑与数据的迁移、新系统核心接口的压测与服务治理。这里【新系统核心接口的压测与服务治理】又可以引申到题目渲染这个项目上来，就串起来了。

准备项目的方式：

- 看看涉及哪些技术点，可以适当的构造一些场景。
- 一定要想清楚为什么要做这个项目或者需求，原因和动机往往更重要，不然都是白搭。
- 想清楚技术选型的原因，比如为什么用本地消息表+MQ 实现数据一致性，根据业务场景、团队内部的现状、公司的技术栈等来决定技术选型。适用更重要。
- 对于系统架构，考虑情况如果依赖的中间件、数据库、上下游出问题之后，应该怎么应对。比如依赖了 Redis 或者 MQ，那这俩挂了怎么办，是直接报错？还是走降级的思路先记录下来后面再补偿等。
- 思考一下当前系统还有什么可以提升的点，比如有哪些优化方向，系统流量番十倍还需要做什么，如果重新设计会怎么办等等。
- 画一下系统的架构图，比如单个服务的、整个业务的、整个系统的，有的面试官会让画一下架构图。
- 让 DeepSeak 看看可能会问哪些问题，提前准备。
- 面试官喜欢关注：技术选型、系统架构、方案设计、沟通对齐软实力、风险把控、成本预估等。
