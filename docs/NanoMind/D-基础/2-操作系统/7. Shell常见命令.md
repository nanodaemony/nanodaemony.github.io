# 基础命令

## 输出命令

### echo

`echo` 是往终端打印的基本命令。

默认情况下，`echo` 调用后会添加一个换行符。输出字符串时，可以不用引号。

```
$ echo Welcome to bash !
Welcome to bash !
```

可以使用单引号。**单引号中的所有内容都被当成普通字符串输出**。

```
$ echo 'Welcome to bash !'
Welcome to bash !
```

可以使用双引号。**双引号中某些字符会保留其特殊含义**，比如 `"!"`，此时可以加上转义符号。

```
$ echo "Welcome to bash \!"
Welcome to bash !
```

如果输出字符串时**如果有变量替换，则需要使用双引号**。如打印变量值：

```
#!/bin/sh
read name
echo "$name It is a test"
```

结果重定向至文件。

```
echo "It is a test" > myfile
```

## history(历史命令)

历史命令默认存在 `~/.bash_history` 文件中。

历史命令重复调用：

- 使用上下箭头调用。
- 使用 `"!!"` 重复执行上一条历史命令。
- 使用 `"!n"` 重复执行第 `n` 条历史命令(`history` 命令结果中每个命令都有一个编号)。
- 使用 "`!字符串`" 重复执行最后一条以该字符串开头的命令。

## 文件操作命令

### ls 命令

列出文件或者目录的信息，目录的信息就是其中包含的文件。

```
$ ls [-aAdfFhilnrRSt] file|dir
-a : 列出全部文件
-d : 仅列出目录本身
-l : 以长数据串行列出, 包含文件属性与权限等数据
```

### cd 命令

更换当前目录。

```
$ cd [相对路径或绝对路径]
$ cd # 进入/home目录
```

### mkdir 命令

创建目录。

```
$ mkdir [-mp] 目录名称
-m : 配置目录权限
-p : 递归创建目录
```

### rmdir 命令

删除目录，目录必须为空。

```
$ rmdir [-p] 目录名称
-p : 递归删除目录
```

### touch 命令

更新文件时间或者建立新文件。

```
$ touch [-acdmt] filename
-a :  更新 atime
-c :  更新 ctime, 若该文件不存在则不建立新文件
-m :  更新 mtime
-d :  后面可以接更新日期而不使用当前日期, 也可以使用 --date="日期或时间"
-t :  后面可以接更新时间而不使用当前时间, 格式为[YYYYMMDDhhmm]
```

### cp 命令

复制文件。如果源文件有两个以上，则目标文件一定要是目录才行。

`-r` 代表递归拷贝。

### rm 命令

删除文件。

```
$ rm [-fir] 文件或目录
-r : 递归删除
```

递归删除：

```
$ rm -rf folder
```

### mv 命令

移动文件。

```
$ mv [-fiu] source destination
$ mv [options] source1 source2 source3 .... directory
-f :  force 强制移动, 如果目标文件已经存在, 不会询问而直接覆盖
```

## 命令寻找

`which`，`whereis`，`file` 与 `whatis` 命令可以给出文件和目录的相关信息。

### which 命令

**which 命令用来找出某个命令的位置**。

```
$ which ls
/bin/ls
```

当输入命令时，终端会在环境变量 `PATH` 指定的一组目录中搜索并执行所找到的第一个可执行文件。

```
$ echo $PATH
/usr/local/opt/thrift@0.9/bin:/usr/local/opt/thrift@0.9/bin:/usr/local/bin:/usr/local/sbin:/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin:/Users/nano/Desktop/software/fenbi-code-tool/bin
```

根据对 `PATH` 变量的定义，可以直接使用 `/bin`，`/usr/local/bin` 或 `/opt/PACKAGENAME/bin` 目录下的命令。

### whereis 命令

`whereis` 与 `which` 命令类似，它不仅会返回命令的路径，还能够打印出其对应的命令手册以及源代码的路径(如果有的话)。

```
$ whereis ls
ls: /bin/ls /usr/share/man/man1/ls.1.gz
```

### whatis 命令

`whatis` 会输出指定命令的一行简短描述。

## locate 命令(文件搜索)

文件搜索。可以用关键字或正则表达式进行搜索。

`locate` 使用 "`/var/lib/mlocate/`" 这个数据库来进行搜索，它存储在内存中，并且每天更新一次，所以**无法用来** `**locate**` **搜索新建的文件**。可以使用 `updatedb` 来立即更新数据库。

```
$ locate [-ir] keyword
-r: 正则表达式
```

## cat 命令(文件拼接)🌟

`**cat**` **命令能够显示或拼接文件内容**。`cat` 能够将标准输入数据与文件数据组合在一起。通常的做法是将 `stdin` 重定向到一个文件，然后再合并两个文件，而 `cat` 命令一次就能搞定这些操作。

读取文件语法。

```
$ cat file1 file2 file3 ...
```

`cat` 命令将作为命令行选项的文件内容拼接在一起并将结果发送到 `stdout`。

打印单个文件的内容。

```
$ cat file.txt
This is a line inside file.txt
This is the second line inside file.txt
```

打印多个文件的内容。

```
$ cat one.txt two.txt
This line is from one.txt
This line is from two.txt
```

`cat` 命令不仅可以读取文件，拼接数据，还能够从标准输入中读取。管道操作符可以将数据作为 `cat` 命令的标准输入。

```
$ OUTPUT_FROM_SOME COMMANDS | cat
```

## tac 命令(cat 反操作)

是 `cat` 的反向操作，**从最后一行开始打印**。

## tr 命令(字符转换)🌟

`tr` 可以**对来自标准输入的内容进行字符替换，字符删除以及重复字符压缩**。`tr` 是 `translate` (转换)的简写，因为它**可以将一组字符转换成另一组字符**。

`tr` 只能通过 `stdin` (标准输入)接收输入(无法通过命令行参数接收)。其调用格式如下：

```
$ tr [options] set1 set2
```

在 `tr` 中利用集合的概念，可以轻松地将字符从一个集合映射到另一个集合中。

来自 `stdin` 的输入字符会按照位置从 `set1` 映射到 `set2`(set1 中的第一个字符映射到 `set2` 中的第一个字符，以此类推)，然后将输出写入 `stdout` (标准输出)。`set1` 和 `set2` 是字符类或字符组。如果两个字符组的长度不相等，那么 `set2` 会不断复制其最后一个字符，直到长度与 `set1` 相同。如果 `set2` 的长度大于 `set1`，那么在 `set2` 中超出 `set1` 长度的那部分字符则全部被忽略。

例子：将输入中的字符由大写转换成小写。

```
$ echo "HELLO WHO IS THIS" | tr 'A-Z' 'a-z'
hello who is this
```

**用 tr 删除字符(-d)**

选项 `-d` 可以通过指定需要被删除的字符集合，将出现在 `stdin` 中的特定字符清除掉。

```
$ cat file.txt | tr -d '[set]'
```

```
# 将stdin中的数字删除并打印删除后的结果
$ echo "Hello 123 world 456" | tr -d '0-9'
Hello world
```

**压缩字符(-s)**

`-s` 选项可以删除字符串中重复出现的字符，只保留一个。

```
$ tr -s '[需要被压缩的一组字符]'
```

删除多余空格。

```
$ echo "GNU is       not     UNIX. Recursive  right ?" | tr -s ' '
GNU is not UNIX. Recursive right ?
```

**字符类集合**

`tr` 可以将不同的字符类作为集合使用，常用字符类有：

- `alpha`：字母
- `lower`：小写字母
- `upper`：大写字母
- `digit`：数字
- `alnum`：字母和数字
- `cntrl`：控制(非打印)字符
- `space`：空白字符

使用方式：

```
$ tr [:class:] [:class:]
```

删除所有大写字母。

```
$ echo GOT THE Book. | tr -d '[:upper:]'
ook.
```

## sort 命令(行排序)🌟

`sort` 命令能够对文本文件和 `stdin` 进行**排序**。`sort` 命令可以从特定的文件或 `stdin` 中获取输入，并将输出写入 `stdout`。

```
$ sort [-fbMnrtuk] [file or stdin]
-f : 忽略大小写
-b : 忽略最前面的空格
-M : 以月份的名字排序, 如 JAN, DEC
-n : 使用数字
-r : 反向排序
-u : 即 unique 去重, 重复的内容只出现一次
-t : 分隔符, 默认为 tab
-k : 指定排序的区间
```

对一个或一组文件排序。

```
$ cat file1.txt
1
3
5
$ cat file2.txt
8
2
3
# 对多个文件排序后输出到另一个临时文件
$ sort file1.txt file2.txt > sorted.txt
$ cat sorted.txt
1
2
3
3
5
8
```

按照数字顺序排序。

```
$ sort -n file2.txt
2
3
8
```

按照逆序排序。

```
$ sort -r file2.txt
8
3
2
```

合并两个已排序过的文件。

```
$ sort -m sorted1 sorted2
```

**依据键或列排序**

`-k` 指定了排序所依据的字符。如果是单个数字，则指的是列号。`-r` 选项指定按照**逆序**进行排序。

```
$ cat data.txt
1 mac 2000
2 winxp 4000 12
3 bsd 1000
4 linux 1000
```

示例：

```
# 依据第1列, 以逆序形式排序 -nr表明按照数字顺序, 采用逆序形式排序
$ sort -nrk 1 data.txt
4 linux 1000
3 bsd 1000
2 winxp 4000
1 mac 2000
```

```
# 依据第2列进行排序
$ sort -k 2 data.txt
3 bsd 1000
4 linux 1000
1 mac 2000
2 winxp 4000
```

一定要留意用于按数字顺序进行排序的选项 `-n`。`sort` 命令对于字母表排序和数字排序有不同的处理方式。因此如果要采用数字顺序排序，就应该明确地给出 `-n` 选项。

`-k` 选项后的整数指定了文本文件中的某一列。列与列之间由空格分隔。如果需要将特定范围内的一组字符(例如，第 2 列中的第 4~5 个字符)作为键，应该使用由点号分隔的两个整数来定义一个字符位置，然后将该范围内的第一个字符和最后一个字符用逗号连接起来：

```
$ sort -bk 2.3,2.4 data.txt		# 按照c,d,n的顺序排序
1 mac 2000
3 bsd 1000
4 linux 1000
2 winxp 4000 12
```

示例：`/etc/passwd` 文件内容以 `:` 来分隔，要求以第三列进行排序。

```
$ cat /etc/passwd | sort -t ':' -k 3
root:x:0:0:root:/root:/bin/bash
dmtsai:x:1000:1000:dmtsai:/home/dmtsai:/bin/bash
alex:x:1001:1002::/home/alex:/bin/bash
arod:x:1002:1003::/home/arod:/bin/bash
```

## uniq 命令(提取唯一行)🌟

`uniq` 命令可以从给定输入中(`stdin` 或命令行参数指定的文件)**找出唯一的行**，报告或删除那些重复的行。

`uniq` 只能**作用于排过序的数据**!!! 所以 `uniq` 命令经常与 `sort` 一同使用。`uniq` 命令可以从特定的文件或 `stdin` 中获取输入，并将输出写入 `stdout`。

找出已排序文件中不重复的行。

```
$ sort file1.txt file2.txt | uniq
```

提取唯一行：

```
$ cat data.txt
Jack
Lucy
Tom
Alice
Tom
```

```
# 文件无序 输出不变
$ uniq data.txt
Jack
Lucy
Tom
Alice
Tom
```

```
# 先排序 uniq仅作用于排过序的数据, 删掉了重复的行
$ sort data.txt | uniq
Alice
Jack
Lucy
Tom
```

**找出不重复(唯一)的行：**

写法 1。

```
# 先排序
$ sort data.txt > sorted.txt
# Tom重复所以忽略
$ uniq -u sorted.txt
Alice
Jack
Lucy
```

写法 2。

```
$ sort data.txt | uniq -u
Alice
Jack
Lucy
```

找出文件中重复的行：

```
$ sort data.txt | uniq -d
Tom
```

统计各行在文件中出现的次数：

```
$ sort data.txt | uniq -c
1 Alice
1 Jack
1 Lucy
2 Tom
```

取得每个用户的登录总次数：

```
$ last | cut -d ' ' -f 1 | sort | uniq -c
1
6 unknown
47 dmtsai
4 reboot
7 root
1 wtmp
```

指定提取的位置：

可以结合 `-s` 和 `-w` 选项来**指定键**。

- `-s` 指定跳过前 N 个字符。
- `-w` 指定用于比较的最大字符数。

```
$ cat data.txt
u:01:gnu
d:04:linux
u:01:bash
u:01:hack
```

使用 `-s 2` 跳过前两个字符，使用 `-w 2` 选项指定后续的两个字符。因此提取唯一行时按照第 3 和 4 个字符进行提取。

```
$ sort data.txt | uniq -s 2 -w 2
d:04:linux
u:01:bash
```

## split 命令(分割文件)🌟

`split` 命令可以用来**分割文件**。该命令接受文件名作为参数，然后创建出一系列体积更小的文件，其中依据字母序排在首位的那个文件对应于原始文件的第一部分，排在次位的文件对应于原始文件的第二部分，以此类推。

**按文件大小分割**

将 `100KB` 的文件分成一系列 `10KB` 的小文件。

```
$ split -b 10k data.file
$ ls
data.file  xaa  xab  xac  xad  xae  xaf  xag  xah  xai  xaj
```

上面的命令将 `data.file` 分割成了 10 个大小为 `10KB` 的文件。这些新文件以 `xab`，`xac`，x`a`d 的形式命名。`split` 默认使用字母后缀。

如果想使用数字后缀，需要使用 `-d` 选项。此外，`-a length` 可以指定后缀长度：

```
$ split -b 10k data.file -d -a 4
$ ls
data.file x0009  x0019  x0029  x0039  x0049  x0059  x0069  x0079
```

默认分割后的文件名都是以 `x` 作为前缀。如果要分割的文件不止一个就容易分不清，可以指定使用 `split_file` 作为文件名前缀。

```
$ split -b 10k data.file -d -a 4 split_file
$ ls
data.file split_file0002 split_file0005 split_file0008 strtok.c
split_file0000 split_file0003 split_file0006 split_file0009 split_file0001 split_file0004 split_file0007
```

**按文件行数分割**

使用 `-l no_of_lines` 指定根据行数来分割文件。

```
# 分割成多个文件, 每个文件包含10行
$ split -l 10 data.file
```

## 根据扩展名切分文件名

一个文件名为 `name.extension`，则通过 `%` 操作符可以从这种格式中提取 `name` 部分(文件名)，通过 `#` 操作符可以提取出扩展名 `extension` 部分。

```
$ fileName="sample.jpg"
$ name=${fileName%.*}
$ extension=${fileName#*.}
$ echo $name
sample
$ echo $extension
jpg
```

`"${VAR%.*}"` 的含义为：

- 从 `$VAR` 中删除位于 `%` 右侧的通配符(上例中是 `.*`)所匹配的字符串。通配符从右向左进行匹配。
- 给 `VAR` 赋值，即 `VAR=sample.jpg`。通配符从右向左匹配到的内容是 `.jpg`，因此从 `$VAR` 中删除匹配结果，得到输出 `sample`。

`%` 属于非贪婪(non-greedy)操作。它从右向左找出匹配通配符的最短结果。还有另一个操作符 `%%`，它与 `%` 相似，但行为模式却是贪婪的，这意味着它会匹配符合通配符的最长结果。

例子：

```
$ VAR=hack.fun.book.txt
# 使用%操作符从右向左执行非贪婪匹配
$ echo ${VAR%.*}
hack.fun.book
# 使用%%操作符从右向左执行贪婪匹配
$ echo ${VAR%%.*}
hack
```

`#` 操作符可以从文件名中提取扩展名。这个操作符与 `%` 类似，不过求值方向是从左向右。

`"${VAR#*.}"` 的含义为：

- 从 `$VAR` 中删除位于 `#` 右侧的通配符(上例中是 `*.`)从左向右所匹配到的字符串。和 `%%` 类似，`#` 也有一个对应的贪婪操作符 `##`。

例子：

```
$ VAR=hack.fun.book.txt
# 使用#操作符从左向右执行非贪婪匹配
$ echo ${VAR#*.}
fun.book.txt
# 使用##操作符从左向右执行贪婪匹配
$ echo ${VAR##*.}
txt
```

考虑到文件名中可能包含多个 `.` 字符，所以相较于 `#`，`##` 更适合于从中提取扩展名。`##` 执行的是贪婪匹配，因而总是能够准确地提取出扩展名。

上述操作也非常适用于网址或邮件地址的内容提取。

## more 与 less 命令(文件内容读取)

`more` 和 `cat` 不同的是它可以**一页一页查看文件内容**，比较适合查看大文件。

`less` 和 `more` 类似，但多了一个向前翻页的功能。

## head 与 tail 命令(打印文件前后行)🌟

`**cat**` **命令并不适合查看上千行的大文件，因为它会把整个文件内容全部给打印出来**。如果只想**查看文件的一小部分内容，如文件的前 n 行或后 n 行**，也可能是除了前 n 行或后 n 行之外所有的行，亦或是第 m 行至第 n 行。`head` 和 `tail` 命令可以实现这些需求。

`head` 命令总是读取输入文件的起始部分。

打印前 10 行：

```
$ head file
```

从 `stdin` 读取数据：

```
$ cat text | head
```

指定打印前几行：

```
$ head -n 4 file
```

打印除了最后 M 行之外所有的行：这里的 `-M` 表示的是负数，并非选项。

```
$ head -n -M file
```

打印出除最后 5 行之外的所有行。

```
$ seq 11 | head -n -5
1
2
3
4
5
6
```

打印文件最后 10 行：

```
$ tail file
```

打印最后 5 行：

```
$ tail -n 5 file
```

打印除前 5 行之外的所有行：

```
# M+1=6
$ seq 100 | tail -n +6
```

`tail` 命令的一个常见用法是**监视一个内容不断增加的文件**(例如系统日志文件)中出现的新行。为了能够监视文件的增长，`tail` 有一个特殊的选项 `-f` 或 `--follow`，允许 `tail` 关注文件内容的更新并将其显示出来：

```
$ tail -n 1000 -f /var/log/messages
```

## wc 命令(统计数量)🌟

`wc` 是一个用于**统计行，单词和字符数量的实用工具**。它是 Word Count(单词计数)的缩写。

不使用任何选项时，`wc` 会**打印出行，单词和字符的数量**。

```
$ wc file
1435   15763  112200
```

统计行数：

```
$ wc -l file
# 将stdin作为输入
$ cat file | wc -l
```

统计单词数：

```
$ wc -w file
$ cat file | wc -w
```

统计字符数：

```
$ wc -c file
$ cat file | wc -c
```

## grep 命令(文本搜索)🌟

`grep` 可以**定位包含特定文本模式的文件，能够接受正则表达式**，生成各种格式的输出。`grep` 命令默认使用基础正则表达式。也可以使用默认启用扩展正则表达式的 `egrep` 命令。

**搜索匹配特定模式的文本行**

在 `stdin` 中搜索。

```
$ echo -e "this is a word\nnext line" | grep word
this is a word
```

在文件中搜索。

```
$ grep pattern filename
this is the line containing pattern
```

在多个文件中搜索匹配特定模式的文本行。

```
$ grep "match_text" file1 file2 file3 ...
```

**匹配多个模式**

选项 `-e` 可以指定多个匹配模式。

```
$ grep -e "pattern1" -e "pattern2"
```

上述命令会打印出匹配任意一种模式的行，每个匹配对应一行输出。

```
$ echo this is a line of text | grep -o -e "this" -e "line"
this
line
```

**只输出匹配的文本**

选项 `-o` 可以只输出匹配到的文本。

```
$ echo this is a line. | egrep -o "[a-z]+\."
line.
```

**输出不匹配的行**

选项 `-v` 可以打印出不匹配 `Pattern` 的所有行。

```
$ grep -v match_pattern file
```

**统计匹配的文本行数**

选项 `-c` 能够统计出匹配模式的文本行数。

```
$ grep -c "text" filename
```

注意 `-c` 只是统计匹配行的数量，并不是匹配的次数。

```
$ echo -e "1 2 3 4\nhello\n5 6" | egrep -c "[0-9]"
# 只有两个匹配行,在单行中出现的多次匹配只被计为一次
2
```

**统计文件中匹配项的数量**

可以与 `wc` 命令配合统计匹配的数量。

```
$ echo -e "1 2 3 4\nhello\n5 6" | egrep -o "[0-9]" | wc -l
6
```

**忽略大小写**

选项 `-i` 可以在匹配模式时不考虑字符的大小写。

```
$ echo hello world | grep -i "HELLO"
hello
```

**打印匹配文本的前后行**

选项 `-A` 可以打印匹配结果之后的行，选项 `-B` 可以打印匹配结果之前的行，两者也可以一起用(与 `-C` 选项效果一致)。

```
$ seq 10 | grep 5 -A 3
$ seq 10 | grep 5 -B 3
$ seq 10 | grep 5 -C 3
```

## cut 命令(按列切分文件)🌟

`cut` 命令可以按列来**切分文件**。该命令可用于**处理使用固定宽度字段的文件**，`CSV` 文件或是由空格分隔的文件(例如标准日志文件)。

`cut` 命令能够提取指定位置或列之间的字符。**可以指定每列的分隔符，每列被称为一个字段。制表符是字段默认的分隔符，没有使用分隔符的行，会被原样打印出来**。

**指定提取的字段**

选项 `-f` 可以**指定要提取的字段**。`FIELD_LIST` 是需要显示的列。它由列号组成，彼此之间用逗号分隔。

```
$ cut -f FIELD_LIST filename
```

```
# 分隔符为\t
$ cat student.txt
No	Name	Mark	Percent
1	Sarath	45	90
2	Alex	49	98
3	Anu	45	90
```

```
$ cut -f 1 student.txt
No
1
2
3
```

提取多个字段使用逗号分隔字段号。

```
$ cut -f 2,3 student.txt
Name	Mark
Sarath	45
Alex	49
Anu	45
```

`--complement` 选项用于切分没有被 `-f` 选项指定的其他字段。

```
$ cut -f3 --complement student_data.txt
No  Name    Percent
1   Sarath  90
2   Alex    98
3 Anu 90
```

**设置分隔符**

选项 `-d` 能够设置分隔符。

```
$ cat student.txt
No;Name;Mark;Percent
1;Sarath;45;90
2;Alex;49;98
3;Anu;45;90
```

```
$ cut -f 2 -d ";" student.txt
Name
Sarath
Alex
Anu
```

**指定提取字符范围**

`cut` 命令可以根据字节或者字符来指定选择范围。

|       |                                                                               |
| ----- | ----------------------------------------------------------------------------- |
| 符号  | 含义                                                                          |
| `N-`  | 从第 `N` 个字节，字符或字段开始到行尾                                         |
| `N-M` | 从第 `N` 个字节，字符或字段开始到第 `M` 个(包括第 `M` 个在内)字节，字符或字段 |
| `-M`  | 从第 `1` 个字节，字符或字段开始到第 `M` 个(包括第 `M` 个在内)字节，字符或字段 |

相关选项有：

- `-b` 表示字节
- `-c` 表示字符
- `-f` 用于定义字段

例子：

```
$ cat student.txt
abcdefghijklmnopqrstuvwxyz
abcdefghijklmnopqrstuvwxyz
abcdefghijklmnopqrstuvwxyz
abcdefghijklmnopqrstuvwxy
```

打印 2-5 个字符。

```
$ cut -c 2-5 student.txt
bcde
bcde
bcde
bcde
```

## sed 命令(替换文本)🌟

`sed` 是 stream editor(流编辑器)的缩写。它最常见的用法是进行**文本替换**。

`sed` 可以**使用另一个字符串来替换匹配模式，模式可以是简单的字符串或正则表达式**。

```
$ sed 's/pattern/replace_string/' file
```

`sed` 也可以从 `stdin` 中读取输入。

```
$ cat file | sed 's/pattern/replace_string/'
```

例子：

```
$ cat data.txt
This is a book.
Her name is Lucy.
```

```
$ sed 's/is/was/' data.txt
Thwas is a book.
Her name was Lucy
```

注意这里默认只替换第一个匹配的内容。

`g` 标记可以使 `sed` 执行全局替换。

```
$ sed 's/pattern/replace_string/g' file
```

```
$ sed 's/is/was/g' data.txt
Thwas was a book.
Her name was Lucy.
```

**修改原始文件**

选项 `-i` 会使得 `sed` 用修改后的数据替换原始文件。

```
$ sed -i 's/text/replace/' file
```

**删除空行**

空行可以用正则表达式 `^$` 进行匹配。最后的 `/d` 告诉 `sed` 不执行替换操作，而是直接删除匹配到的空行。

```
$ sed '/^$/d' file
```

**已匹配字符串标记(&)**

在 `sed` 中可以用 `&` 指代模式所匹配到的字符串，这样就能够在替换字符串时使用已匹配的内容。

```
$ echo this is an example | sed 's/\w\+/[&]/g'
[this] [is] [an] [example]
```

这里正则表达式 `\w+` 匹配每一个单词，然后用 `[&]` 替换它。`&` 对应于之前所匹配到的单词。

**多次替换**

可以利用管道组合多个 `sed` 命令，多个模式之间也可以用分号分隔，或是使用选项 `-e PATTERN`。

```
# 管道
$ sed 'expression' | sed 'expression'
# 分号
$ sed 'expression; expression'
# -e选项
$ sed -e 'expression' -e 'expression'
```

**变量引用**

`sed` 表达式通常用单引号来引用。不过也可以使用双引号。shell 会在调用 `sed` 前先扩展双引号中的内容。如果想在 `sed` 表达式中使用变量，则需要使用双引号。

```
$ text=hello
$ echo hello world | sed "s/$text/HELLO/"
HELLO world
```

## awk(高级文本处理)🌟

### 基础

`awk` 命令可以处理数据流。它支持关联数组，递归函数，条件语句等功能。

`awk` 脚本的结构如下：

```
$ awk 'BEGIN{ print "start" } pattern { commands } END { print "end" }' file
```

`awk` 脚本通常由 3 部分组成：`**BEGIN**`**，**`**END**` **和带模式匹配选项的公共语句块**。这 3 个部分都是可选的，可以不用出现在脚本中。

**工作方式**

`**awk**` **以\*\***逐行的形式处理文件\***\*。**`**BEGIN**` **之后的命令会先于公共语句块执行。对于匹配** `**PATTERN**` **的行，**`**awk**` **会对其执行** `**PATTERN**` **之后的命令。最后在处理完整个文件之后，awk 会执行 END 之后的命令**。

1. 首先执行 `BEGIN { commands }` 语句块中的语句。
2. 接着从文件或 `stdin` 中读取一行，如果能够匹配 `pattern`，则执行随后的 `commands` 语句块。重复这个过程，直到文件全部被读取完毕。
3. 当读至输入流末尾时，执行 `END { commands }` 语句块。

`BEGIN` 语句块在 `awk` 开始从输入流中读取行之前被执行。这是一个可选的语句块，诸如变量初始化(如 `var=0;`)，打印输出表格的表头等语句通常都可以放在 `BEGIN` 语句块中。

`END` 语句块和 `BEGIN` 语句块类似。它在 `awk` 读取完输入流中所有的行之后被执行。像打印所有行的分析结果这种常见任务都是在 `END` 语句块中实现。

最重要的部分就是和 `pattern` 关联的语句块。这个语句块同样是可选的。如果不提供，则默认执行 `{ print }`，即**打印所读取到的每一行**。`awk` 对于读取到的每一行都会执行该语句块。这就**像一个用来读取行的** `**while**` **循环，在循环体中提供了相应的语句**。每读取一行，`awk` 就会检查该行是否匹配指定的模式。模式本身可以是正则表达式，条件语句以及行范围等。如果当前行匹配该模式，则执行 `{ }` 中的语句。模式是可选的。如果没有提供模式，那么 `awk` 就认为所有行都是匹配的。

`{ }` 就像一个循环体，对文件中的每一行进行迭代。

`awk` 命令也可以从 `stdin` 中读取输入。

简单的 `awk` 脚本可以放在单引号或双引号中：

```
$ awk 'BEGIN { statements } { statements } END { end statements }'
$ awk "BEGIN { statements } { statements } END { end statements }"
```

输出文件行数。

```
$ awk 'BEGIN { i=0 } { i++ } END { print i }' filename
$ awk "BEGIN { i=0 } { i++ } END { print i }" filename
```

例子：

```
$ echo -e "line1\nline2" | awk 'BEGIN { print "Start" } { print } END { print "End" }'
Start
line1
line2
End
```

`print` 能够接受参数。这些参数以逗号分隔，在打印参数时则以空格作为参数之间的分隔符。在 `awk` 的 `print` 语句中，双引号被当作拼接操作符使用。

```
$ echo | awk '{ var1="v1"; var2="v2"; var3="v3"; print var1,var2,var3 ; }'
v1 v2 v3
```

`echo` 命令向标准输出写入一行，因此 `awk` 的 `{ }` 语句块中的语句只被执行一次。如果 `awk` 的输入中包含多行，那么 `{ }` 语句块中的语句也就会被执行相应的次数。

### 特殊变量

`awk` 命令是一个解释器，它能够解释并执行程序，和 shell 一样，它也包括了一些特殊变量。

以下是 `awk` 可以使用的一些特殊变量。

- `NR`：表示记录编号，当 `awk` 将行作为记录时，该变量相当于当前行号。
- `NF`：表示字段数量，在处理当前记录时，相当于字段数量。默认的字段分隔符是空格。
- `$0`：该变量包含当前记录的文本内容。
- `$1`：该变量包含第一个字段的文本内容。
- `$2`：该变量包含第二个字段的文本内容。

可以用 `print $NF` 打印一行中最后一个字段，用 `$(NF-1)` 打印倒数第二个字段，其他字段以此类推。`awk` 也支持 `printf()` 函数，其语法和 C 语言中的同名函数一样。

打印出每一行的第二和第三个字段。

```
$ awk '{ print $3, $2 }' file
```

可以使用 `NR` 统计文件的行数。

```
$ awk 'END{ print NR }' file
```

这里只用到了 `END` 语句块。每读入一行，`awk` 都会更新 `NR`。当到达文件末尾时，`NR` 中的值就是最后一行的行号。

打印第 5 列。

```
$ awk '{ print $5 }' filename
```

### 示例

**将外部变量值传递给 awk**

借助选项 `-v`，可以将外部值(并非来自 `stdin`)传递给 `awk`：

```
$ VAR=10000
$ echo | awk -v VARIABLE=$VAR '{ print VARIABLE }'
10000
```

还有另一种灵活的方法可以将多个外部变量传递给 `awk`。

```
$ var1="Variable1" ; var2="Variable2"
$ echo | awk '{ print v1,v2 }' v1=$var1 v2=$var2
Variable1 Variable2
```

当输入来自于文件而非标准输入时，使用下列命令：

```
$ awk '{ print v1,v2 }' v1=$var1 v2=$var2 filename
```

在上面的方法中，变量以键值对的形式给出，使用空格分隔`(v1=$var1 v2=$var2)`，作为 `awk` 的命令行参数紧随在 `BEGIN`，`{}` 和 `END` 语句块之后。

**用 getline 读取行**

`awk` 默认读取文件中的所有行。如果只想读取某一行，可以使用 `getline` 函数。它可以用于在 `BEGIN` 语句块中读取文件的头部信息，然后在主语句块中处理余下的实际数据。

该函数的语法为：`getline var`。变量 `var` 中包含了特定行。如果调用时不带参数，可以用 `$0`，`$1` 和 `$2` 访问文本行的内容。

```
$ seq 5 | awk 'BEGIN { getline; print "Read ahead first line", $0 } { print $0 }'
Read ahead first line 1
2
3
4
5
```

**打印从 M 行到 N 行的文本**

```
$ awk 'NR==M, NR==N' filename
$ cat filename | awk 'NR==M, NR==N'
```

**打印匹配模式的文本**

打印位于模式 `start_pattern` 与 `end_pattern` 之间的文本。`awk` 中使用的模式为正则表达式。

```
$ awk '/start_pattern/, /end_pattern/' filename
```

```
$ cat section.txt
line with pattern1
line with pattern2
line with pattern3
line end with pattern4
line with pattern5
$ awk '/pa.*3/, /end/' section.txt
line with pattern3
line end with pattern4
```

**对行过滤**

可以为需要处理的行指定一些条件：

```
$ awk 'NR < 5' 			# 行号小于5的行
$ awk 'NR==1,NR==4'		# 行号在1到5之间的行
$ awk '/linux/'			# 包含模式为linux的行(可以用正则表达式来指定模式)
$ awk '!/linux/'		# 不包含模式为linux的行
```

**设置字段分隔符**

默认的字段分隔符是空格。可以用选项 `-F` 指定不同的分隔符。

```
$ awk -F: '{ print $NF }' /etc/passwd
# 或者
$ awk 'BEGIN { FS=":" } { print $NF }' /etc/passwd
```

**读取命令输出**

`awk` 可以调用命令并读取输出。把命令放入引号中，然后利用管道将命令输出传入 `getline`。

```
$ "command" | getline output ;
```

**在 awk 中使用循环**

在 `awk` 中可以使用 `for` 循环，其格式与 C 语言中的差不多。

```
for(i=0;i<10;i++) { print $i ; }
```

**内建的字符串处理函数**

`awk` 有很多内建的字符串处理函数。

- `length(string)`：返回字符串 `string` 的长度。
- `index(string, search_string)`：返回 `search_string` 在字符串 `string` 中出现的位置。
- `split(string, array, delimiter)`：以 `delimiter` 作为分隔符，分割字符串 `string`，将生成的字符串存入数组 `array`。
- `substr(string, start-position, end-position)`：返回字符串 `string` 中以 `start-position` 和 `end-position` 作为起止位置的子串。
- `sub(regex, replacement_str, string)`：将正则表达式 `regex` 匹配到的第一处内容替换成 `replacment_str`。

## 数学运算

shell 使用 `let`，`(())` 和 `[]` 执行基本的数学运算，工具 `expr` 和 `bc` 可以执行**高级运算**。

变量赋数值也直接用 "`=`"。

### let 命令

`let` 命令可以直接执行基本算术操作。使用 `let` 时，结果变量名前无需添加 `"$"` 符号。

```
#!/bin/bash
num1=4;
num2=5;
let res=num1+num2
echo $res
```

自增自减操作：

```
let num1++
let num2--
```

### 算术操作符

操作符 "`[]`" 使用方法与 `let` 命令一致。`[]` 中也可以使用 "`$`" 前缀。

```
#!/bin/bash
num1=4;
num2=5;
res=$[ num1 + num2 ]
echo $res

res1=$[ $num2 - num1 ]
echo $res1
```

操作符 "`(())`" 也可以用于算术操作，出现在 `(())` 中的变量名之前需要加上 "`$`"。

```
#!/bin/bash
num1=4;
num2=5;
res=$(( $num1 + 50 ))
echo $res
```

### expr 命令

`expr` 也可以用于基本算术操作。

```
#!/bin/bash
num1=4;
num2=5;
res=`expr 3 + 4`
res2=$(expr $num2 + 12)
echo $res
echo $res2
```

以上方法均只支持整数运算，不支持浮点数运算。

### bc 命令

`bc` 是用于数学运算的高级工具，可以进行**浮点数**运算，也支持高级函数。

```
#!/bin/bash
# 浮点运算
echo "4 * 3.14" | bc
# 进制转换
num=9
echo "obase=2;$num" | bc
# 计算平方与平方根
echo "sqrt(100)" | bc
echo "10^10" | bc
```

```
12.56
1001
10
10000000000
```

## ssh 命令(远程连接主机)

连接运行了 `SSH` 服务器的远程主机：


- `username` 是远程主机上的用户。
- `remoteHost` 可以是域名或 IP 地址。

**通过秘钥连接**

设置 `SSH` 认证自动化需要两步：

- 在本地主机上创建 `SSH` 密钥。
- 将**生成的公钥传给远程主机并将其加入到文件** `~/.ssh/authorized_keys` 中(这一步需要访问远程主机)。

输入命令 `ssh-keygen` 创 `SSH` 密钥，指定加密算法类型为 `RSA`。

```
$ ssh-keygen -t rsa
```

`ssh-keygen` 程序会生成两个文件：`~/.ssh/id_rsa.pub` 和 `~/.ssh/id_rsa`。其中前者是公钥，后者是私钥。**公钥必须添加到想要自动登入的远程服务器的** `**~/.ssh/authorized_keys**` **文件中**。

可以使用下列命令添加密钥文件：

```
$ ssh USER@REMOTE_HOST "cat >> ~/.ssh/authorized_keys" < ~/.ssh/id_rsa.pub
```

## 校验和与核实

校验和(checksum)程序用来**从文件中生成相对较小的唯一密钥**。可以重新计算该密钥，用以检查文件是否发生改变。许多软件发行版都包含了一个校验和文件。

`md5sum` 和 `sha1sum` 程序可以对数据应用对应的算法来生成校验和。

```
$ md5sum filename
68b329da9893e34099c7d8ad5cb9c940 filename
```

`md5sum` 是一个长度为 `32` 个字符的十六进制串。

`SHA-1` 是另一种常用的校验和算法。用来计算 `SAH-1` 校验和的命令是 `sha1sum`，其用法和 `md5sum` 类似。

`md5deep` 或 `sha1deep` 命令可以遍历目录树，计算其中所有文件的校验和。
