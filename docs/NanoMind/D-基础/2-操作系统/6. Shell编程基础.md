可以通过 Shell 请求内核提供服务，Bash 是 Shell 的一种。

## 脚本基础

### shebang

Shell 脚本的第一行通常以 shebang 起始：

```
#!/bin/bash
```

它表示使用 `/bin/bash` 命令作为脚本的解释器。

### 执行脚本

有两种方式。

(1) **脚本名作为命令行参数**。这里指定脚本作为参数传给 bash 命令，这时脚本可以不用 shebang 开头。

```
$ bash test.sh
```

(2) **如果脚本以 shebang 开头，则可以直接执行脚本(需要赋予脚本执行权限)**。

```
$ chmod 755 test.sh	# 增加执行权限
$ /root/sh/test.sh	# 绝对路径执行
$ ./test.sh			# 当前目录相对路径执行
```

这里内核会读取脚本的首行并注意到 `shebang` 为 "`#!/bin/bash`"，它会识别出 "`/bin/bash`" 并执行该脚本，即：

```
$ /bin/bash test.sh
```

## 命令基础 🌟

### 命令搜索与执行顺序

命令的搜索与执行的顺序如下：

- **以绝对或相对路径**来执行指令，例如 `/bin/ls` 或 `./ls`。
- 由别名找到该指令来执行。
- **由 Bash 内置的指令**来执行。
- 按 `$PATH` 变量**指定的搜索路径的顺序找到第一个指令**来执行。

命令补全是按照 `PATH` 环境变量所定义的路径查找命令的，因此想使用 Tab 补全命令需要把命令放到 `PATH` 包含的路径中。

### 命令别名

命令别名格式：

```
$ alias # 查询已有别名
$ alias 别名='原命令'
```

查看系统已有别名。

```
$ alias
alias egrep='egrep --color=auto'
alias fgrep='fgrep --color=auto'
alias grep='grep --color=auto'
alias l.='ls -d .* --color=auto'
alias ll='ls -l --color=auto'
alias ls='ls --color=auto'
alias which='alias | /usr/bin/which --tty-only --read-alias --show-dot --show-tilde'
```

命令别名的优先级高于命令本身。**bash 中别名是临时的，永久生效需要把别名写入环境变量配置文件** `**~/.bashrc**` **中**。

### 命令连接符

当需要一次执行多个命令的时候，命令之间需要用连接符连接，不同的连接符有不同的效果。

#### 符号 ；

**格式：命令 1; 命令 2**

**没有任何逻辑关系的连接符**。当多个命令用分号连接时，**各命令之间的执行成功与否彼此没有任何影响，都会一条一条执行下去。前面执行出错后面也会执行**。

#### 符号 &&

**格式：命令 1 && 命令 2**

**逻辑与**。当用此连接符连接多个命令时，**前面的命令执行成功，才会执行后面的命令；前面的命令执行失败，后面的命令不会执行**。

Bash 的预定义变量 "`$?`" 表示**上一条指令的执行结果**，值为 0 表示执行正确，非 0 表示执行错误。

`&&` 可以用于连续操作。比如安装一个软件的多个流程顺序执行。

#### 符号 ||

**格式：命令 1 || 命令 2**

**逻辑或**。当用此连接符连接多个命令时，**前面的命令执行成功，则后面的命令不会执行。前面的命令执行失败，后面的命令才会执行**。

判断一个命令是否正确执行：

```
$ 命令 && echo "yes" || echo "no"
```

#### 符号 |

管道符。当用此连接符连接多个命令时，**前面命令执行的正确输出，会交给后面的命令继续处理**。若前面的命令执行失败，则会报错，若后面的命令无法处理前面命令的输出，也会报错。

## 变量 🌟

### 变量基础

变量名由字母，数字和下划线构成。通常大写字母命名环境变量，驼峰命名或小写字母命名其他变量。

**Bash 的变量可以声明为数组和整数数字**。注意数字类型没有浮点数。如果不进行声明，默认是字符串类型。变量的声明使用 `declare` 命令：

```
$ declare [-aixr] variable
-a :  定义为数组类型
-i :  定义为整数类型
-x :  定义为环境变量
-r :  定义为 readonly 类型
```

### 变量操作

#### 变量赋值

使用 "`=`" 为**变量赋值**。注意这里**等号两边没有空格**！如果**等号两边加上空格表示\*\***等量关系测试\*\*。

```
varName=value
```

**变量内容(value)如果包含空格，必须使用双引号或单引号表示一个整体**。

- **双引号内的特殊字符可以保留原本特性**，例如 `x="lang is $LANG"`，则 x 的值为 `lang is zh_TW.UTF-8`；
- **单引号内的字符串就是原本的普通字符串**，例如 `x='lang is $LANG'`，则 x 的值为 `lang is $LANG`。

注意：可以使用 `` `指令` `` 或者 `$(指令)` 的方式将指令的执行结果赋值给变量。例如 `version=$(uname -r)`，则 `version` 的值为 "`4.15.0-22-generic`"。

#### 访问变量值

变量名前加 `"$"` 符号可以**访问变量内容**，也可以用 `${}` 的形式。

```
# 变量赋值
var="test"
# 输出变量值
echo $var
echo ${var}
```

可以在 `echo`，`printf` 命令或其他命令的双引号中引用变量值并进行输出。

```
#!/bin/bash
fruit=apple
count=5
echo "We have $count ${fruit}."
```

获取变量包含的字符数。

```
length=${#var}
```

例子：

```
$ var=1234567
$ echo ${#var}
7
```

位置参数变量主要是向脚本中传递数据，变量名不能自定义，变量作用是固定的。

### 环境变量配置文件

环境变量配置文件中主要是定义对系统的操作环境生效的默认环境变量，比如：`PATH`，`PS1`，`HOSTNAME` 等。

```

/etc/profile.d/*.sh

/etc/bashrc
```

上面一共 5 个配置文件，在 `/etc` 目录下的三个文件如果改变，所有用户都要生效，另外 2 个只对当前用户生效。

## 特殊符号 🌟

Bash 中的一些特殊符号。

|        |                                                                                                              |
| ------ | ------------------------------------------------------------------------------------------------------------ |
| 符号   | 含义                                                                                                         |
| `''`   | 单引号，在单引号中所有的**特殊符号都没有了特殊的意义**。就是一个**普通字符串**了。                           |
| `""`   | 双引号，在双引号中 `"$"`，`` "`" `` 和 `""`，分别表示 "**调用变量的值**"，"**引用命令**"，"**转移符**"的含义 |
| ` `` ` | 反引号，**反引号括起来的内容是系统命令**，在 bash 中会优先执行，和 `$()` 效果一样，不过反引号容易看错        |
| `$()`  | 引用系统命令                                                                                                 |
| `()`   | 用于一串命令执行时，`()` 中的命令会在子 shell 中执行                                                         |
| `{}`   | 用于一串命令执行时，`{}` 中的命令会在当前 shell 中执行，也可以用于变量变形与替换                             |
| `$`    | 用于调用变量的值，如需要调用变量 name 的值时，需要用 `$name` 的方式得到变量的值                              |

**字段分隔符**

IFS(内部字段分隔符)是一个环境变量，保存了**用于分隔的字符**，是 shell 中默认使用的定界字符串。IFS 默认值为空白字符(换行符，制表符或空格等)。

读取 CSV 数据实例。

```
#!/bin/bash
data="Jack,Alice,Lucy"
# 保存原始IFS
oldIFS=$IFS
# 修改IFS
IFS=,
# 遍历data
for item in $data;
do
	echo $item
done

IFS=oldIFS
```

这里 IFS 被设置为逗号，于是 shell 将逗号看做定界符，因此变量 `"$item"` 在每次迭代中读取由逗号分隔的子串作为变量值。

## 文件描述符 🌟

### 命令退出状态

命令执行完成后，无论成功与否，其状态都保存到特殊变量 `"$?"` 中。

- **命令执行成功。返回为 0 的退出状态**。
- **命令执行失败。返回为非 0 的退出状态**。

命令执行完成后马上访问变量 `"$?"` 即可知道命令执行的状态。

### 标准输入输出

文件描述符是与输入与输出流相关联的整数。可以将某个文件描述符的内容重定向到另一个文件描述符中。

文件描述符 0，1，2 由系统预留，分别表示：

- 0：标准输入(stdin)
- 1：标准输出(stdout)
- 2：标准错误(stderr)

### 数据流重定向

#### 重定向符

重定向指使用文件代替标准输入，标准输出和标准错误输出。

|                      |          |               |
| -------------------- | -------- | ------------- |
| **类型**             | **代码** | **运算符**    |
| 标准输入(stdin)      | 0        | `<` 或 `<<`   |
| 标准输出(stdout)     | 1        | `>` 或 `>>`   |
| 标准错误输出(stderr) | 2        | `2>` 或 `2>>` |

可以通过重定向操作符将命令输出重定向保存到文件中，而不是终端。使用一个箭头表示以覆盖的方式重定向，而有两个箭头的表示以追加的方式重定向。

```
$ echo "This is the content." > temp.txt	# 覆盖写
$ echo "This is the content." >> temp.txt	# 追加写
```

可以将不需要的标准输出以及标准错误输出重定向到文件 `"/dev/null"` 中，它是一个特殊的设备文件，会丢弃接收到的任何数据。

如果需要将标准输出以及标准错误输出同时重定向到一个文件，需要将某个输出转换为另一个输出，例如 `2>&1` 表示将标准错误输出转换为标准输出。

```
$ find /home -name .bashrc > list 2>&1
```

#### tee 命令(双向输出重定向)

`tee` 命令可以将数据重定向到文件中，同时提供一份重定向数据的副本作为管道中后续命令的 `stdin`。也就是 `tee` 命令可以从 `stdin` 中读取数据，然后将输入数据重定向到 `stdout` 以及一个或多个文件中。

输出重定向会将输出内容重定向到文件中，`tee` 不仅能够完成这个功能，还能保留屏幕上的输出。因此使用 `tee` 指令，一个输出会同时传送到文件和屏幕上。

```
$ command | tee FILE1 FILE2 | otherCommand
```

```
$ tee [-a] file
```

## 管道 🌟

管道是将一个命令的标准输出作为另一个命令的标准输入，在数据需要经过多个步骤的处理之后才能得到想要的内容时就可以使用管道`(|)`。在命令之间使用 `"|"` 分隔各个管道命令。

```
$ ls -al /etc | less
```

命令输入通常来 `stdin` 或参数。输出可以发送给 `stdout` 或 `stderr`。当组合多个命令时，通常将 `stdin` 用于输入，`stderr` 用于输出。这些命令被称为过滤器。使用管道操作符 `"|"` 连接每个过滤器。

```
$ cmd1 | cmd2 | cmd3
```

例子：

(1) 重定向内容到文件中。

```
$ ls | cat n > out.txt
```

(2) 将命令序列的输出赋给变量。有两种形式。

```
$ cmdOutput=$(commands)
$ cmdOutput=`commands`
```

```
#!/bin/bash
out=$(ls)
echo $out

out=`ls`
echo $out
```

## 数组与关联数组

普通数组以整数作为索引，关联数组使用字符串作为索引。

### 普通数组

**定义数组**

**单行中使用数值列表可以定义一个数组，这些值会存储到以 0 为起始索引的连续数组中**。

```
arrayVar=(v1 v2 v3 v4)
```

也可以通过 "**索引-值**" 的形式定义数组。

```
arrayVar[0]="test0"
arrayVar[1]="test1"
arrayVar[2]="test2"
arrayVar[3]="test3"
```

**读取数组元素**

通过索引读取数组指定位置元素。

```
$ echo ${arrayVar[1]}
test1
```

以列表形式打印数组所有元素。

```
# 写法等价
$ echo ${arrayVar[*]}
$ echo ${arrayVar[@]}
test0 test1 test2 test3
```

获取元素个数：

```
$ echo ${#arrayVar[*]}
```

### 关联数组

关联数组可以使用任意文本作为数组索引。

需要使用声明语句将一个变量定义为关联数组。

```
$ declare -A assArray
```

声明后添加元素的方式与普通数组类似。

```
$ newmap[name]="Jack"
$ newmap[designation]=SSE
$ newmap[company]="Google"
```

获取元素值。

```
$ echo ${newmap[company]}
$ echo ${newmap[name]}
```

## 循环

循环可用于迭代处理系列值。

**面向列表的循环**

```
for var in list;
do
    commands;	# 使用变量$var
done
```

`list` 可以是一个字符串或值序列。

可以使用 `echo` 命令生成各种值序列。

```
# 生成1-50的值序列
$ echo {1..50}
# 生成大小写字母序列
$ echo {a..z} {A..Z}
```

如：

```
#!/bin/bash
for var in {1..10};
do
    echo $var;	# 使用变量$var
    sleep 1;	# 暂停一秒
done
```

**迭代指定范围的数字**

```
for((i=0;i<10;i++))
{
    commands;	# 使用变量$i
}
```

如：

```
#!/bin/bash
for((i=0;i<10;i++))
{
    echo $i;	# 使用变量$i
    sleep 1;
}
```

**循环到满足条件为止**

```
while condition
do
    statements;
done
```

用 true 作为条件可以无限循环。

```
#!/bin/bash
# 计算从m加到n的值
read m
read n
sum=0

while ((m<=n))
do
    ((sum+=m))
    ((m++))
done

echo "The sum is $sum"
```

## 比较与测试 🌟

程序流程控制由比较语句和测试语句处理。

### 条件判断

**if 条件**

```
if condition;
then
    commands;
fi
```

**else if 与 else**

```
if condition;
then
    commands;
else if condition; then
    commands;
else
    commands;
fi
```

可以通过逻辑运算符简化上述结构。

- `&&` 是逻辑与运算符。
- `||` 是逻辑或运算符。

```
[ condition ] && action;	# 如果condition为真,则执行action
[ condition ] || action;	# 如果condition为假,则执行action
```

### 算术比较

比较条件通常被放置在封闭的中括号 `[ ]` 内。要注意 `[ ]` 与**操作数之间有一个空格**。

操作符：

- `eq`：等于。
- `ne`：不等于。
- `gt`：大于。
- `lt`：小于。
- `ge`：大于等于。
- `le`：小于等于。

如有多个比较条件可以用逻辑操作符。

- `-a`：逻辑与操作符(and)。
- `-o`：逻辑或操作符(or)。

```
[ $var -gt 0 -a $var2 -le 2 ]  # 当变量大于0且小于等于2时为真
```

### 文件系统相关测试

可以使用不同的条件标志测试各种文件系统相关的属性。

**文件名/文件类型相关**

- `[ -e $var ]`：如果该文件名存在，则返回真。
- `[ -f $var ]`：如果该文件名存在且为正常的文件，则返回真。
- `[ -d $var ]`：如果给定变量名为目录，则返回真。
- `[ -S $var ]`：如果给定变量名为一个 socket 文件，则返回真。
- `[ -L $var ]`：如果给定变量名为一个连接文件，则返回真。

**文件权限相关**

- `[ -r $var ]`：如果给定变量名包含的文件可读，则返回真。
- `[ -w $var ]`：如果给定变量名包含的文件可写，则返回真。
- `[ -x $var ]`：如果给定变量名包含的文件可执行，则返回真。

例子：

```
#!/bin/bash

fpath="/etc/passwd"
if [ -e $fpath ]; then
    echo File exist;
else
    echo File not exist;
fi
```

### 字符串比较

进行字符串比较用双中括号，有时候单中括号会产生错误。

- `[[ $str1 = $str2 ]]`：当 str1 等于 str2 时，返回真。
- `[[ $str1 == $str2 ]]`：等价写法。当 str1 等于 str2 时，返回真。
- `[[ $str1 != $str2 ]]`：当 str1 不等于 str2 时，返回真。

字符串通过 ASCII 码进行比较。

- `[[ $str1 > $str2 ]]`：当 str1 字母序大于 str2 时，返回真。
- `[[ $str1 < $str2 ]]`：当 str1 字母序小于 str2 时，返回真。
- `[[ -z $str1 ]]`：当 str1 为空串时，返回真。
- `[[ -n $str1 ]]`：当 str1 为不空串时，返回真。

使用逻辑运算符 `&&` 和 `||` 能组合多个条件。

例子：

```
#!/bin/bash

str1="Not Empty"
str2=""
if [[ -n $str1 ]] && [[ $str1 != $str2 ]];
then
    echo YES
fi
```

### test 命令

**test 命令可以用于测试条件，可以避免使用过多的括号**。进行比较时 `[[]]` 中的条件同样适用于 test 命令。

下面两句等价。

```
if [ $var -eq 0 ]; then echo OK; fi
if test $var -eq 0 ; then echo OK; fi
```

## 函数 🌟

### 函数定义

函数的定义包含 `function` 命令，函数名，括号以及函数体。

```
function fname()
{
    statements;
}
```

或者

```
fname()
{
    statements;
}
```

或者

```
fname() { statements; }
```

### 函数调用

**通过函数名调用函数**。

```
$ fname ;  # 调用函数
```
